Tarea extra: Desarrollo de una aplicación ToDo completa (für Carlos)

Objetivo del proyecto

El objetivo de este proyecto es desarrollar una aplicación web de tareas pendientes totalmente funcional. La aplicación debe permitir a los usuarios crear, gestionar, editar y eliminar tareas. Los datos se almacenarán en una base de datos MySQL. La aplicación está realizada con el framework Express.js y utiliza EJS (Embedded JavaScript Templates) para el renderizado del lado del servidor de la interfaz de usuario.

Tecnologías

Backend: Node.js con Express.js

Motor de plantillas: EJS

Base de datos: MySQL

Frontend: HTML5, CSS3, JavaScript opcional para interactividad

Conexión a base de datos: Paquete MySQL node (por ejemplo, mysql2 o sequelise)

Requisitos funcionales

Gestión de tareas

  • Creación de nuevas tareas
  • Edición de tareas existentes
  • Eliminación de tareas
  • Marcar tareas como completadas/no completadas
  • Categorización o priorización de tareas (por ejemplo, "Urgente", "Más tarde")

Interfaz de usuario

  • Vista general de todas las tareas con visualización del estado
  • Posibilidad de filtrar (por ejemplo, "todas", "abiertas", "completadas")
  • Diseño adaptable (también puede utilizarse en dispositivos móviles)

Persistencia de datos

Todas las tareas y su estado se almacenan en una base de datos MySQL

Base de datos estructurada con al menos dos tablas:

  • Tareas (por ejemplo, id, título, descripción, estado, prioridad, created_at, updated_at)
  • Usuarios opcionales (para operaciones multiusuario)

Enrutamiento y lógica de servidor

  • Rutas compatibles con REST para el tratamiento de datos (GET, POST, PUT, DELETE)
  • Separación de rutas, controladores y lógica de base de datos

Requisitos no funcionales

Estructura de código limpia y modular (se recomienda una arquitectura MVC)

Plantillas EJS legibles con un diseño razonable

Tratamiento de errores en formularios y accesos a bases de datos

Documentación del proyecto (README.md con instrucciones de instalación y uso)

Opciones de extensión (opcional)

Autenticación de usuarios (inicio de sesión/registro)

Uso de AJAX para la recarga dinámica de tareas

Ordenación de tareas mediante arrastrar y soltar

Exportación de la lista de tareas como PDF o CSV

Si lo desea, también puedo ayudarle a diseñar la estructura de la base de datos, la estructura de directorios o los fragmentos de código iniciales.
