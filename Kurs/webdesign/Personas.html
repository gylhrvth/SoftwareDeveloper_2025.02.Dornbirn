Unser Arbeitsprozess – Ein Lernkreislauf für erfolgreiche Entwickler

In der Softwareentwicklung folgt der Arbeitsprozess einem stetigen Kreislauf, der euch hilft, systematisch zu lernen und besser zu werden. Jeder Schritt baut auf dem vorherigen auf und führt euch zu einem tieferen Verständnis und besseren Ergebnissen.

🔍 1. Recherche – Bevor wir eine Lösung entwickeln, müssen wir das Problem genau verstehen. Wir analysieren Anforderungen, suchen nach bestehenden Lösungen und sammeln Wissen.

Für Recherce stehen unzählige Werkzeuge zur Verfügung

  • Dein Trainer & die anderen Kursteilnehmern
  • ChatGPT
  • GitHub Copilot
  • Google
  • Bing
  • Sachbücher

Um gute Ergebnis für die Recherce zu bekommen, musst du klare, spezifische Ziele formulieren. Die Kontext bietet einen Hintergrund an, klar formulierte Wunsch auf dem Ergebnis.

📝 2. Eigene Worte finden & Begriffe klären – Um das Gelernte wirklich zu verstehen, formulieren wir es in unseren eigenen Worten. Wir definieren zentrale Begriffe, strukturieren unser Wissen und entwickeln daraus einen klaren Lösungsansatz.

💻 3. Implementieren – Jetzt wird es praktisch! Wir setzen unser Konzept um, programmieren die Lösung und testen sie schrittweise, um Fehler zu erkennen und zu optimieren.

📖 4. Erfahrung dokumentieren – Nach der Umsetzung reflektieren wir unsere Arbeit. Was hat gut funktioniert? Wo gab es Herausforderungen? Die Erkenntnisse dokumentieren wir, um daraus zu lernen und unseren nächsten Zyklus zu verbessern.

Dieser Kreislauf wiederholt sich immer wieder – mit jeder neuen Aufgabe werdet ihr effizienter, strukturierter und sicherer in der Entwicklung. 🚀

Aufgabe 1: Recherche

Definition und Begriffe

  • Was ist eine Persona?
  • Woher stammt das Wort?
  • Wie kann ich eine Persona vorstellen?
  • Wie sieht eine Persona aus?

Nutzen

  • Wozu brauchen wir eine Persona?
  • Wozu brauchen wir als Software Entwickler eine Persona?
  • Wie können wir nutzen als Software Entwickler eine Persona?

Prozess der Erstellung

  • Welche Schritte brauche ich, um eine Persona zu erstellen?
  • Welche Aspekte muss ich bei Erstellung einer Persona betrachten?
  • Wie dokumentiere ich meine Persona?

Qualitätsmerkmale

  • Was sind die Merkmale von schwachen/ungeigneten Personas?
  • Was sind die Merkmale von gute Personas?

Aufgabe 2: Eigene Worte finden

Dokumentiere deine Recherege auf deine "Journal" Seite!

Vorbereitung

  1. Visual Studio Code herunterladen & installieren
  2. Git herunterladen und installieren
  3. Auto-Save, Live Server und Git Graph als Addon in Visual Studio installieren
  4. Checkout das Projekt https://github.com/gylhrvth/SoftwareDeveloper_2025.02.Dornbirn
  5. Projektstruktur kennenlernen
  6. Eintrag in Hello World! ändern
  7. Ergenis im Live Server überprüfen
  8. Gemeinsam das erste GIT (Commit-Pull-Push) machen

Dokumentation

  1. Erfahrung in Journal Seite eintragen
  2. mit Git Commit die Änderungen festlegen
  3. mit Git Push die Änderungen auf dem gemeinsame Repository hochladen

Aufgabe 3: Implementieren

Erstelle eine Persona

  1. Erstelle eine erste Version der Persona
  2. Was sind die guten Eigenschaften der Perosna?
  3. Was sind noch weitere Potenziale in der Persona?
  4. Bespreche die Persona, mit einem anderen TN!
  5. (optional) Erweitere/Korrigiere deine Persona!
  6. Stelle die Persona für alle vor.
