Aufgabe 1: Recherche

Definition und Begriffe

  • Was ist eine VCS (Version Control System)?
  • Was ist Git?
  • Was ist ein Git Commit?
  • Was ist ein Git Repository?
  • Was ist der Unterschied zwischen ein lokal und ein remote Repository?
  • Was ist ein Merge?
  • Was ist ein Merge Conflict?

Nutzen

  • Wozu brauchen wir Version Control?
  • Welche Vorteile bietet ein Version Control?
  • Was sind die potenzielle Schwierigkeiten bei der Umsetzung eines VCS?
  • Was ist GIT?
  • Was ist Git CLI?
  • Was ist GitHub?
  • Was sind GitHub Pages?

Prozess der Erstellung

  • Welche Schritte muss ich in Visual Studio Code machen?
  • Welche Befehle muss ich in Terminal machen?
  • Was ist der Unterschied zwischen Befehle im Terminal oder Schritte in Visual Studio Code?
  • Mach eine Änderung in deinem "Journal" und geh bewusst durch an jeden Schritt...
  • Wann kann ein Merge automatisch stattfinden?
  • Was kann/muss ich machen im Fall eines Merge conflicts?

Qualitätsmerkmale

  • Woher weiß ich, dass ich VCS richtig verwende?
  • Was sind die typische Problemen/Schwierigkeiten von Junior Entwicklern?

GIT CLI Befehle

  • git add
  • git commit
  • git pull
  • git push
  • git init
  • git clone
