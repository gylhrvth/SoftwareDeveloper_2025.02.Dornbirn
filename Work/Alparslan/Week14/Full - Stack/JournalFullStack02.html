Typescript

  • Was ist Typescript? Woher kommt es?

    Typescript ist eine von Microsoft entwickelte Programmiersprache, die auf JavaScript basiert, aber statische Typisierung und moderne Features wie Interfaces, Enums und Klassen bietet.

    📝 Kurz gesagt:

    • Typescript = JavaScript + Typen
    • Wird zu plain JavaScript kompiliert (für Browser oder Node.js)
    • Entwickelt von Microsoft, erstes Release: 2012


  • Was sind die Vorteile / Nachteile von Typescript gegenüber JavaScript?

    ✅ Vorteile von TypeScript gegenüber JavaScript:

    1. Statische Typen → Fehler früher erkennen
    2. Bessere IDE-Unterstützung → Autovervollständigung, Refactoring
    3. Klarere Struktur durch Interfaces & Klassen
    4. Große Projekte wartbarer und skalierbarer
    5. Modernes JavaScript + Extras (z.B. Enums, Zugriffskontrolle)

    ❌ Nachteile von TypeScript:

    1. Build-Schritt nötig → TypeScript muss nach JavaScript kompiliert werden
    2. Mehr Schreibarbeit → Vor allem bei kleinen Projekten
    3. Einarbeitung → Besonders für Anfänger
    4. Konfiguration (tsconfig.json) kann komplex sein

    Fazit:

    • Für große Projekte: Sehr empfehlenswert
    • Für kleine Skripte oder Prototypen: JavaScript reicht oft aus

  • Wie erstelle ich ein Projekt mit Typescript?

    1. Neues Projektverzeichnis erstellen
      2. mkdir mein-ts-projekt cd mein-ts-projekt

    2. npm initialisieren
      3. npm init -y

    3. Typescript installieren
      4. npm install typescript --save-dev

    4. TypeScript - Konfiguration erstellen
      5. npx tsc --init

      → Dadurch entsteht eine tsconfig.json, in der du Einstellungen wie Ziel-JS-Version oder Ordnerstrukturen anpassen kannst.



  • Wie verwende ich Type Annotation für Variablen / Parameter / Rückgabewert?

    In TypeScript kannst du Typen (Type Annotations) an Variabeln, Funktionsparametern und Rückgabewerten angeben.

    1. 🟦 Variabeln:
      2. let age: number = 25; let username: string = "Anna"; let isOnline: boolean = true;

    2. 🟨 Parameter:
      3. function greet(name: string){ console.log("Hello " + name); }

    3. 🟩 Rückgabewert:
      4. function getAge(): number{ return 30; }

      💡 Kombiniert:

      function add(x: number, y: number): number{ return x + y; }

      TypeScript erkennt oft die Typen automatisch - aber Annotationen helfen beim Verständnis und bei Fehlern.



  • Wie beschreibe ich Arrays in Typescript?

    In TypeScript gibt es zwei Hauptmethoden, um Arrays zu typisieren:

    1. Mit eckigen Klammern type[ ]:
      2. let zahlen: number[] = [1, 2, 3]; let namen: string[] = ["Anna", "Ben"];

    2. Mit Generics Array:
      3. let zahlen: Array<number> = [1, 2, 3]; let aktiv: Array<boolean> = [true, false];

    3. 💡 Beispiel mit Funktionen:
      4. function printNames(namen: string[]): void { namen.forEach(n => console.log(n)); }


  • Wie beschreibe ich Objekte in Typescript?

    In TypeScript beschreibst du Objekte mit Typeannotationen für die Eigenschaften - entweder inline oder mit type oder interface.

    1. 🟦 Inline-Typdefinition:
      2. let user: { name: string; age: number } = { name: "Anna", age: 30 };

    2. 🟨 Mit type:
      3. type User = { name: string; age: number; }; let u: User = { name: "Tom", age: 25 };

    3. 🟩 Mit interface:
      4. interface Product { id: number; title: string; price: number; } let item: Product = { id: 1, title: "Buch", price: 12.99 };

      type und interface sind sehr ähnlich - interface ist besser für Klassen und Erweiterung geeignet, type flexibler für komplexe Typkombinationen.



  • Wie verwende ich Type in Typescript?

    In TypeScript kannst du mit dem type-Schlüsselwort eigene Typdefinitionen erstellen - ähnlich wie mit interface, aber mit mehr Flexibilität.



  • Was ist der Unterschied zwischen Interface und Type in Typescript?

    Sowohl interface als auch type können verwendet werden, um die Struktur von Objekten zu definieren:

    interface User { name: string; age: number; } type UserType = { name: string; age: number; };

    📝 Beides funktioniert gleich, wenn es nur um einfache Objektformen geht.


    🔍 Unterschiede:

    Aspekt interface type
    Erweiterbarkeit ✅ kann erweitert/zusammengeführt werden ❌ kann nicht zusammengeführt werden
    Verwendung mit Union ❌ nicht möglich ✅ möglich (z. B. type Role = "admin" | "user")
    Komplexe Typen ❌ nur Objekte ✅ auch für union, tuple, primitive
    Mehrfach-Implementierung ✅ in Klassen erlaubt ✅ auch möglich (mit Einschränkungen)

    Verwende interface, wenn du Objekte oder Klassen beschreibst und Erweiterung brauchst.

    Verwende type, wenn du mehr Flexibilität brauchst - z.B. für Union Types, Primitive, Tuples.



  • Wie verwende ich Union Types?

    Union Types erlauben es, dass eine Variable oder ein Parameter mehrere Typen annehmen kann. Du verwendest das |-Zeichen, um die erlaubten Typen zu kombinieren.

    let id: number | string; id = 123; // gültig id = "abc"; // auch gültig function printStatus(status: "success" | "error") { console.log("Status:", status); } printStatus("success"); printStatus("error");

    📝 Fazit: Union Types machen deinen Code flexibler, weil mehrere Typen erlaubt sind.



  • Wie verwende ich Intersection Types?

    Intersection Types erlauben es, mehrere Typen zu einem neuen Typ zu kombinieren. Du verwendest das &-Zeichen, um die Typen zu "verschneiden". Das Ergebnis muss alle Eigenschaften aller Typen enthalten.

    type Person = { name: string }; type Worker = { company: string }; type Employee = Person & Worker; let mitarbeiter: Employee = { name: "Anna", company: "Tech AG" };

    📝 Fazit: Intersection Types sind praktisch, wenn ein Objekt die Eigenschaften mehrerer Typen gleichzeitig haben soll.



  • Was sind Generic? Wie verwende ich sie?

    Generics sind ein TypeScript-Feature, mit dem du Funktionen, Klassen oder Typen so definieren kannst, dass sie mit verschiedenen Typen arbeiten können. Sie machen deinen Code wiederverwendbar und typsicher.

    function identity(value: T): T { return value; } let zahl = identity(42); // Typ: number let text = identity("Hallo"); // Typ: string

    📝 Fazit: Mit Generics kannst du flexibel und sicher mit beliebigen Typen arbeiten, ohne auf Typüberprüfung zu verzichten.



  • Was sind enum in Typescript? Wie verwende ich sie?

    Enums (Aufzählungstypen) sind ein Feature in TypeScript, mit dem du eine Gruppe von benannten Konstanten definieren kannst. Sie machen den Code lesbarer und helfen, feste Werte übersichtlich zu verwalten.

    enum Color { Red, Green, Blue } let c: Color = Color.Green; console.log(c); // Ausgabe: 1

    📝 Fazit: Enums sind praktisch, wenn du mit festen Wertemengen (z.B. Farben, Status, Wochentage) arbeiten möchtest.
