    Definition und Begriffe!

    Was ist ein VCS (Versions Control System)?
    Ein VCS ist wie ein Logserver, welcher jeden Schritt und jede Änderung an einem Projekt speichert.
    Diese Schritte oder Änderungen können dann zu jederzeit nachverfolgt werden, wer wann welche Änderung warum vorgenommen hat.
    Zudem kann man auch jederzeit eine frühere Version des Projektes wiederherstellen.

    Ein VCS ist in Entwicklerumgebungen unverzichtbar, weil mehrere Entwickler zeitgleich am selben Projekt arbeiten und sich nicht überschreiben wollen.

    Nutzen!

    Was ist Git?
    Logspeicher
    GIT ist auch ein VCS (Versions Control System) allerdings verteilt, d.h. Es hilft Entwicklern Änderungen am Code, Projekten zu verfolgen, verwalten und zu koordinieren.
    Was ist ein Git Commit?
    Speicherung, Teilung auf Visual Studio Code
    Ein Commit ist ein Snapshot oder Speicherung von Änderungen an Dateien oder Projekten.
    Es speichert den aktuellen Stand der Änderungen und ermöglicht es, diese später nachzuvollziehen oder zu einem früheren Zustand zurückzukehren.

    Was ist ein Git Repository?
    Respositories sind (Projekt-Daten)
    Ein Git Repository speichert alle Dateien und die Versionshistorie eines Projekts. Es ermöglicht das Verfolgen von Änderungen und die Zusammenarbeit, sowohl lokal auf deinem Computer als auch remote auf einem Server (z. B. GitHub).

    Was ist der Unterschied zwischen ein lokal und ein remote Repository? Ein lokales Repository befindet sich auf deinem Computer und speichert alle Änderungen an deinem Projekt lokal auf dem Rechner, während ein remote Repository auf einem externen Server (z. B. GitHub) liegt und dazu dient, den Code zu teilen und mit anderen Entwicklern zusammenzuarbeiten. Änderungen können lokal vorgenommen und dann mit "git push" ins remote Repository hochgeladen oder mit "git pull" heruntergeladen werden.

    Was ist ein Merge?
    Merge = Zusammenführung
    Ein Merge in Git ist der Prozess, bei dem Änderungen aus zwei verschiedenen Branches zusammengeführt werden. Wenn du an einem Feature-Branch arbeitest und deine Änderungen in den Haupt-Branch (z. B. main oder master) einfügen möchtest, führst du einen Merge durch.

    Was ist ein Merge Conflict?
    Merge Conflict = Konflikt von Codezeilen
    Ein Merge Conflict tritt auf, wenn Git zwei Änderungen an denselben Zeilen einer Datei nicht automatisch zusammenführen kann. Dies passiert, wenn verschiedene Branches dieselbe Stelle im Code unterschiedlich geändert haben..

    Was sind Branches?
    Branches = Bereiche
    Branches in Git sind separate Entwicklungszweige innerhalb eines Repositories, die es ermöglichen, an verschiedenen Features, Bugfixes oder Experimenten parallel zu arbeiten, ohne den Hauptcode zu beeinträchtigen.

    Was ist GIT CLI?
    Git CLI (Command Line Interface) ist das Kommandozeilenwerkzeug von Git. Es ermöglicht dir, Git-Befehle direkt über die Eingabeaufforderung (Terminal) auszuführen, z. B.: • "git init" – Repository erstellen
    • "git add" – Änderungen hinzufügen
    • "git commit" – Änderungen speichern
    • "git push" – Änderungen hochladen

    Was ist GitHub?
    GitHub ist eine Online-Plattform zum Hosten von Git-Repositories. Entwickler nutzen GitHub, um ihre Projekte online zu speichern, mit anderen zu teilen und gemeinsam an Software zu arbeiten.
    Zusammengefasst: GitHub ist eine Cloud-Plattform für Git-Repositories zur Zusammenarbeit und Versionskontrolle.

    Was sind GitHub Pages?
    GitHub Pages ist eine Funktion von GitHub, mit der du Websites direkt aus einem GitHub-Repository veröffentlichen kannst. Besonders für einfache Projekte, Portfolios oder Dokumentationen geeignet.
    Zusammengefasst: GitHub Pages ermöglicht dir, statische Websites direkt von GitHub aus online zu stellen.

    Aufgabe 1: Recherche

    Definition und Begriffe!

    Was ist UX / UI ?
    1. UI ist das User Interface (Benutzeroberfläche)
    - UI ist optisches Design, wie sieht die APP oder Website aus
    - UI ist z.B. auch ob die Farbpalette, Buttons, Schriftart und das Layout in einer Website gross genug sind
    Also UI ist das was der User sieht.

    2. UX – User Experience (Benutzererfahrung)
    - UX ist die Benutzererfahrung, wie sich die Website anfühlt
    - UX ist die usability, Logik und Nutzbarkeit einer Website
    Also UX ist das was der User fühlt, beim Verwenden der App/Website.

    Woher Stammt die Begriffe?
    1970 - CLI Textbasierte Benutzeroberfläche
    1980 - GUI Grafische Oberfläche (Macintosh, Windows)
    1990 - UX Begriff entsteht (Donald Norman, Apple)
    2000 - UX und UI werden immer wichtiger mit Smartphones, Apps, Webdesign
    Heute - UX/UI Design ist ein eingenständiger Berufszweig

    Also:
    UI entstand aus der Computernutzung (wie wir mit der Maschine interagieren).
    UX entstand aus der Design- und Nutzerpsychologie (wie fühlt sich die gesamte Erfahrung für den Menschen an).




    Wie sieht ein UX/UI Design aus?
    Es ist ein Plan welches anhand einer Anfroderung z.B. vom Kunden erstellt wurde.
    Also eigentlich ein Design(APP oder Website) mit vielen Grafiken, Buttons und Texten, das wichtigste Merkmal ist jedoch,
    dass man immer auf eine weitere Site geleitet wird sobald man etwas anklickt.
    z.B. man klickt auf einem Stellenportal auf "Jetzt Bewerben" und wird dann auf die Firmenwebsite weitergeleitet.



    Nutzen!

    Warum ist UX / UI wichitg für die heutige Zeit?
    UI ist für die heutige Zeit sehr wichtig, weil wir fast alles digital machen,
    vom Einkaufen bis Banking.
    Gutes UI sorgt für schöne und übersichtliche Oberflächen, auf Smartphones und PCs,
    womit wir heutzutage alles Mögliche erledigen.
    Gutes UX hingegen stellt sicher, dass Bedienung einfach, schnell und angenehm sind.
    Wenn eine APP oder Website kompliziert wäre, würden die User abspringen und zur Konkurrenz gehen.

    Gutes UX/UI spart Zeit, macht Spass und entscheidet oft, ob ein Produkt erfolgreich ist oder nicht,
    bestes Beispiel hierfür sind Apple oder Netflix, beide sehen gut aus (UI) und sind einfach zu bedienen (UX).



    Do’s and dont’s in UX und UI?
    Dos (Mach das):
    • Einfach & intuitiv: Nutzer sollen sich schnell zurechtfinden.
    • Klare Struktur: Wichtige Infos hervorheben, logische Navigation.
    • Lesbarkeit: Große Schrift, guter Kontrast, genug Abstand.
    • Einheitliches Design: Farben, Buttons und Schriftarten durchgehend gleich.
    • Mobilfreundlich: Optimierung für Smartphones.

    Don’ts (Vermeide das):
    • Überladenes Design: Zu viele Farben, Texte oder Animationen.
    • Komplizierte Abläufe: Zu viele Klicks oder unnötige Formulare.
    • Schlechte Lesbarkeit: Kleine Schrift, schwacher Kontrast.
    • Verwirrende Navigation: Wichtige Funktionen schwer auffindbar.
    • Inkonsistenz: Unterschiedliche Stile und Buttonformen mischen.



    Wie betrifft UX/UI uns als Entwickler?
    Für uns Entwickler ist UX/UI-Design ausschlaggebend, weil es beeinflusst, ob unsere Software erfolgreich ist.
    Gutes UX/UI spart Zeit bei der Entwicklung, verbessert die Zusammenarbeit mit Designern und sorgt für zufriedene Kunden sowie Nutzer. Es hilft uns, klare Vorgaben zu haben und Fehler oder Nachbesserungen zu vermeiden.



    Prozesse der Erstellung!

    Welche groben Schritte brauche ich um ein UI/UX Design zu erstellen?
    1. Forschung & Analyse
    • Zielgruppenanalyse: Wer wird die App/Website nutzen? Welche Bedürfnisse haben sie?
    • Wettbewerbsanalyse: Schau dir ähnliche Produkte an, um zu sehen, was funktioniert.

    2. User Cases & Anforderungsanalyse
    • User Cases erstellen: Beschreibe, was Nutzer tun möchten (z. B. "Der Nutzer will einen Artikel kaufen").
    • Features und Funktionen festlegen: Was muss die App/Website können? Muss/Kann Ziele festlegen.

    3. Wireframes erstellen (UX)
    • Skizziere grobe Layouts und Strukturen der App/Website (z. B. mit Papier oder digitalen Tools wie Figma).

    4. Prototyping
    • Erstelle interaktive Prototypen, um den Ablauf und die Funktionalität zu testen (Tools wie Figma, Adobe XD).

    5. Visuelles Design (UI)
    • Farben, Typografie und Icons: Wähle Design-Elemente aus, die zur Marke und Zielgruppe passen.
    • Design verfeinern: Mach das Design hübsch, konsistent und benutzerfreundlich.

    6. Testing & Go-Live (Implementierung)
    • Teste das Design mit echten Nutzern (Usability-Tests).
    • Setze das Vorhaben um und gehe Online mit deinem Produkt.

    7. Finalisierung & Übergabe
    • Erstelle die finalen Design-Dateien und Übergabe an Entwickler, um die App/Website zu bauen.

    8. Feedback & Nacharbeiten
    • Feedback einholen und Nacharbeiten durchführen.
    • Optimiere das Design basierend auf den Ergebnissen.



    Welche Aspekte sind bei der Erstellung von UX / UI Designs zu beachten?
    1. Benutzerzentrierung
    • Verstehe die Zielgruppe: Wer wird die App/Website nutzen? Welche Bedürfnisse, Wünsche und Probleme haben sie?
    • User Journeys: Plane, wie Nutzer durch die App oder Website navigieren und welche Ziele sie erreichen möchten.

    2. Benutzerfreundlichkeit (Usability)
    • Intuitive Navigation: Stelle sicher, dass Nutzer schnell und einfach finden, was sie suchen.
    Navigations-Elemente sollten logisch und leicht verständlich sein.
    • Einfachheit: Vermeide unnötige Komplexität. Je weniger der Nutzer tun muss, desto besser.
    • Fehlervermeidung: Biete klare Hinweise und Rückmeldungen, wenn Fehler auftreten (z. B. bei falschen Eingaben).

    3. Visuelles Design (UI)
    • Konsistenz: Verwende einheitliche Farben, Schriftarten und Layouts auf allen Seiten, um eine klare Markenidentität zu schaffen.
    • Lesbarkeit: Wähle gut lesbare Schriftarten und sorge für ausreichenden Kontrast zwischen Text und Hintergrund.
    • Hierarchie: Wichtige Elemente wie Call-to-Action-Buttons sollten auffallen, während weniger wichtige Informationen zurücktreten.
    • Responsivität: Das Design muss auf verschiedenen Geräten (Smartphones, Tablets, Desktops) gut aussehen und funktionieren.

    4. Interaktivität und Feedback
    • Visuelle Rückmeldungen: Gib Nutzern klare Rückmeldungen (z. B. durch Animationen oder Farbänderungen),
    wenn sie mit der App interagieren (z. B. Button-Klicks).
    • Interaktive Elemente: Achte darauf, dass Buttons und Links klar erkennbar sind und sich beim Anklicken oder Überfahren optisch verändern.

    5. Zugänglichkeit (Accessibility)
    • Barrierefreiheit: Stelle sicher, dass deine App auch für Menschen mit Einschränkungen zugänglich ist (z. B. Bildschirmleser-Unterstützung, Farbkontraste).
    • Touchfreundlichkeit: Achte darauf, dass Buttons und Links groß genug sind, um sie bequem auf mobilen Geräten zu bedienen.

    6. Ladezeiten & Performance
    • Schnelligkeit: Das Design sollte die Ladezeit minimieren,
    indem unnötige Elemente vermieden werden und optimierte Bilder verwendet werden.

    7. Testing & Iteration
    • Usability-Tests: Teste dein Design mit echten Nutzern und sammle Feedback.
    Achte darauf, wie sie die Navigation, das Design und die Funktionen erleben.
    • Kontinuierliche Verbesserung: Iteriere das Design basierend auf den Testergebnissen und optimiere es,
    um eine bessere Nutzererfahrung zu bieten.



    Wie Dokumentiere ich meine UX/UI Desings?
    Ich dokumentiere meine Designs mit Grafischen Darstellungen (Bildern) und Textbeschreibungen.
    Z.B. als Word-Dokument oder wie meine OneNote Dokumentationen.



    Qualitätsmerkmale!

    Was sind Merkmale für schlechte UX/UI Designs?
    Schlechtes UI/UX-Design führt zu Frustration und einer schlechten Nutzererfahrung.
    • Unklare Navigation: Nutzer finden nicht, was sie suchen.
    • Überladene Oberflächen: Zu viele Elemente, die ablenken.
    • Inkonsequente Gestaltung: Unterschiedliche Designelemente ohne klare Struktur.
    • Schlechte Lesbarkeit: Schlechter Kontrast und kleine Schriftgrößen.
    • Lange Ladezeiten: Verzögerungen, die frustrieren.
    • Fehlendes Feedback: Nutzer wissen nicht, ob ihre Aktionen erfolgreich waren.
    • Nicht mobilfreundlich: Kein gutes Design für mobile Geräte.



    Was sind die Merkmale von gute UI/UX Design?
    • Intuitive Navigation: Nutzer finden schnell, was sie suchen.
    • Klarheit und Einfachheit: Weniger Ablenkung, klare Struktur.
    • Konsistenz: Einheitliches Design und wiederkehrende Elemente.
    • Gute Lesbarkeit: Passender Kontrast, klare Schriftarten.
    • Schnelle Ladezeiten: Optimierte Performance.
    • Visuelles Feedback: Klare Rückmeldungen bei Interaktionen.
    • Mobile Optimierung: Funktioniert auf allen Geräten.
    • Zugänglichkeit: Für alle Nutzer, auch mit Einschränkungen.
    Gutes Design sorgt für eine angenehme, effiziente und benutzerfreundliche Erfahrung.
