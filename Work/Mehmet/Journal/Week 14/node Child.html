1. Was ist ein JS Engine?

Eine JavaScript Engine ist eine Software, die JavaScript-Code liest, interpretiert und ausführt.

Sie übersetzt den Code in Maschinensprache, damit der Computer ihn verstehen kann.

  • V8 ist eine bekannte Engine, die von Google entwickelt wurde und sowohl in Chrome als auch in Node.js verwendet wird.

2. Was ist der Unterschied zwischen JS Engine in einem Webbrowser und Node.js?

Wie bereits erwähnt, ist es dieselbe Engine (V8), aber das Umfeld ist anders.

Bereich Browser (z. B. Chrome) Node.js
Ziel Webseiten anzeigen und manipulieren Serverseitige Logik und Tools ausführen
Zugriff auf DOM ✅ Ja (z. B. document.getElementById) ❌ Nein (kein HTML/DOM verfügbar)
Umgebung Fenster (Browser-Fenster) Betriebssystem (Dateien, Netzwerk)
Modul-System ES Modules (import) CommonJS (require) oder ES Modules
Eingebaute APIs alert(), fetch(), localStorage fs, http, path, os

Die JS Engine ist gleich, aber im Browser läuft JS für Webseiten – in Node.js läuft JS für Server, Tools und Dateien.

3. Was funktioniert NICHT in Node.js?

Alles, was mit DOM zu tun hat, funktioniert nicht, da es kein document gibt, nur .js-Dateien. Daher auch keine:

  • document.querySelector, getElementById etc.
  • window.alert()
  • localStorage, sessionStorage
  • Direkte Manipulationen von HTML oder CSS. Kein document = keine Manipulation.

Stattdessen werden Dateien bei Node.js als JSON-Dateien gespeichert oder, wenn mehrere Benutzer angelegt werden müssen, in Datenbanken verwaltet.

Funktion Browser Node.js Alternative
Dauerhafte Speicherung localStorage (bleibt auch nach Browser-Neustart) Datei speichern (z. B. fs), Datenbank (MongoDB, MySQL)
Temporäre Speicherung (nur für Tab) sessionStorage (bleibt nur für aktuellen Tab offen) express-session (Session-Management), temporärer RAM-Speicher (z. B. Map)
Kleine Daten im Browser Cookies cookie-parser + express-session

4. Wofür steht npm?

npm = Node Package Manager

Es ist das offizielle Tool, mit dem du in Node.js Bibliotheken (Packages) suchst, installierst, verwaltest und aktualisierst.

  • Du kannst damit fertige Codes von Entwicklern in dein Projekt einbinden.
  • Wieso du das machen solltest:

Typische Befehle:

  • Paket installieren: npm install express
  • Alle Pakete aktualisieren: npm update
Vorteil Erklärung
Pakete nutzen Fertige Bibliotheken installieren statt alles selbst programmieren. Beispiel: Express benutzen und statt 15 Zeilen Code nur 5 schreiben.
Abhängigkeiten verwalten Automatische Verwaltung der richtigen Paketversionen
Updates & Sicherheit Pakete leicht aktualisieren, um Bugs und Sicherheitslücken zu schließen
Projekt teilen package.json zeigt anderen Entwicklern, welche Pakete dein Projekt braucht

5. Was bedeutet die Abkürzung CLI?

CLI steht für Command Line Interface (auf Deutsch: Befehlszeilenschnittstelle).

Das ist eine textbasierte Schnittstelle, über die du mit deinem Computer oder Programmen kommunizierst, indem du Befehle in eine Eingabeaufforderung oder ein Terminal eingibst.

Befehl Bedeutung / Zweck
npm init Erstellt eine neue package.json für dein Projekt
npm install paket Installiert ein Paket und fügt es als Dependency hinzu
npm start Startet dein Projekt (definiert in package.json)
node app.js Führt deine Node.js Datei aus
npm update Aktualisiert alle installierten Pakete
npm uninstall paket Entfernt ein installiertes Paket

6. Was ist ein Node.js Project?

Ein Node.js-Projekt ist ein Ordner mit JavaScript-Dateien, einer package.json und ggf. installierten Paketen. Es nutzt Node.js, um JavaScript außerhalb des Browsers auszuführen (z. B. auf dem Server oder lokal als Tool).
