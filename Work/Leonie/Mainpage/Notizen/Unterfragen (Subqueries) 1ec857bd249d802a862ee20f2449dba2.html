Unterabfragen (auch Subqueries genannt) sind ein grundlegendes Konzept in SQL, das dir erlaubt, eine Abfrage innerhalb einer anderen Abfrage auszuführen. Sie sind sehr nützlich, wenn du komplexe Datenabfragen durchführen möchtest, die in einem Schritt nicht möglich sind.

Was ist eine Unterabfrage?

Eine Unterabfrage ist eine SQL-Abfrage, die innerhalb einer anderen SQL-Abfrage eingebettet ist. Die Unterabfrage wird zuerst ausgeführt, und das Ergebnis wird von der äußeren Abfrage verwendet.

Arten von Unterabfragen

Es gibt verschiedene Arten von Unterabfragen, die je nach Anwendungsfall verwendet werden können:

1. Skalare Unterabfrage

  • Gibt einen einzigen Wert zurück (z. B. eine Zahl oder einen Text).
  • Wird oft in der WHEREoder SELECTKlausel verwendet.

Beispiel:
Finde alle Städte, deren Bevölkerung größer ist als die durchschnittliche Bevölkerung aller Städte:

SELECT name
FROM city
WHERE population > (
    SELECT AVG(population)
    FROM city
);
  • Unterabfrage: Berechnet den Durchschnitt der Bevölkerung (AVG(population)).
  • Hauptabfrage: Wählt Städte aus, deren Bevölkerung größer ist als dieser Durchschnitt.

2. Mehrzeilige Unterabfrage

  • Gibt mehrere Werte zurück (z. B. eine Liste von IDs oder Namen).
  • Wird oft mit Operatoren wie IN, ANY, oder ALL verwendet.

Beispiel:
Finde alle Länder, die eine Stadt mit mehr als 1 Million Einwohnern haben:

SELECT name
FROM country
WHERE code IN (
    SELECT country
    FROM city
    WHERE population > 1000000
);
  • Unterabfrage: Gibt die countryCodes aller Städte mit mehr als 1 Million Einwohnern zurück.
  • Hauptabfrage: Wählt Länder aus, deren code in der Liste der Ergebnisse der Unterabfrage enthalten ist.

3. Korrelierte Unterabfrage

  • Eine korrelierte Unterabfrage bezieht sich auf eine Spalte aus der äußeren Abfrage.
  • Sie wird für jede Zeile der äußeren Abfrage separat ausgeführt.

Beispiel:
Finde alle Städte, die die größte Bevölkerung in ihrem Land haben:

SELECT name, country, population
FROM city c1
WHERE population = (
    SELECT MAX(c2.population)
    FROM city c2
    WHERE c2.country = c1.country
);
  • Unterabfrage: Berechnet die maximale Bevölkerung (MAX(c2.population)) für das Land der aktuellen Stadt (c1.country).
  • Hauptabfrage: Wählt Städte aus, deren Bevölkerung gleich der maximalen Bevölkerung ihres Landes ist.

4. Unterabfrage in der FROMKlausel

  • Wird verwendet, um eine temporäre Tabelle zu erstellen, die von der äußeren Abfrage verwendet wird.
  • Diese Art von Unterabfrage wird oft als Inline-View bezeichnet.

Beispiel:
Berechne die durchschnittliche Bevölkerung pro Kontinent und zeige nur Kontinente mit einer durchschnittlichen Bevölkerung über 100 Millionen:

SELECT continent, avg_population
FROM (
    SELECT continent, AVG(population) AS avg_population
    FROM country
    GROUP BY continent
) AS subquery
WHERE avg_population > 100000000;
  • Unterabfrage: Berechnet die durchschnittliche Bevölkerung pro Kontinent.
  • Hauptabfrage: Wählt nur Kontinente aus, deren durchschnittliche Bevölkerung größer als 100 Millionen ist.

Wann solltest du Unterabfragen verwenden?

  • Wenn du Zwischenergebnisse benötigst, die von der Hauptabfrage verwendet werden.
  • Wenn du komplexe Filterbedingungen hast, die nicht direkt in der Hauptabfrage formuliert werden können.
  • Wenn du hierarchische Daten oder Abhängigkeiten zwischen Tabellen analysieren möchtest.

Alternativen zu Unterabfragen

In manchen Fällen können Unterabfragen durch Joins ersetzt werden, was oft effizienter ist. Zum Beispiel:

Mit Unterabfrage:

SELECT name
FROM country
WHERE code IN (
    SELECT country
    FROM city
    WHERE population > 1000000
);

Mit Join:

SELECT DISTINCT c.name
FROM country c
INNER JOIN city ci ON c.code = ci.country
WHERE ci.population > 1000000;

Zusammenfassun

  • Unterabfragen sind Abfragen innerhalb von Abfragen, die Zwischenergebnisse liefern.
  • Sie können in verschiedenen SQL-Klauseln verwendet werden (SELECT, WHERE, FROM, etc.).
  • Es gibt skalare, mehrzeilige, korrelierte und Inline-ViewUnterabfragen.
  • Unterabfragen sind mächtig, aber manchmal können sie durch Joins ersetzt werden, um die Leistung zu verbessern.

Woher weiß ich, welche Unterabfrage zu wählen und diese anzuwenden

Das ist eine sehr gute Frage! Die Wahl der richtigen Art von Unterabfrage hängt davon ab, was du erreichen möchtest und wie die Daten organisiert sind. Lass uns das anhand deines Beispiels – "das Land mit den meisten anerkannten Religionen" – Schritt für Schritt durchgehen.

1. Verstehe die Aufgabe

Du möchtest:

  • Das Land finden, das die meisten anerkannten Religionen hat.
  • Dazu musst du die Anzahl der Religionen pro Land berechnen und dann das Land mit der höchsten Anzahl auswählen.

2. Analysiere die Datenstruktur

Angenommen, du hast eine Tabelle religion, die so aussieht:

FranceChristianity
FranceIslam
FranceJudaism

USAIslam
USAJudaism
USABuddhism
  • Jede Zeile repräsentiert eine Religion, die in einem Land anerkannt ist.
  • Du musst die Anzahl der Religionen pro Land zählen und dann das Land mit der höchsten Anzahl finden.

3. Wähle die richtige Art von Unterabfrage

Option 1: Skalare Unterabfrage

Wenn du nur einen einzigen Wert (z. B. das Land mit der höchsten Anzahl von Religionen) benötigst, kannst du eine skalare Unterabfrage verwenden.

Abfrage:

SELECT country, COUNT(religion_name) AS religion_count
FROM religion
GROUP BY country
HAVING COUNT(religion_name) = (
    SELECT MAX(religion_count)
    FROM (
        SELECT country, COUNT(religion_name) AS religion_count
        FROM religion
        GROUP BY country
    ) AS subquery
);

Erklärung:

  1. Innere Unterabfrage (subquery):
    • Zählt die Anzahl der Religionen pro Land (COUNT(religion_name)) und gruppiert die Ergebnisse nach Ländern.
    • Beispielergebnis:
      country   | religion_count
--------------------------
Germany   | 2
France    | 3
USA       | 4
  1. Skalare Unterabfrage (SELECT MAX(religion_count)):
    • Findet die maximale Anzahl von Religionen (MAX(religion_count)) aus der inneren Unterabfrage.
    • Beispielergebnis: 4.
  1. Hauptabfrage:
    • Wählt das Land aus, dessen Anzahl von Religionen gleich der maximalen Anzahl ist (HAVING COUNT(religion_name) = 4).
    • Beispielergebnis:
      country   | religion_count
--------------------------
USA       | 4

Option 2: Mehrzeilige Unterabfrage

Wenn du mehrere Länder mit derselben maximalen Anzahl von Religionen finden möchtest (z. B. bei einem Gleichstand), kannst du eine mehrzeilige Unterabfrage verwenden.

Abfrage:

SELECT country, COUNT(religion_name) AS religion_count
FROM religion
GROUP BY country
HAVING COUNT(religion_name) IN (
    SELECT MAX(religion_count)
    FROM (
        SELECT country, COUNT(religion_name) AS religion_count
        FROM religion
        GROUP BY country
    ) AS subquery
);

Unterschied:

  • Statt = wird IN verwendet, um mehrere Länder mit derselben maximalen Anzahl von Religionen zu berücksichtigen.

Option 3: Unterabfrage in der FROMKlausel

Wenn du die Daten zuerst berechnen und dann filtern möchtest, kannst du eine Unterabfrage in der FROM-Klausel verwenden.

Abfrage:

SELECT country, religion_count
FROM (
    SELECT country, COUNT(religion_name) AS religion_count
    FROM religion
    GROUP BY country
) AS subquery
WHERE religion_count = (
    SELECT MAX(religion_count)
    FROM (
        SELECT country, COUNT(religion_name) AS religion_count
        FROM religion
        GROUP BY country
    ) AS subquery2
);

Erklärung:

  1. Die erste Unterabfrage (subquery) berechnet die Anzahl der Religionen pro Land.
  1. Die zweite Unterabfrage (subquery2) berechnet die maximale Anzahl von Religionen.
  1. Die äußere Abfrage filtert das Land mit der maximalen Anzahl von Religionen.

4. Wie wählst du die richtige Methode?

Fragen, die du dir stellen solltest:

  1. Wie viele Ergebnisse benötigst du?
    • Nur ein Land: Verwende eine skalare Unterabfrage (=).
    • Mehrere Länder bei Gleichstand: Verwende eine mehrzeilige Unterabfrage (IN).
  1. Wie komplex sind die Daten?
    • Wenn die Daten einfach sind, kannst du eine direkte Unterabfrage verwenden.
    • Wenn die Daten komplex sind (z. B. viele Berechnungen), ist eine Unterabfrage in der FROMKlausel oft übersichtlicher.
  1. Wie wichtig ist die Lesbarkeit?
    • Unterabfragen in der FROMKlausel sind oft leichter zu lesen und zu debuggen.

5. Beispielergebnis

Angenommen, die Tabelle religion enthält die oben genannten Daten. Die Abfrage würde folgendes Ergebnis liefern:

country   | religion_count
--------------------------
USA       | 4

Zusammenfassung

  • Skalare Unterabfragen: Gut für einfache Fälle, wenn du nur einen Wert benötigst.
  • Mehrzeilige Unterabfragen: Gut, wenn du mehrere Ergebnisse (z. B. bei Gleichstand) benötigst.
  • Unterabfragen in der FROMKlausel: Gut für komplexe Berechnungen oder wenn du die Daten zuerst berechnen und dann filtern möchtest.

Ist es besser bei komplexen Abfragen von Innen nach Außen zu bauen?

Ja, genau! Bei komplexen SQL-Abfragen, insbesondere wenn Unterabfragen verwendet werden, ist es oft sinnvoll, von innen nach außen zu arbeiten. Das bedeutet, dass du zuerst die innerste Abfrage baust, testest und sicherstellst, dass sie korrekt funktioniert, bevor du sie in eine äußere Abfrage einfügst.

Warum von innen nach außen arbeiten?

  1. Schrittweises Debugging:
    • Du kannst jede Unterabfrage einzeln testen und sicherstellen, dass sie die erwarteten Ergebnisse liefert.
    • Das macht es einfacher, Fehler zu finden und zu beheben.
  1. Bessere Lesbarkeit:
    • Wenn du die inneren Abfragen zuerst baust, kannst du die Logik klarer verstehen und die äußeren Abfragen darauf aufbauen.
  1. Komplexität reduzieren:
    • Indem du die Abfrage schrittweise aufbaust, vermeidest du, dass du dich in der Verschachtelung verlierst.

Wie baust du eine Abfrage von innen nach außen?

Beispiel: Finde das Land mit den meisten Religionen

  1. Innerste Abfrage: Zähle die Religionen pro Land
    • Beginne mit der Abfrage, die die Anzahl der Religionen pro Land berechnet:
    SELECT c2.Name, COUNT(r2.Name) AS religion_count
FROM religion r2
INNER JOIN country c2 ON r2.Country = c2.Code
GROUP BY c2.Name;
    • Ergebnis: Eine Liste aller Länder mit der Anzahl ihrer Religionen:
      Name           | religion_count
-------------------------------
Germany        | 2
France         | 3
United States  | 4
  1. Nächste Ebene: Finde die maximale Anzahl von Religionen
    • Verwende die Ergebnisse der ersten Abfrage, um die maximale Anzahl von Religionen zu berechnen:
    SELECT MAX(religion_count)
FROM (
    SELECT c2.Name, COUNT(r2.Name) AS religion_count
    FROM religion r2
    INNER JOIN country c2 ON r2.Country = c2.Code
    GROUP BY c2.Name
) AS countRel;
    • Ergebnis: Die maximale Anzahl von Religionen:
      MAX(religion_count)
-------------------
4
  1. Äußere Abfrage: Filtere Länder mit der maximalen Anzahl
    • Verwende die maximale Anzahl aus der zweiten Abfrage, um die Länder zu filtern:
    SELECT c.Name, COUNT(r.Name) AS religion_count
FROM religion r
INNER JOIN country c ON r.Country = c.Code
GROUP BY c.Name
HAVING COUNT(r.Name) = (
    SELECT MAX(religion_count)
    FROM (
        SELECT c2.Name, COUNT(r2.Name) AS religion_count
        FROM religion r2
        INNER JOIN country c2 ON r2.Country = c2.Code
        GROUP BY c2.Name
    ) AS countRel
);
    • Ergebnis: Das Land mit der höchsten Anzahl von Religionen:
      Name           | religion_count
-------------------------------
United States  | 4

Vorteile dieser Methode:

  1. Einfaches Testen:
    • Du kannst jede Abfrage einzeln ausführen und sicherstellen, dass sie korrekt funktioniert, bevor du sie in die nächste Ebene einfügst.
  1. Klarheit:
    • Du verstehst genau, was jede Abfrage macht, und kannst die Logik Schritt für Schritt aufbauen.
  1. Fehler vermeiden:
    • Wenn ein Teil der Abfrage nicht funktioniert, kannst du den Fehler isolieren und beheben, ohne die gesamte Abfrage zu überarbeiten.

Zusammenfassung:

Ja, bei komplexen Abfragen mit Unterabfragen ist es eine sehr gute Praxis, von innen nach außen zu arbeiten. Es hilft dir, die Logik zu verstehen, Fehler zu vermeiden und die Abfrage schrittweise aufzubauen.

Unterabfragen Beispiel:

SELECT DISTINCT e.Continent, m.Name, m.Height
FROM encompasses e 
INNER JOIN geo_mountain gm ON e.Country = gm.Country
INNER JOIN mountain m ON gm.Mountain = m.Name
WHERE m.Height = (
    SELECT MAX(m2.Height)
    FROM mountain m2
    INNER JOIN geo_mountain gm2 ON m2.Name = gm2.Mountain
    INNER JOIN encompasses e2 ON gm2.Country = e2.Country
    WHERE e2.Continent = e.Continent
) 
ORDER BY m.Height DESC

Was macht dieser Code (Unterabfrage)?

Dieser Code ist ein Filter in der Hauptabfrage. Er sorgt dafür, dass nur die Berge ausgewählt werden, die die höchste Höhe (MAX(m2.Height)) in ihrem jeweiligen Kontinent haben.

Schritt-für-Schritt-Erklärung:

  1. m.Height = (...):
    • Hier wird geprüft, ob die Höhe eines Berges (m.Height) gleich der höchsten Höhe ist, die in der Unterabfrage berechnet wird.
    • Das bedeutet: Wir wollen nur den Berg, der der höchste in seinem Kontinent ist.
  1. Unterabfrage (SELECT MAX(m2.Height) ...):
    • Diese Unterabfrage berechnet die maximale Höhe (MAX(m2.Height)) aller Berge in einem bestimmten Kontinent.
  1. FROM mountain m2:
    • Die Unterabfrage arbeitet mit einer zweiten Version der Tabelle mountain, die wir hier m2 nennen.
    • m2 enthält alle Berge mit ihren Namen und Höhen.
  1. INNER JOIN geo_mountain gm2 ON m2.Name = gm2.Mountain:
    • Hier wird die Tabelle mountain (m2) mit der Tabelle geo_mountain (gm2) verknüpft.
    • Diese Verknüpfung stellt sicher, dass wir wissen, in welchem Land jeder Berg liegt.
  1. INNER JOIN encompasses e2 ON gm2.Country = e2.Country:
    • Jetzt wird die Tabelle geo_mountain (gm2) mit der Tabelle encompasses (e2) verknüpft.
    • Dadurch können wir herausfinden, zu welchem Kontinent das Land gehört, in dem der Berg liegt.
  1. WHERE e2.Continent = e.Continent:
    • Hier wird gefiltert: Wir wollen nur die Berge betrachten, die im gleichen Kontinent liegen wie der aktuelle Kontinent in der Hauptabfrage (e.Continent).
    • Das bedeutet: Für jeden Kontinent wird separat die maximale Höhe berechnet.
  1. Ergebnis der Unterabfrage:
    • Die Unterabfrage gibt die maximale Höhe (MAX(m2.Height)) für den aktuellen Kontinent zurück.
  1. Vergleich mit m.Height:
    • In der Hauptabfrage wird geprüft, ob die Höhe des aktuellen Berges (m.Height) gleich der maximalen Höhe ist, die die Unterabfrage berechnet hat.
    • Wenn das der Fall ist, wird dieser Berg in das Ergebnis aufgenommen.

Korrelierte Unterfrage Beispiel:

Eine korrelierte Unterabfrage ist eine Unterabfrage, die für jede Zeile der äußeren Abfrage separat ausgeführt wird. Sie bezieht sich auf Werte aus der äußeren Abfrage, wodurch eine Abhängigkeit zwischen der äußeren und der inneren Abfrage entsteht.

Wie funktioniert eine korrelierte Unterabfrage?

  1. Die äußere Abfrage verarbeitet eine Zeile nach der anderen.
  1. Für jede Zeile der äußeren Abfrage wird die Unterabfrage ausgeführt.
  1. Die Unterabfrage verwendet Werte aus der aktuellen Zeile der äußeren Abfrage, um ihre Ergebnisse zu berechnen.
  1. Das Ergebnis der Unterabfrage wird verwendet, um die aktuelle Zeile der äußeren Abfrage zu bewerten (z. B. in einer WHEREKlausel).

Beispiel:

Finde alle Städte, die die größte Bevölkerung in ihrem Land haben.

SQL-Abfrage:

SELECT c1.name, c1.country, c1.population
FROM city c1
WHERE c1.population = (
    SELECT MAX(c2.population)
    FROM city c2
    WHERE c2.country = c1.country
);

Wie funktioniert diese Abfrage?

  1. Äußere Abfrage:
    • Die äußere Abfrage iteriert über jede Zeile in der Tabelle city (jede Stadt).
    • Für jede Stadt wird die Unterabfrage ausgeführt.
  1. Unterabfrage:
    • Die Unterabfrage berechnet die maximale Bevölkerung (MAX(c2.population)) für das Land (c2.country), das zur aktuellen Stadt (c1.country) gehört.
    • Die Unterabfrage verwendet den Wert c1.country aus der äußeren Abfrage.
  1. Vergleich:
    • Die äußere Abfrage prüft, ob die Bevölkerung der aktuellen Stadt (c1.population) gleich der maximalen Bevölkerung des Landes ist, die von der Unterabfrage zurückgegeben wurde.

Mehrzeilige Unterabfrage Schritt für Schritt

Was macht diese Abfrage?

Die Abfrage sucht das Land (oder die Länder), das die höchste Anzahl an anerkannten Religionen hat. Sie verwendet dazu eine Kombination aus Gruppierung, Aggregatfunktionen und Unterabfragen.

Die Abfrage im Überblick:

SELECT c.Name, COUNT(r.Name) AS religion_count
FROM religion r
INNER JOIN country c ON r.Country = c.Code
GROUP BY c.Name
HAVING COUNT(r.Name) IN (
    SELECT MAX(religion_count)
    FROM (
        SELECT c2.Name, COUNT(r2.Name) AS religion_count
        FROM religion r2
        INNER JOIN country c2 ON r2.Country = c2.Code
        GROUP BY c2.Name
    ) AS countRel
)

Schritt-für-Schritt-Erklärung

1. Hauptabfrage:

SELECT c.Name, COUNT(r.Name) AS religion_count
FROM religion r
INNER JOIN country c ON r.Country = c.Code
GROUP BY c.Name

  • SELECT c.Name, COUNT(r.Name) AS religion_count:
    • Wählt den Namen des Landes (c.Name) und die Anzahl der Religionen (COUNT(r.Name)) aus.
    • COUNT(r.Name) zählt, wie viele Religionen in der Tabelle religion für jedes Land (c.Name) existieren.
  • INNER JOIN country c ON r.Country = c.Code:
    • Verknüpft die Tabelle religion mit der Tabelle country, sodass jede Religion einem Land zugeordnet wird.
    • r.Country ist der Ländercode in der Tabelle religion, und c.Code ist der Ländercode in der Tabelle country.
  • GROUP BY c.Name:
    • Gruppiert die Daten nach Ländern (c.Name), sodass die Anzahl der Religionen (COUNT(r.Name)) für jedes Land berechnet werden kann.
  • HAVING COUNT(r.Name) IN (...):
    • Filtert die Länder, deren Anzahl an Religionen (COUNT(r.Name)) in der Liste enthalten ist, die von der Unterabfrage zurückgegeben wird.

2. Erste Unterabfrage (im HAVING):

SELECT MAX(religion_count)
FROM (
    SELECT c2.Name, COUNT(r2.Name) AS religion_count
    FROM religion r2
    INNER JOIN country c2 ON r2.Country = c2.Code
    GROUP BY c2.Name
) AS countRel
  • Was macht diese Unterabfrage?
    • Sie berechnet die maximale Anzahl von Religionen (MAX(religion_count)) über alle Länder.

3. Rückkehr zur Hauptabfrage:

SELECT c.Name, COUNT(r.Name) AS religion_count
FROM religion r
INNER JOIN country c ON r.Country = c.Code
GROUP BY c.Name
HAVING COUNT(r.Name) IN (...)
  • HAVING COUNT(r.Name) IN (...):
    • Filtert die Länder, deren Anzahl an Religionen (COUNT(r.Name)) gleich der maximalen Anzahl ist, die von der Unterabfrage zurückgegeben wurde.
    • Beispiel: Nur Länder mit religion_count = 4 werden ausgewählt.

Zusammenfassung:

  1. Die Hauptabfrage zählt die Anzahl der Religionen pro Land.
  1. Die innere Unterabfrage berechnet die Anzahl der Religionen für jedes Land.
  1. Die äußere Unterabfrage findet die maximale Anzahl von Religionen.
  1. Die Hauptabfrage filtert Länder, deren Anzahl an Religionen gleich der maximalen Anzahl ist.

Die Erklärung (leicht angepasst):

  1. Das erste SELECT:
    • "Ich möchte alle Länder mit der zusammengezählten Anzahl an Religionen haben."
    • Das passiert durch COUNT(r.Name) in der Hauptabfrage, die die Religionen pro Land zählt.
  1. Das HAVING:
    • "Ich möchte nur die Länder, deren Anzahl an Religionen gleich der maximalen Anzahl an Religionen ist."
    • Die Bedingung im HAVINGTeil (HAVING COUNT(r.Name) IN (...)) sorgt dafür, dass nur Länder mit der höchsten Anzahl an Religionen ausgewählt werden.
  1. Die Tabelle countRel:
    • Diese Tabelle (aus der inneren Unterabfrage) enthält alle Länder mit der zusammengezählten Anzahl an Religionen.
    • Beispiel: Germany = 2, France = 3, United States = 4.
  1. Das zweite SELECT:
    • "Finde die maximale Anzahl an Religionen aus der Tabelle countRel."
    • Das passiert durch MAX(religion_count) in der äußeren Unterabfrage.

Zusammenfassung:

Deine Erklärung ist fast korrekt, aber der Fokus liegt darauf, dass die innere Unterabfrage (countRel) zuerst die Anzahl der Religionen pro Land berechnet und die äußere Unterabfrage dann die maximale Anzahl dieser Religionen findet. Das HAVING filtert schließlich die Länder, die diese maximale Anzahl haben.
