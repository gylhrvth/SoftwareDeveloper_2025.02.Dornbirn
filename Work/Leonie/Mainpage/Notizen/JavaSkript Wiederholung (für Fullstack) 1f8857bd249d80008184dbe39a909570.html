Hier ist eine Übersicht der wichtigsten Konzepte, die du beim Wiederholen von JavaScript im Kontext von Fullstack-Entwicklung beachten solltest:

1. Datentypen

JavaScript hat primitive und komplexe Datentypen. Wichtig ist, die Unterschiede und ihre Verwendung zu verstehen.

Primitive Datentypen:

  • string, number, boolean, null, undefined, bigint, symbol
  • Beispiel:
    let name = "John"; // string
let age = 25; // number
let isActive = true; // boolean
let nothing = null; // null
let notDefined; // undefined
let bigNumber = 12345678901234567890n; // bigint

Komplexe Datentypen:

  • Object (inkl. Arrays, Funktionen)
  • Beispiel:
    let person = { name: "John", age: 25 }; // Object
let numbers = [1, 2, 3]; // Array
function greet() { console.log("Hello!"); } // Funktion

Wichtige Punkte:

  • typeof: Prüft den Datentyp.
    console.log(typeof "Hello"); // string
console.log(typeof 42); // number
console.log(typeof null); // object (historischer Bug)
console.log(typeof undefined); // undefined
  • Type Coercion: Automatische Typkonvertierung.
    console.log(1 + "2"); // "12" (Zahl wird zu String)
console.log("5" - 2); // 3 (String wird zu Zahl)

2. Arrays

Arrays sind geordnete Listen von Werten. Sie sind in Fullstack-Anwendungen häufig für Datenstrukturen wichtig.

Wichtige Methoden:

  • Hinzufügen/Entfernen:
    let arr = [1, 2, 3];
arr.push(4); // [1, 2, 3, 4]
arr.pop(); // [1, 2, 3]
arr.unshift(0); // [0, 1, 2, 3]
arr.shift(); // [1, 2, 3]
  • Iterieren:
    arr.forEach((item) => console.log(item)); // Gibt jedes Element aus
let doubled = arr.map((item) => item * 2); // [2, 4, 6]
let filtered = arr.filter((item) => item > 1); // [2, 3]
  • Suchen:
    console.log(arr.includes(2)); // true
console.log(arr.indexOf(3)); // 2
  • Spread-Operator:
    let newArr = [...arr, 4, 5]; // [1, 2, 3, 4, 5]

3. Objects

Objekte sind Schlüssel-Wert-Paare und essenziell für die Arbeit mit APIs und Datenbanken.

Erstellen und Zugriff:

  • Literal-Syntax:
    let user = { name: "Alice", age: 30 };
console.log(user.name); // "Alice"
console.log(user["age"]); // 30
  • Dynamische Schlüssel:
    let key = "email";
user[key] = "alice@example.com";
console.log(user); // { name: "Alice", age: 30, email: "alice@example.com" }

Wichtige Methoden:

  • Objekt-Iterationen:
    for (let key in user) {
  console.log(key, user[key]); // name Alice, age 30, email alice@example.com
}
  • Object-Methoden:
    let keys = Object.keys(user); // ["name", "age", "email"]
let values = Object.values(user); // ["Alice", 30, "alice@example.com"]
let entries = Object.entries(user); // [["name", "Alice"], ["age", 30], ["email", "alice@example.com"]]
  • Spread-Operator:
    let updatedUser = { ...user, age: 31 }; // { name: "Alice", age: 31, email: "alice@example.com" }

4. Funktionen

Funktionen sind zentral in JavaScript. Sie werden für Logik, API-Aufrufe und Datenverarbeitung verwendet.

Arten von Funktionen:

  • Normale Funktionen:
    function add(a, b) {
  return a + b;
}
console.log(add(2, 3)); // 5
  • Arrow-Funktionen:
    const multiply = (a, b) => a * b;
console.log(multiply(2, 3)); // 6
  • Anonyme Funktionen:
    setTimeout(function () {
  console.log("Hello after 1 second");
}, 1000);

Wichtige Konzepte:

  • Default-Parameter:
    function greet(name = "Guest") {
  console.log("Hello, " + name);
}
greet(); // "Hello, Guest"
  • Rest-Parameter:
    function sum(...numbers) {
  return numbers.reduce((acc, num) => acc + num, 0);
}
console.log(sum(1, 2, 3, 4)); // 10
  • Callback-Funktionen:
    function processUserInput(callback) {
  let name = "Alice";
  callback(name);
}
processUserInput((name) => console.log("Hello, " + name)); // "Hello, Alice"
  • IIFE (Immediately Invoked Function Expression):
    (function () {
  console.log("IIFE läuft sofort!");
})();

Zusammenfassung

Beim Wiederholen solltest du dich auf folgende Punkte konzentrieren:

  1. Datentypen: Primitive vs. komplexe Typen, typeof, Type Coercion.
  1. Arrays: Manipulation, Iteration, Spread-Operator.
  1. Objects: Zugriff, Iteration, Spread-Operator, dynamische Schlüssel.
  1. Funktionen: Arrow-Funktionen, Callback-Funktionen, Default-Parameter, Rest-Parameter.

Diese Grundlagen sind essenziell für die Arbeit mit APIs, Datenbanken und Frontend-Backend-Kommunikation in der Fullstack-Entwicklung.

Die map-Funktion in JavaScript ist eine Methode von Arrays, die ein neues Array erstellt, indem sie eine Callback-Funktion auf jedes Element des ursprünglichen Arrays anwendet. Sie ist besonders nützlich, wenn du ein Array transformieren möchtest, ohne das Original zu verändern.

Syntax

array.map(callback(currentValue, index, array), thisArg)
  • callback: Die Funktion, die auf jedes Element angewendet wird.
    • currentValue: Das aktuelle Element des Arrays.
    • index (optional): Der Index des aktuellen Elements.
    • array (optional): Das ursprüngliche Array.
  • thisArg (optional): Ein Wert, der als this in der Callback-Funktion verwendet wird.

Beispiele

1. Transformation eines Arrays

const numbers = [1, 2, 3, 4];
const squared = numbers.map((num) => num * num);
console.log(squared); // [1, 4, 9, 16]

2. Objekte aus einem Array erstellen

const names = ["Alice", "Bob", "Charlie"];
const users = names.map((name) => ({ name: name, active: true }));
console.log(users);
// [{ name: "Alice", active: true }, { name: "Bob", active: true }, { name: "Charlie", active: true }]

3. Zugriff auf Index und Array

const numbers = [10, 20, 30];
const withIndex = numbers.map((num, index) => `Index ${index}: ${num}`);
console.log(withIndex);
// ["Index 0: 10", "Index 1: 20", "Index 2: 30"]

Wichtige Punkte

  1. map verändert das Original-Array nicht:
    const arr = [1, 2, 3];
const doubled = arr.map((num) => num * 2);
console.log(arr); // [1, 2, 3] (unverändert)
console.log(doubled); // [2, 4, 6]
  1. map gibt immer ein neues Array zurück, auch wenn keine Änderungen vorgenommen werden:
    const arr = [1, 2, 3];
const same = arr.map((num) => num);
console.log(same); // [1, 2, 3]
  1. map ist nicht geeignet, wenn du nur durch ein Array iterieren möchtest. Dafür solltest du forEach verwenden.

Vergleich mit forEach

  • map: Gibt ein neues Array zurück.
  • forEach: Führt nur eine Aktion für jedes Element aus, gibt aber nichts zurück.

Beispiel:

const numbers = [1, 2, 3];

// map
const doubled = numbers.map((num) => num * 2);
console.log(doubled); // [2, 4, 6]

// forEach
numbers.forEach((num) => console.log(num * 2)); // Gibt 2, 4, 6 aus, aber kein neues Array

Zusammenfassung

Die map-Funktion ist ideal, wenn du ein Array transformieren möchtest, z. B. durch Berechnungen, Formatierungen oder das Erstellen neuer Objekte. Sie ist ein unverzichtbares Werkzeug in der modernen JavaScript-Programmierung, insbesondere in der Fullstack-Entwicklung, z. B. beim Verarbeiten von API-Daten.

Wiederholung V03:

Die Funktion getMaxValueObjects ist darauf ausgelegt, aus einem Array von Objekten diejenigen Objekte zurückzugeben, die den höchsten Wert für eine bestimmte Eigenschaft (Key) haben. Hier ist eine detaillierte Erklärung:

Funktionsdefinition

function getMaxValueObjects(arr, key) {
    const maxValue = Math.max(...arr.map(obj => obj[key]));
    return arr.filter(obj => obj[key] === maxValue);
}

Parameter

  1. arr: Ein Array von Objekten, in dem nach dem höchsten Wert gesucht wird.
  1. key: Der Schlüssel (Eigenschaft), dessen Wert in jedem Objekt überprüft wird.

Rückgabewert

Die Funktion gibt ein Array von Objekten zurück, die den höchsten Wert für die angegebene Eigenschaft (key) haben.

Schritt-für-Schritt-Erklärung

1. Höchsten Wert finden

const maxValue = Math.max(...arr.map(obj => obj[key]));
  • arr.map(obj => obj[key]):
    • Erstellt ein neues Array, das nur die Werte der angegebenen Eigenschaft (key) aus jedem Objekt enthält.
    • Beispiel: Wenn arr = [{ age: 30 }, { age: 25 }, { age: 30 }] und key = "age", dann ergibt arr.map(obj => obj[key]) das Array [30, 25, 30].
  • Math.max(...arr.map(...)):
    • Verwendet den Spread-Operator (...), um das Array in einzelne Argumente für Math.max umzuwandeln.
    • Math.max gibt den höchsten Wert aus diesen Argumenten zurück.
    • Beispiel: Math.max(...[30, 25, 30]) ergibt 30.

2. Objekte mit dem höchsten Wert filtern

return arr.filter(obj => obj[key] === maxValue);
  • arr.filter(...):
    • Iteriert über jedes Objekt im Array und überprüft, ob der Wert der angegebenen Eigenschaft (key) gleich dem höchsten Wert (maxValue) ist.
    • Nur die Objekte, die diese Bedingung erfüllen, werden in das Ergebnis-Array aufgenommen.
    • Beispiel: Wenn maxValue = 30, dann werden nur die Objekte { age: 30 } in das Ergebnis aufgenommen.

Beispiel

Eingabe:

const data = [
    { name: "John", age: 30 },
    { name: "Jane", age: 25 },
    { name: "Jim", age: 30 }
];
console.log(getMaxValueObjects(data, "age"));

Ablauf:

  1. arr.map(obj => obj["age"]):
    • Erstellt das Array [30, 25, 30].
  1. Math.max(...[30, 25, 30]):
    • Gibt 30 zurück.
  1. arr.filter(obj => obj["age"] === 30):
    • Filtert die Objekte { name: "John", age: 30 } und { name: "Jim", age: 30 }.

Ausgabe:

[
    { name: "John", age: 30 },
    { name: "Jim", age: 30 }
]

Zusammenfassung

Die Funktion getMaxValueObjects:

  1. Findet den höchsten Wert für eine bestimmte Eigenschaft (key) in einem Array von Objekten.
  1. Gibt alle Objekte zurück, die diesen höchsten Wert haben.

Sie ist besonders nützlich, wenn du mehrere Objekte mit demselben höchsten Wert finden möchtest.

Ist obj der Name, worüber er über alle Objekte iteriert wie bei foreach?

Ja, genau! Im arr.map ist obj ein einzelnes Objekt aus dem Array arr, über das die Methode map iteriert. Es funktioniert ähnlich wie bei einer forEach-Schleife, aber map erstellt ein neues Array, das die Ergebnisse der Callback-Funktion enthält.

Was passiert in arr.map(obj => obj[key])?

  1. arr: Das ist das Array, das du an die Funktion übergibst. In deinem Beispiel:
    [
    { name: "John", age: 30 },
    { name: "Jane", age: 25 },
    { name: "Jim", age: 30 }
]
  1. map: Die Methode iteriert über jedes Element (Objekt) im Array. Jedes Element wird in der Callback-Funktion als obj bezeichnet.
  1. obj[key]: Hier greifst du auf den Wert der Eigenschaft (key) des aktuellen Objekts zu. Wenn der key z. B. "age" ist, dann wird obj["age"] für jedes Objekt im Array aufgerufen.
  1. Ergebnis: map erstellt ein neues Array, das nur die Werte der angegebenen Eigenschaft (key) enthält.

Beispiel: Schritt-für-Schritt-Erklärung

Eingabe:

const arr = [
    { name: "John", age: 30 },
    { name: "Jane", age: 25 },
    { name: "Jim", age: 30 }
];
const key = "age";

Code:

const values = arr.map(obj => obj[key]);
console.log(values); // [30, 25, 30]

Ablauf:

  1. Erste Iteration:
    • obj = { name: "John", age: 30 }
    • obj[key] = obj["age"] = 30
    • Das Ergebnis wird dem neuen Array hinzugefügt: [30].
  1. Zweite Iteration:
    • obj = { name: "Jane", age: 25 }
    • obj[key] = obj["age"] = 25
    • Das Ergebnis wird dem neuen Array hinzugefügt: [30, 25].
  1. Dritte Iteration:
    • obj = { name: "Jim", age: 30 }
    • obj[key] = obj["age"] = 30
    • Das Ergebnis wird dem neuen Array hinzugefügt: [30, 25, 30].

Vergleich mit forEach

map und forEach sind ähnlich, aber sie haben unterschiedliche Zwecke:

  • map: Erstellt ein neues Array mit den Ergebnissen der Callback-Funktion.
  • forEach: Führt eine Aktion für jedes Element aus, gibt aber nichts zurück.

Beispiel mit forEach:

const values = [];
arr.forEach(obj => values.push(obj[key]));
console.log(values); // [30, 25, 30]

Beispiel mit map:

const values = arr.map(obj => obj[key]);
console.log(values); // [30, 25, 30]

Zusammenfassung

  • obj ist ein einzelnes Objekt aus dem Array arr, über das map iteriert.
  • Es funktioniert ähnlich wie bei forEach, aber map erstellt ein neues Array mit den Ergebnissen der Callback-Funktion.
  • In deinem Beispiel wird obj[key] verwendet, um den Wert der Eigenschaft (key) aus jedem Objekt zu extrahieren.
