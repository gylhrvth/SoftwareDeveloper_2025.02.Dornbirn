Wichtige Konzepte zu Objekten in JavaScript

Ein Objekt in JavaScript ist eine Sammlung von Schlüssel-Wert-Paaren. Es wird verwendet, um strukturierte Daten zu speichern und zu organisieren. Objekte können auch andere Objekte, Arrays und Funktionen enthalten, was sie sehr flexibel macht.

Grundlagen von Objekten

  1. Erstellen eines Objekts:
    • Objekte werden mit geschweiften Klammern {} definiert.
    • Schlüssel (Properties) sind Strings oder Symbole, und Werte können beliebige Datentypen sein.
    let person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    isMarried: false,
};
  1. Zugriff auf Eigenschaften:
    • Dot-Notation: person.firstName
    • Bracket-Notation: person["firstName"] (nützlich, wenn der Schlüssel dynamisch ist).
    console.log(person.firstName); // "John"
console.log(person["age"]); // 30
  1. Hinzufügen/Ändern von Eigenschaften:
    person.job = "Developer"; // Hinzufügen
person.age = 31; // Ändern
  1. Löschen von Eigenschaften:
    delete person.isMarried;

Objekte und Arrays

  1. Objekte in Arrays:
    • Arrays können Objekte enthalten, was nützlich ist, um Listen von Daten zu speichern.
    let cars = [
    { brand: "Opel", model: "Astra", year: 2017 },
    { brand: "BMW", model: "X5", year: 2020 },
];
console.log(cars[0].brand); // "Opel"
  1. Arrays in Objekten:
    • Objekte können Arrays als Werte enthalten, um mehrere Werte für eine Eigenschaft zu speichern.
    let car = {
    brand: "Opel",
    services: ["oil change", "brake pads replacement"],
};
console.log(car.services[1]); // "brake pads replacement"
  1. Iterieren über Arrays in Objekten:
    • Mit Schleifen wie for oder forEach kannst du über Arrays iterieren.
    car.services.forEach(service => console.log(service));

Wichtige Punkte zu beachten

  1. Referenztypen:
    • Objekte und Arrays sind Referenztypen. Änderungen an einer Referenz wirken sich auf das Original aus.
    let obj1 = { name: "John" };
let obj2 = obj1;
obj2.name = "Jane";
console.log(obj1.name); // "Jane"
  1. Deep Copy vs. Shallow Copy:
    • Eine flache Kopie (shallow copy) kopiert nur die oberste Ebene eines Objekts.
    • Eine tiefe Kopie (deep copy) kopiert auch verschachtelte Objekte.
    let shallowCopy = { ...obj1 }; // Shallow Copy
let deepCopy = JSON.parse(JSON.stringify(obj1)); // Deep Copy
  1. Iterieren über Objekte:
    • Verwende for...in, um über die Schlüssel eines Objekts zu iterieren.
    for (let key in person) {
    console.log(key, person[key]);
}
  1. thisSchlüsselwort:
    • In Objekten verweist this auf das aktuelle Objekt.
    let car = {
    brand: "Opel",
    getBrand: function () {
        return this.brand;
    },
};
console.log(car.getBrand()); // "Opel"

Beispiele

Objekte mit verschachtelten Arrays und Objekten

let myCar = {
    brand: "VW",
    model: "Golf",
    owner: {
        name: "Leonie",
        address: {
            city: "Dornbirn",
            country: "Austria",
        },
    },
    services: [
        { date: "2018-05-15", type: "oil change", cost: 150 },
        { date: "2019-05-15", type: "brake pads replacement", cost: 300 },
    ],
};

// Zugriff auf verschachtelte Eigenschaften
console.log(myCar.owner.address.city); // "Dornbirn"

// Iteration über das Service-Array
myCar.services.forEach(service => {
    console.log(`${service.type} am ${service.date} für ${service.cost}€`);
});

Funktion zur Ausgabe eines Objekts

function printObject(obj, prefix = "") {
    for (let key in obj) {
        if (typeof obj[key] === "object" && obj[key] !== null) {
            console.log(`${prefix}${key}:`);
            printObject(obj[key], prefix + "  ");
        } else {
            console.log(`${prefix}${key}: ${obj[key]}`);
        }
    }
}

printObject(myCar);

Zusammenfassung

  • Objekte sind flexibel und können beliebige Datenstrukturen enthalten.
  • Arrays und Objekte können ineinander verschachtelt werden, um komplexe Datenstrukturen zu modellieren.
  • Referenzen und Kopien sind wichtig, um ungewollte Änderungen zu vermeiden.
  • Verwende Schleifen (for...in, forEach) und Rekursion, um über verschachtelte Strukturen zu iterieren.

Beispiel:

// Defining the object
const person = {
    firstName: "Leonie",
    lastName: "Schmidt",
    age: 25,
    greet: function() {
        console.log("Hello, my name is " + this.firstName + " " + this.lastName);
    }
};

// Accessing properties
console.log(person.firstName); // Output: Leonie
console.log(person["lastName"]); // Output: Schmidt

// Calling the method
person.greet(); // Output: Hello, my name is Leonie Schmidt

Die Methode greet ist eine Funktion, die als Eigenschaft des Objekts person definiert wurde. Sie wird verwendet, um eine Begrüßung auszugeben, die den Vor- und Nachnamen der Person enthält. Hier ist eine detaillierte Erklärung:

1. Definition der Methode

greet: function() {
    console.log("Hello, my name is " + this.firstName + " " + this.lastName);
}
  • greet: Der Name der Methode (eine Eigenschaft des Objekts person).
  • function(): Die Methode ist eine normale JavaScript-Funktion, die innerhalb des Objekts definiert wurde.
  • console.log(...): Gibt eine Begrüßung in der Konsole aus, die den Vor- und Nachnamen der Person enthält.

2. Verwendung von this

  • this: In einer Methode verweist this auf das Objekt, in dem die Methode aufgerufen wurde. In diesem Fall ist this das Objekt person.
  • this.firstName: Greift auf die Eigenschaft firstName des Objekts person zu.
  • this.lastName: Greift auf die Eigenschaft lastName des Objekts person zu.

3. Funktionsweise

Wenn die Methode greet aufgerufen wird, wird der Wert von this auf das Objekt person gesetzt. Dadurch kann die Methode auf die Eigenschaften firstName und lastName des Objekts zugreifen.

Beispiel:

person.greet(); // Ruft die Methode auf

Ablauf:

  1. this verweist auf person.
  1. this.firstName wird zu "Leonie".
  1. this.lastName wird zu "Schmidt".
  1. Die Ausgabe ist:
    Hello, my name is Leonie Schmidt

4. Warum ist this wichtig?

  • this ermöglicht es, innerhalb einer Methode auf die Eigenschaften des Objekts zuzugreifen, ohne den Objektnamen explizit anzugeben.
  • Wenn das Objekt umbenannt wird oder die Methode in einem anderen Kontext verwendet wird, bleibt der Zugriff auf die Eigenschaften dynamisch.

5. Beispiel mit einem anderen Objekt

Wenn du ein anderes Objekt mit einer ähnlichen Methode hast, funktioniert this entsprechend für dieses Objekt:

const anotherPerson = {
    firstName: "Max",
    lastName: "Mustermann",
    greet: function() {
        console.log("Hello, my name is " + this.firstName + " " + this.lastName);
    }
};

anotherPerson.greet(); // Output: Hello, my name is Max Mustermann

6. Zusammenfassung

  • greet ist eine Methode, die innerhalb des Objekts person definiert wurde.
  • this verweist auf das aktuelle Objekt (person), wodurch die Methode auf dessen Eigenschaften zugreifen kann.
  • Die Methode gibt eine Begrüßung aus, die den Vor- und Nachnamen des Objekts enthält.

Object.keys(), Object.values() und Object.entries()

Die Methoden Object.keys(), Object.values() und Object.entries() sind JavaScript-Funktionen, die verwendet werden, um die Eigenschaften eines Objekts zu durchlaufen. Sie unterscheiden sich darin, welche Teile des Objekts sie zurückgeben.

1. Object.keys()

  • Gibt ein Array mit den Schlüsseln (Keys) eines Objekts zurück.
  • Die Schlüssel sind die Namen der Eigenschaften des Objekts.

Beispiel:

let game = {
    title: "The Legend of Zelda",
    genre: "Action-adventure",
    platform: "Nintendo Switch",
};

console.log(Object.keys(game));

Ausgabe:

["title", "genre", "platform"]

Wann verwenden?

  • Wenn du nur die Schlüssel eines Objekts benötigst.
  • Zum Beispiel, um die Namen der Eigenschaften zu durchlaufen.

2. Object.values()

  • Gibt ein Array mit den Werten der Eigenschaften eines Objekts zurück.
  • Die Werte sind die Inhalte der Eigenschaften.

Beispiel:

let game = {
    title: "The Legend of Zelda",
    genre: "Action-adventure",
    platform: "Nintendo Switch",
};

console.log(Object.values(game));

Ausgabe:

["The Legend of Zelda", "Action-adventure", "Nintendo Switch"]

Wann verwenden?

  • Wenn du nur die Werte eines Objekts benötigst.
  • Zum Beispiel, um die Inhalte der Eigenschaften zu durchlaufen.

3. Object.entries()

  • Gibt ein Array von Arrays zurück, wobei jedes innere Array ein Schlüssel-Wert-Paar des Objekts enthält.
  • Jedes Paar besteht aus [key, value].

Beispiel:

let game = {
    title: "The Legend of Zelda",
    genre: "Action-adventure",
    platform: "Nintendo Switch",
};

console.log(Object.entries(game));

Ausgabe:

[
    ["title", "The Legend of Zelda"],
    ["genre", "Action-adventure"],
    ["platform", "Nintendo Switch"]
]

Wann verwenden?

  • Wenn du sowohl die Schlüssel als auch die Werte eines Objekts benötigst.
  • Zum Beispiel, um Schlüssel-Wert-Paare zu durchlaufen.

Vergleich der Methoden

MethodeRückgabeBeispiel
Object.keys()Array mit den Schlüsseln des Objekts["title", "genre", "platform"]
Object.values()Array mit den Werten des Objekts["The Legend of Zelda", "Action-adventure", "Nintendo Switch"]
Object.entries()Array von Arrays, wobei jedes Array ein Schlüssel-Wert-Paar enthält[["title", "The Legend of Zelda"], ["genre", "Action-adventure"], ["platform", "Nintendo"]]

Beispiele für die Verwendung

1. Iteration mit Object.keys()

Object.keys(game).forEach(key => {
    console.log(key); // Gibt die Schlüssel aus
});

Ausgabe:

title
genre
platform

2. Iteration mit Object.values()

Object.values(game).forEach(value => {
    console.log(value); // Gibt die Werte aus
});

Ausgabe:

The Legend of Zelda
Action-adventure
Nintendo Switch

3. Iteration mit Object.entries()

Object.entries(game).forEach(([key, value]) => {
    console.log(`${key}: ${value}`); // Gibt Schlüssel und Werte aus
});

Ausgabe:

title: The Legend of Zelda
genre: Action-adventure
platform: Nintendo Switch

Zusammenfassung

  • Object.keys(): Gibt die Schlüssel eines Objekts zurück.
  • Object.values(): Gibt die Werte eines Objekts zurück.
  • Object.entries(): Gibt Schlüssel-Wert-Paare als Arrays zurück.

Verwende:

  • Object.keys() für die Schlüssel.
  • Object.values() für die Werte.
  • Object.entries() für beides zusammen.

Das for...inSchleifenprinzip in JavaScript

Die for...in-Schleife wird in JavaScript verwendet, um über die Eigenschaften eines Objekts zu iterieren. Sie gibt die Schlüssel (Property-Namen) eines Objekts zurück.

Syntax

for (let key in object) {
    // Codeblock, der für jede Eigenschaft ausgeführt wird
}
  • key: Eine Variable, die den Namen der aktuellen Eigenschaft (Schlüssel) enthält.
  • object: Das Objekt, dessen Eigenschaften durchlaufen werden.

Beispiel: Iteration über ein Objekt

let person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
};

for (let key in person) {
    console.log(key + ": " + person[key]);
}

Ausgabe:

firstName: John
lastName: Doe
age: 30

Wichtige Punkte zu for...in

  1. Iteriert nur über enumerable Eigenschaften:
    • Die Schleife durchläuft nur Eigenschaften, die als enumerable markiert sind (standardmäßig sind alle Eigenschaften von Objekten enumerable).
  1. Schlüssel sind Strings:
    • Die zurückgegebenen Schlüssel sind immer Strings, auch wenn sie als Zahlen definiert wurden.
    let obj = { 1: "one", 2: "two" };
for (let key in obj) {
    console.log(typeof key); // "string"
}
  1. Vererbung:
    • Die Schleife iteriert auch über Eigenschaften, die von der Prototypkette geerbt wurden. Um dies zu vermeiden, kannst du die Methode hasOwnProperty() verwenden.
    for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
        console.log(key);
    }
}
  1. Nicht für Arrays empfohlen:
    • Obwohl for...in auch über Arrays iterieren kann, ist es nicht ideal, da es auch über nicht-numerische Eigenschaften iteriert (z. B. Methoden oder Prototypen). Verwende stattdessen for oder forEach für Arrays.
    let arr = [10, 20, 30];
for (let index in arr) {
    console.log(index); // Gibt die Indizes als Strings zurück: "0", "1", "2"
    console.log(arr[index]); // Gibt die Werte zurück: 10, 20, 30
}

Anwendung in deinem Code

In deinem Code wird for...in verwendet, um die Eigenschaften des Objekts myCar zu durchlaufen:

function printObject(obj, prefix = '') {
    for (let key in obj) {
        if (typeof obj[key] === 'object' && obj[key] !== null) {
            if (obj[key].constructor.name === 'Date') {
                printLn(prefix + '  ' + obj[key].toLocaleDateString('de-AT', { day: '2-digit', month: 'long', year: 'numeric' }) + " (" + obj[key].constructor.name + ")");
            } else {
                printLn(prefix + key + " (" + obj[key].constructor.name + ")" + ": ");
                printObject(obj[key], prefix + '  ');
            }
        } else {
            printLn(prefix + key + ": " + obj[key] + " (" + typeof obj[key] + ")");
        }
    }
}

Wie funktioniert das?

  1. for...in iteriert über die Schlüssel von obj:
    • Für jedes Schlüssel-Wert-Paar im Objekt wird der Schlüssel in der Variablen key gespeichert.
  1. Prüfung des Werts:
    • Wenn der Wert ein Objekt ist, wird rekursiv die Funktion printObject aufgerufen.
    • Wenn der Wert ein Datum ist, wird es formatiert und ausgegeben.
    • Andernfalls wird der Schlüssel und der Wert direkt ausgegeben.

Zusammenfassung

  • for...in ist ideal, um die Schlüssel eines Objekts zu durchlaufen.
  • Es iteriert auch über geerbte Eigenschaften, daher sollte hasOwnProperty() verwendet werden, wenn nur eigene Eigenschaften berücksichtigt werden sollen.
  • Für Arrays ist for...in nicht optimal; verwende stattdessen for oder forEach.

Wird in einer for in schleife das key selber definiert? Kann es z.b. x heißen? Welchen Wert trägt dann die variable?

Ja, in einer for...in-Schleife kannst du die Variable, die die Schlüssel (Keys) des Objekts speichert, selbst benennen. Sie kann z. B. x, key, property oder einen beliebigen anderen gültigen Variablennamen haben.

Beispiel: Variable x als Key

for (let x in myCar) {
    console.log(x); // Gibt die Schlüssel (Keys) des Objekts `myCar` aus
}

Ausgabe:

type
model
color
year
owner
service
fuel
mileage
built
hasAutomatic

Was speichert die Variable x?

Die Variable x enthält bei jedem Durchlauf der Schleife den aktuellen Schlüssel (Key) des Objekts. In deinem Beispiel myCar wären das die Namen der Eigenschaften wie type, model, color, usw.

Zugriff auf den Wert des Schlüssels

Um den Wert zu erhalten, der mit dem aktuellen Schlüssel verknüpft ist, kannst du die Bracket-Notation verwenden:

for (let x in myCar) {
    console.log(x + ": " + myCar[x]); // Gibt Schlüssel und zugehörigen Wert aus
}

Ausgabe:

type: Opel
model: Astra
color: black
year: 2017
owner: [object Object]
service: [object Object],[object Object]
fuel: diesel
mileage: 120000
built: Thu Jun 15 2017 00:00:00 GMT+0200 (Central European Summer Time)
hasAutomatic: false

Zusammenfassung

  • Die Variable in einer for...inSchleife (z. B. x) speichert den aktuellen Schlüssel (Key) des Objekts.
  • Du kannst die Variable beliebig benennen, solange es ein gültiger Variablenname ist.
  • Um den Wert des Schlüssels zu erhalten, verwende die Bracket-Notation: obj[x].

for...in und forEach:

In JavaScript gibt es Unterschiede zwischen for...in und forEach, und sie haben unterschiedliche Anwendungsfälle. Hier ist eine Erklärung, wann du welche verwenden solltest:

1. for...in für Objekte

Die for...in-Schleife wird verwendet, um über die Schlüssel (Properties) eines Objekts zu iterieren.

Wann for...in verwenden?

  • Wenn du die Schlüssel (Keys) eines Objekts durchlaufen möchtest.
  • Wenn du sowohl die Schlüssel als auch die zugehörigen Werte benötigst.
  • Wenn du mit Objekten arbeitest, da forEach direkt nicht auf Objekte angewendet werden kann.

Beispiel: Iteration über ein Objekt

let game = {
    title: "The Legend of Zelda",
    genre: "Action-adventure",
    platform: "Nintendo Switch",
};

for (let key in game) {
    console.log(key + ": " + game[key]);
}

Ausgabe:

title: The Legend of Zelda
genre: Action-adventure
platform: Nintendo Switch

Vorteile von for...in:

  • Einfach zu verwenden, wenn du die Schlüssel eines Objekts benötigst.
  • Funktioniert direkt mit Objekten.

Nachteile von for...in:

  • Es iteriert auch über geerbte Eigenschaften aus der Prototypkette. Um dies zu vermeiden, solltest du hasOwnProperty() verwenden:
    for (let key in game) {
    if (game.hasOwnProperty(key)) {
        console.log(key + ": " + game[key]);
    }
}

2. forEach für Arrays

Die forEach-Methode wird verwendet, um über die Elemente eines Arrays zu iterieren. Sie ist speziell für Arrays gedacht und kann nicht direkt auf Objekte angewendet werden.

Wann forEach verwenden?

  • Wenn du mit Arrays arbeitest.
  • Wenn du nur die Werte eines Arrays durchlaufen möchtest.
  • Wenn du eine Callback-Funktion verwenden möchtest, um die Iteration zu steuern.

Beispiel: Iteration über ein Array

let services = [
    { date: "2018-05-15", type: "oil change", cost: 150 },
    { date: "2019-05-15", type: "brake pads replacement", cost: 300 },
];

services.forEach(service => {
    console.log(`${service.type} on ${service.date} cost ${service.cost}€`);
});

Ausgabe:

oil change on 2018-05-15 cost 150€
brake pads replacement on 2019-05-15 cost 300€

Vorteile von forEach:

  • Einfach zu verwenden, wenn du nur mit Arrays arbeitest.
  • Unterstützt Callback-Funktionen, was die Iteration flexibler macht.

Nachteile von forEach:

  • Kann nicht direkt mit Objekten verwendet werden.
  • Du kannst die Iteration nicht abbrechen (dafür wäre eine forSchleife besser geeignet).

3. Objekte in Arrays umwandeln

Wenn du forEach für ein Objekt verwenden möchtest, kannst du das Objekt in ein Array umwandeln, z. B. mit Object.keys(), Object.values() oder Object.entries().

Beispiel: Object.keys() mit forEach

let game = {
    title: "The Legend of Zelda",
    genre: "Action-adventure",
    platform: "Nintendo Switch",
};

Object.keys(game).forEach(key => {
    console.log(key + ": " + game[key]);
});

Ausgabe:

title: The Legend of Zelda
genre: Action-adventure
platform: Nintendo Switch

Beispiel: Object.entries() mit forEach

Object.entries(game).forEach(([key, value]) => {
    console.log(key + ": " + value);
});

Ausgabe:

title: The Legend of Zelda
genre: Action-adventure
platform: Nintendo Switch

Zusammenfassung

  • Verwende for...in, wenn du über die Schlüssel eines Objekts iterieren möchtest.
  • Verwende forEach, wenn du über die Elemente eines Arrays iterieren möchtest.
  • Wenn du forEach für ein Objekt verwenden möchtest, konvertiere das Objekt mit Object.keys(), Object.values() oder Object.entries() in ein Array.

if (typeof obj[key])

Die Bedingung if (typeof obj[key]) wird verwendet, um den Datentyp des Werts zu überprüfen, der mit einem bestimmten Schlüssel (key) in einem Objekt (obj) verknüpft ist. Hier ist eine detaillierte Erklärung:

1. Was macht typeof?

  • Der Operator typeof gibt den Datentyp eines Werts als String zurück.
  • Beispiele:
    console.log(typeof 42); // "number"
console.log(typeof "Hello"); // "string"
console.log(typeof true); // "boolean"
console.log(typeof {}); // "object"
console.log(typeof undefined); // "undefined"

2. Was ist obj[key]?

  • obj[key] greift auf den Wert zu, der mit dem Schlüssel key im Objekt obj verknüpft ist.
  • Beispiel:
    let person = { name: "Leonie", age: 20 };
console.log(person["name"]); // "Leonie"
console.log(person["age"]); // 20

3. Kombination: typeof obj[key]

  • typeof obj[key] gibt den Datentyp des Werts zurück, der mit dem Schlüssel key im Objekt obj verknüpft ist.
  • Beispiel:
    let person = { name: "Leonie", age: 20, isStudent: true };
console.log(typeof person["name"]); // "string"
console.log(typeof person["age"]); // "number"
console.log(typeof person["isStudent"]); // "boolean"

4. Verwendung in einer ifBedingung

Die Bedingung if (typeof obj[key]) wird verwendet, um den Datentyp zu überprüfen und darauf basierend eine Aktion auszuführen.

Beispiel: Überprüfung auf object

let person = {
    name: "Leonie",
    age: 20,
    address: { city: "Dornbirn", country: "Austria" },
};

for (let key in person) {
    if (typeof person[key] === "object") {
        console.log(`${key} ist ein Objekt.`);
    } else {
        console.log(`${key} ist kein Objekt.`);
    }
}

Ausgabe:

name ist kein Objekt.
age ist kein Objekt.
address ist ein Objekt.

5. Typische Anwendungsfälle

a) Überprüfung auf undefined

  • Um zu prüfen, ob ein Schlüssel im Objekt existiert:
    if (typeof obj[key] === "undefined") {
    console.log(`${key} existiert nicht im Objekt.`);
}

b) Überprüfung auf function

  • Um zu prüfen, ob ein Schlüssel eine Funktion enthält:
    if (typeof obj[key] === "function") {
    obj[key](); // Funktion ausführen
}

c) Überprüfung auf object

  • Um zu prüfen, ob ein Schlüssel ein verschachteltes Objekt enthält:
    if (typeof obj[key] === "object" && obj[key] !== null) {
    console.log(`${key} ist ein verschachteltes Objekt.`);
}

Zusammenfassung

  • typeof obj[key] prüft den Datentyp des Werts, der mit dem Schlüssel key im Objekt obj verknüpft ist.
  • Es wird häufig verwendet, um sicherzustellen, dass der Wert den erwarteten Typ hat, bevor eine Aktion ausgeführt wird.
  • Typische Anwendungsfälle:
    • Überprüfung auf undefined, object, function oder andere Datentypen.
