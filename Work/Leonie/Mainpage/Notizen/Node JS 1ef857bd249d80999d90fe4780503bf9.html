1. Was ist eine JS Engine?

Eine JavaScript Engine ist ein Programm, das JavaScript-Code ausführt. Sie interpretiert und kompiliert JavaScript in Maschinencode, damit der Computer ihn ausführen kann. Beispiele:

  • V8: Die Engine von Google Chrome und Node.js.
  • SpiderMonkey: Die Engine von Mozilla Firefox.

2. Was ist der Unterschied zwischen JS Engine in einem Webbrowser und Node.JS?

  • Webbrowser:
    • Die JS Engine (z. B. V8) läuft im Browser und hat Zugriff auf browser-spezifische APIs wie window, document, alert, etc.
    • Hauptsächlich für Frontend-Entwicklung gedacht.
  • Node.js:
    • Verwendet die V8-Engine, aber ohne browser-spezifische APIs.
    • Stattdessen bietet Node.js serverseitige APIs wie fs (Dateisystem), http (Server), etc.
    • Hauptsächlich für Backend-Entwicklung gedacht.

3. Was funktioniert NICHT in Node.JS?

  • Browser-spezifische Funktionen wie:
    • window, document, alert
    • DOM-Manipulation
    • localStorage oder sessionStorage

4. Wofür steht npm?

npm steht für Node Package Manager. Es ist das Standard-Tool, um:

  • Node.js-Pakete (Libraries) zu installieren, zu verwalten und zu aktualisieren.
  • Abhängigkeiten in einem Projekt zu verwalten.

5. Was bedeutet die Abkürzung CLI?

CLI steht für Command Line Interface. Es ist eine textbasierte Benutzeroberfläche, mit der du Befehle direkt in der Konsole/Terminal eingeben kannst.

6. Was ist ein Node.JS Project?

Ein Node.js-Projekt ist eine Anwendung oder ein Skript, das mit Node.js ausgeführt wird. Es enthält typischerweise:

  • Eine package.jsonDatei, die die Abhängigkeiten und Metadaten des Projekts beschreibt.
  • JavaScript-Dateien, die den Code enthalten.

7. Wie sieht die Grundstruktur von package.json aus?

{
  "name": "mein-projekt",
  "version": "1.0.0",
  "description": "Beschreibung des Projekts",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {},
  "devDependencies": {}
}

8. Wie erstelle ich ein neues Projekt?

  1. Öffne das Terminal.
  1. Erstelle ein neues Verzeichnis:
    mkdir mein-projekt
cd mein-projekt
  1. Initialisiere das Projekt:
    (oder
    npm init -y, um die Standardwerte zu verwenden)
    npm init

9. Wie starte ich mein Projekt?

  1. Stelle sicher, dass du eine startScript in der package.json hast:
    "scripts": {
  "start": "node index.js"
}
  1. Starte das Projekt:
    npm start

10. Was sind Node Packages (dependency)?

Node Packages sind wiederverwendbare Module oder Bibliotheken, die du in deinem Projekt installieren kannst. Sie werden in der Datei node_modules gespeichert und in der package.json als Abhängigkeiten aufgeführt.

11. Was sind Kriterien für die Auswahl der Node Packages?

  • Beliebtheit: Anzahl der Downloads und Sterne auf GitHub.
  • Aktualität: Regelmäßige Updates und Wartung.
  • Dokumentation: Gute und verständliche Dokumentation.
  • Sicherheit: Überprüfe bekannte Sicherheitsprobleme (z. B. mit npm audit).
  • Kompatibilität: Passt das Package zu deiner Node.js-Version?

12. Wie installiere ich neue Packages?

  • Installiere ein Package und füge es zu den Abhängigkeiten hinzu:
    npm install <package-name>
  • Für Entwicklungsabhängigkeiten:
    npm install <package-name> --save-dev

13. Wie aktualisiere ich bestehende Packages?

  • Aktualisiere ein bestimmtes Package:
    npm update <package-name>
  • Aktualisiere alle Abhängigkeiten:
    npm update

14. Wie entferne ich Packages, die ich nicht mehr brauche?

  • Entferne ein Package:
    npm uninstall <package-name>

15. Was muss in .gitignore stehen?

In der Datei .gitignore sollten Dateien und Ordner stehen, die nicht in das Git-Repository aufgenommen werden sollen. Für Node.js-Projekte:

node_modules/
.env
.DS_Store
npm-debug.log

Zusammenfassung

Node.js ist eine serverseitige Laufzeitumgebung für JavaScript. Es verwendet die V8-Engine, bietet serverseitige APIs und wird durch npm unterstützt, um Pakete zu verwalten. Die package.json ist das Herzstück eines Node.js-Projekts, und der Umgang mit Abhängigkeiten ist ein zentraler Bestandteil der Arbeit mit Node.js.

1. Grundlagen von Node.js

Was ist Node.js?

  • Node.js ist eine serverseitige JavaScript-Laufzeitumgebung, die auf der V8-Engine von Google basiert.
  • Es ermöglicht die Ausführung von JavaScript außerhalb des Browsers, z. B. auf einem Server.
  • Es ist asynchron und ereignisgesteuert, was es ideal für Anwendungen mit hoher Parallelität macht (z. B. APIs).

Wichtige Eigenschaften von Node.js

  • Single-Threaded: Node.js verwendet einen einzigen Thread, um Anfragen zu bearbeiten, aber es kann dank des Event-Loop-Modells viele Anfragen gleichzeitig verarbeiten.
  • Non-Blocking I/O: Eingabe-/Ausgabeoperationen (z. B. Datenbankabfragen) blockieren den Hauptthread nicht.
  • Modularität: Node.js verwendet das npmÖkosystem, das Tausende von Paketen bietet, um die Entwicklung zu beschleunigen.

Wie wird Node.js erstellt?

Node.js wurde von Ryan Dahl im Jahr 2009 entwickelt. Es basiert auf der V8-JavaScript-Engine von Google, die in C++ geschrieben ist. Die wichtigsten Bestandteile von Node.js sind:

  • V8-Engine: Führt JavaScript aus.
  • libuv: Eine Bibliothek für asynchrone I/O und Event-Loop-Management.
  • C++-Bindings: Verbindet die V8-Engine mit nativen Systemfunktionen.

Installation und Nutzung:

  1. Installation:
    • Lade Node.js von der offiziellen Website herunter: https://nodejs.org.
    • Alternativ kannst du auf einem Mac den Paketmanager brew verwenden:
      brew install node
  1. Erstellen einer Node.js-Anwendung:
    • Erstelle eine Datei, z. B. app.js:
      // app.js
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello, World!\\n');
});

server.listen(3000, () => {
    console.log('Server läuft auf <http://localhost:3000>');
});
    • Starte die Anwendung:
      node app.js
  1. Paketverwaltung mit npm:
    • Initialisiere ein Projekt:
      npm init -y
    • Installiere ein Paket:
      npm install express

Node.js ist besonders nützlich für Echtzeitanwendungen wie Chats, APIs und Streaming-Dienste.

2. Integration mit SQL-Datenbanken

Wahl der Datenbank

  • Relationale Datenbanken (SQL): MySQL, PostgreSQL, SQLite.
  • NoSQL-Datenbanken: MongoDB, Redis (falls du keine relationalen Daten benötigst).

MySQL-Integration mit Node.js

a) MySQL-Paket

  • Verwende das Paket mysql2 (wie in deinem Code), da es modern und effizient ist.
  • Installiere es mit:
    npm install mysql2

b) Verbindung zur Datenbank

  • Stelle sicher, dass du eine Datenbankverbindung erstellst und Fehler abfängst:
    const mysql = require('mysql2');

const db = mysql.createConnection({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME
});

db.connect((err) => {
  if (err) {
    console.error('Fehler bei der Verbindung zur Datenbank:', err);
    return;
  }
  console.log('Erfolgreich mit der Datenbank verbunden.');
});

c) SQL-Abfragen

  • Verwende Prepared Statements, um SQL-Injection zu vermeiden:
    const sqlQuery = 'SELECT * FROM users WHERE username = ?';
db.query(sqlQuery, [username], (err, results) => {
  if (err) {
    console.error('Datenbankfehler:', err);
    return;
  }
  console.log(results);
});

d) Verbindungspool

  • Für größere Anwendungen ist ein Verbindungspool besser, um die Leistung zu verbessern:
    const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

pool.query('SELECT * FROM users', (err, results) => {
  if (err) throw err;
  console.log(results);
});

3. Sicherheitsaspekte

a) SQL-Injection verhindern

  • Verwende immer Prepared Statements (wie oben gezeigt), um Benutzereingaben zu validieren.

b) Umgebungsvariablen

  • Speichere sensible Daten wie Datenbank-Logins in einer .envDatei:
    DB_HOST=localhost
DB_USER=root
DB_PASSWORD=geheim
DB_NAME=meinedatenbank
PORT=3000
  • Lade die Variablen mit dem Paket dotenv:
    require('dotenv').config();

c) Fehlerbehandlung

  • Fange Fehler ab, um Abstürze zu vermeiden:
    db.query('SELECT * FROM users', (err, results) => {
  if (err) {
    console.error('Fehler:', err);
    res.status(500).send('Interner Serverfehler');
    return;
  }
  res.json(results);
});

4. Best Practices

a) Modularisierung

  • Teile deinen Code in Module auf, um ihn übersichtlicher zu machen:
    • server.js: Startet den Server.
    • routes/: Enthält die API-Routen.
    • controllers/: Enthält die Logik für jede Route.
    • models/: Enthält die Datenbankabfragen.

Beispiel:

// routes/userRoutes.js
const express = require('express');
const router = express.Router();
const userController = require('../controllers/userController');

router.get('/users', userController.getAllUsers);

module.exports = router;

b) Asynchrone Programmierung

  • Verwende Promises oder async/await, um asynchrone Operationen übersichtlicher zu machen:
    const getUsers = async () => {
  try {
    const [rows] = await db.promise().query('SELECT * FROM users');
    console.log(rows);
  } catch (err) {
    console.error('Fehler:', err);
  }
};

c) Logging

  • Nutze ein Logging-Tool wie winston, um Fehler und Anfragen zu protokollieren:
    npm install winston

5. Performance-Tipps

a) Caching

  • Verwende ein Caching-Tool wie Redis, um häufige Datenbankabfragen zu vermeiden.

b) Verbindungspool

  • Nutze einen Verbindungspool (wie oben gezeigt), um die Datenbankverbindungen effizient zu verwalten.

c) Pagination

  • Lade nur die benötigten Daten aus der Datenbank, z. B. mit LIMIT und OFFSET:
    SELECT * FROM users LIMIT 10 OFFSET 0;

6. Tools und Frameworks

a) Express.js

  • Express ist das am häufigsten verwendete Framework für Node.js, um APIs zu erstellen.

b) Sequelize

  • Sequelize ist ein ORM (Object-Relational Mapping), das die Arbeit mit SQL-Datenbanken vereinfacht:
    npm install sequelize mysql2

c) Postman

  • Verwende Postman, um deine API zu testen.

7. Beispielprojekt

Hier ist ein einfaches Beispiel für eine Node.js-App mit MySQL:

const express = require('express');
const mysql = require('mysql2');
require('dotenv').config();

const app = express();
const db = mysql.createConnection({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME
});

db.connect((err) => {
  if (err) throw err;
  console.log('Datenbank verbunden');
});

app.get('/users', (req, res) => {
  db.query('SELECT * FROM users', (err, results) => {
    if (err) {
      console.error(err);
      res.status(500).send('Fehler bei der Datenbankabfrage');
      return;
    }
    res.json(results);
  });
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server läuft auf <http://localhost>:${PORT}`);
});

Fazit

  • Node.js ist ideal für APIs und serverseitige Anwendungen.
  • Achte auf Sicherheit (SQL-Injection, Umgebungsvariablen).
  • Verwende Prepared Statements und Verbindungspools für SQL-Datenbanken.
  • Modularisiere deinen Code und nutze Tools wie Postman und winston.

Wie importiere eine Funktion / Variable aus einem anderen Skript?

In Node.js kannst du Funktionen oder Variablen aus einem anderen Skript mithilfe von module.exports und require importieren/exportieren. Ab Node.js 14+ kannst du auch ES6-Module verwenden, die mit export und import arbeiten.

1. Mit module.exports und require (CommonJS)

Exportieren

In der Datei, aus der du etwas exportieren möchtest (z. B. utils.js):

// utils.js
function add(a, b) {
    return a + b;
}

const PI = 3.14;

// Exportiere die Funktion und die Variable
module.exports = { add, PI };

Importieren

In der Datei, in der du die Funktion/Variable verwenden möchtest (z. B. app.js):

// app.js
const { add, PI } = require('./utils');

console.log(add(2, 3)); // 5
console.log(PI); // 3.14

2. Mit export und import (ES6-Module)

Exportieren

In der Datei, aus der du etwas exportieren möchtest (z. B. utils.js):

// utils.js
export function add(a, b) {
    return a + b;
}

export const PI = 3.14;

Importieren

In der Datei, in der du die Funktion/Variable verwenden möchtest (z. B. app.js):

// app.js
import { add, PI } from './utils.js';

console.log(add(2, 3)); // 5
console.log(PI); // 3.14

Hinweis:

  • Um ES6-Module in Node.js zu verwenden, musst du in der package.json die Eigenschaft "type": "module" hinzufügen:
    {
  "type": "module"
}

3. Unterschied zwischen CommonJS und ES6-Module

  • CommonJS (require/module.exports):
    • Standard in Node.js.
    • Unterstützt in allen Node.js-Versionen.
  • ES6-Module (import/export):
    • Moderner Standard.
    • Muss explizit aktiviert werden ("type": "module" in package.json).

Zusammenfassung

  • Verwende module.exports und require für ältere Node.js-Versionen oder wenn du keine ES6-Module aktivieren möchtest.
  • Verwende export und import, wenn du moderne ES6-Module bevorzugst und "type": "module" in deiner package.json gesetzt hast.

Hier ist eine ausführliche Erklärung der Datei server.js, Zeile für Zeile, damit du die Funktionsweise verstehst. Ich werde die Konzepte so einfach wie möglich erklären.

1. Import von Modulen

const express = require('express');
  • require('express'): Importiert das Express.js Framework, das die Erstellung von Webservern und APIs erleichtert.
  • const app = express();: Erstellt eine neue Express-Anwendung, die als Webserver fungiert.
require('dotenv').config();
  • dotenv: Ein Paket, das Umgebungsvariablen aus einer .envDatei lädt.
  • config(): Aktiviert die Nutzung der .envDatei, um sensible Daten wie Datenbank-Logins oder Ports sicher zu speichern.
const mysql = require('mysql2');
  • mysql2: Ein Paket, das die Verbindung zu einer MySQL-Datenbank ermöglicht und SQL-Abfragen ausführt.

2. Verbindung zur MySQL-Datenbank

const db = mysql.createConnection({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME
});
  • mysql.createConnection: Erstellt eine Verbindung zur MySQL-Datenbank.
  • host: Die Adresse des Datenbankservers (z. B. localhost oder eine IP-Adresse).
  • user: Der Benutzername für die Datenbank.
  • password: Das Passwort für den Benutzer.
  • database: Der Name der Datenbank, mit der du arbeiten möchtest.
  • process.env: Liest die Werte aus der .envDatei, um sensible Daten nicht direkt im Code zu speichern.
db.connect((err) => {
  if (err) {
    console.error('Fehler bei der Verbindung zur Datenbank:', err);
    return;
  }
  console.log('Erfolgreich mit der Datenbank verbunden.');
});
  • db.connect: Stellt die Verbindung zur Datenbank her.
  • err: Wenn ein Fehler auftritt (z. B. falsche Zugangsdaten), wird dieser hier abgefangen und ausgegeben.
  • console.log: Gibt eine Erfolgsmeldung aus, wenn die Verbindung erfolgreich hergestellt wurde.

3. Template-Engine konfigurieren

app.set('view engine', 'ejs');
  • app.set: Konfiguriert die Express-Anwendung.
  • view engine: Legt fest, dass EJS (Embedded JavaScript) als Template-Engine verwendet wird. Damit kannst du HTML-Seiten dynamisch mit Daten füllen.

4. Routen definieren

Route 1: / (Startseite)

app.get('/', (req, res) => {
  res.json({message: 'Hello World!'})
});
  • app.get('/'): Definiert eine Route für die URL / (Startseite).
  • req: Das Anfrageobjekt (enthält Informationen über die Anfrage des Clients).
  • res: Das Antwortobjekt (wird verwendet, um eine Antwort an den Client zu senden).
  • res.json: Sendet eine JSON-Antwort mit der Nachricht Hello World!.

Route 2: / (SQL-Abfrage)

app.get('/', (req, res) => {
    db.query('SELECT * from country', (err, rows, fields) => {
        if (err) throw err;

        res.render('pages/index', {
            title: 'SQL Example',
            header: fields,
            content: rows
        });
    });
});
  • db.query: Führt eine SQL-Abfrage aus. Hier wird SELECT * FROM country verwendet, um alle Daten aus der Tabelle country abzurufen.
  • err: Wenn ein Fehler bei der Abfrage auftritt, wird dieser hier abgefangen.
  • rows: Enthält die Ergebnisse der Abfrage (z. B. die Länder aus der Tabelle country).
  • fields: Enthält die Spaltennamen der Tabelle.
  • res.render: Rendert die EJS-Datei pages/index und übergibt die Daten (title, header, content) an die Seite.

Route 3: /city

app.get('/city', (req, res) => {
    let name = "%" + req.query.name + "%";

    let sqlQueryText = 'select * from city where name like ?';
    db.query(sqlQueryText, [name], (err, rows, fields) => {
        if (err) throw err;

        res.render('pages/index', {
            title: 'SQL Example',
            header: fields,
            content: rows,
            searchAction: 'city',
            searchField: 'name'
        });
    });
});
  • req.query.name: Liest den Parameter name aus der URL (z. B. /city?name=Berlin).
  • % + name + %: Fügt Wildcards hinzu, um eine SQL-Suche mit LIKE zu ermöglichen.
  • sqlQueryText: Die SQL-Abfrage sucht nach Städten, deren Name dem Suchbegriff entspricht.
  • res.render: Rendert die Ergebnisse in der EJS-Datei pages/index.

Route 4: /country

app.get('/country', (req, res) => {
    let name = "%" + req.query.name + "%";

    let sqlQueryText = 'select * from country where name like ?';
    db.query(sqlQueryText, [name], (err, rows, fields) => {
        if (err) throw err;

        res.render('pages/index', {
            title: 'SQL Example',
            header: fields,
            content: rows,
            searchAction: 'country',
            searchField: 'name'
        });
    });
});
  • Funktioniert ähnlich wie die /cityRoute, sucht jedoch in der Tabelle country nach Ländern, deren Name dem Suchbegriff entspricht.

5. Fehlerbehandlung für fehlenden Port

if (!process.env.PORT) {
  console.error('Fehler: DB_PORT ist nicht gesetzt. Bitte überprüfen Sie Ihre .env-Datei.');
  process.exit(1);
}
  • process.env.PORT: Überprüft, ob der Port in der .envDatei definiert ist.
  • process.exit(1): Beendet die Anwendung, wenn der Port nicht gesetzt ist.

6. Server starten

const PORT = process.env.PORT;
app.listen(PORT, () => {
  console.log(`Server läuft auf <http://localhost>:${PORT}`);
});
  • app.listen: Startet den Server und lauscht auf dem angegebenen Port.
  • PORT: Der Port wird aus der .envDatei gelesen.
  • console.log: Gibt eine Nachricht aus, dass der Server läuft.
