Async-Funktionen und Promises: Grundlagen und Funktionsweise

Was ist eine asyncFunktion?

Eine async-Funktion ist eine spezielle Funktion in JavaScript, die es ermöglicht, asynchrone Operationen (z. B. Netzwerkaufrufe, Datenbankabfragen) einfacher und lesbarer zu schreiben. Sie basiert auf Promises und erlaubt die Verwendung des await-Schlüsselworts, um den Code wie synchronen Code aussehen zu lassen.

Wichtige Eigenschaften von asyncFunktionen

  1. Rückgabewert ist immer ein Promise:
    • Eine asyncFunktion gibt immer ein Promise zurück, unabhängig davon, ob sie explizit ein Promise zurückgibt oder nicht.
    • Beispiel:
      Hier wird der Rückgabewert
      "Hello, World!" automatisch in ein Promise gewrappt.
      async function example() {
    return "Hello, World!";
}
example().then(console.log); // Ausgabe: "Hello, World!"
  1. awaitSchlüsselwort:
    • Innerhalb einer asyncFunktion kannst du await verwenden, um auf die Auflösung eines Promises zu warten.
    • Während await wartet, wird der Rest des Codes nicht blockiert (non-blocking).
    • Beispiel:
      async function fetchData() {
    const response = await fetch("<https://api.example.com/data>");
    const data = await response.json();
    console.log(data);
}
fetchData();
  1. Fehlerbehandlung mit try...catch:
    • Fehler in einer asyncFunktion können mit try...catch behandelt werden, ähnlich wie bei synchronem Code.
    • Beispiel:
      async function fetchData() {
    try {
        const response = await fetch("<https://api.example.com/data>");
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Fehler beim Abrufen der Daten:", error);
    }
}
fetchData();

Wie funktioniert eine asyncFunktion intern?

  1. Promise-basierte Architektur:
    • Eine asyncFunktion ist syntaktischer Zucker (syntactic sugar) für die Arbeit mit Promises. Intern wird der Code in eine Kette von thenund catchAufrufen umgewandelt.
  1. Event Loop und Microtasks:
    • Wenn await auf ein Promise trifft, wird die Ausführung der Funktion pausiert, und die Kontrolle wird an die Event Loop zurückgegeben.
    • Sobald das Promise aufgelöst ist (fulfilled oder rejected), wird die Funktion fortgesetzt.
    • await arbeitet mit der Microtask-Queue, was bedeutet, dass aufgelöste Promises Vorrang vor anderen Aufgaben in der Event Loop haben.
  1. Nicht-blockierende Ausführung:
    • Während die asyncFunktion auf ein Promise wartet, können andere Teile des Programms weiterlaufen. Dies macht asyncFunktionen ideal für I/O-Operationen wie API-Aufrufe oder Dateizugriffe.

Wichtige Punkte zu Promises

  1. Promise-Zustände:
    • Ein Promise hat drei Zustände:
      • Pending: Das Promise wurde erstellt, aber die Operation ist noch nicht abgeschlossen.
      • Fulfilled: Die Operation wurde erfolgreich abgeschlossen, und ein Wert ist verfügbar.
      • Rejected: Die Operation ist fehlgeschlagen, und ein Fehler ist verfügbar.
  1. Promise-Methoden:
    • then: Wird aufgerufen, wenn das Promise erfolgreich aufgelöst wurde.
    • catch: Wird aufgerufen, wenn das Promise abgelehnt wurde.
    • finally: Wird immer aufgerufen, unabhängig davon, ob das Promise erfüllt oder abgelehnt wurde.

    Beispiel:

    const promise = new Promise((resolve, reject) => {
    setTimeout(() => resolve("Erfolg!"), 1000);
});

promise
    .then(result => console.log(result)) // Ausgabe: "Erfolg!"
    .catch(error => console.error(error))
    .finally(() => console.log("Fertig!"));
  1. Promise.all und Promise.race:
    • Promise.all: Wartet, bis alle Promises in einem Array erfüllt sind, oder gibt einen Fehler zurück, wenn eines fehlschlägt.
      const p1 = Promise.resolve(1);
const p2 = Promise.resolve(2);
const p3 = Promise.resolve(3);

Promise.all([p1, p2, p3]).then(results => console.log(results)); // Ausgabe: [1, 2, 3]
    • Promise.race: Gibt das erste Promise zurück, das erfüllt oder abgelehnt wird.
      const p1 = new Promise(resolve => setTimeout(() => resolve("P1"), 1000));
const p2 = new Promise(resolve => setTimeout(() => resolve("P2"), 500));

Promise.race([p1, p2]).then(result => console.log(result)); // Ausgabe: "P2"

Best Practices

  1. Verwende async/await für Lesbarkeit:
    • async/await macht den Code einfacher zu lesen und zu verstehen, besonders bei komplexen asynchronen Abläufen.
  1. Fehlerbehandlung nicht vergessen:
    • Immer try...catch oder .catch() verwenden, um Fehler zu behandeln.
  1. Promise.all für parallele Operationen:
    • Nutze Promise.all, wenn mehrere asynchrone Operationen unabhängig voneinander ausgeführt werden können.
  1. Vermeide blockierende Operationen:
    • await sollte nur verwendet werden, wenn es notwendig ist. Verwende es nicht in Schleifen, wenn die Operationen parallel ausgeführt werden können.

Zusammenfassung

  • async/await ist eine moderne und lesbare Art, mit asynchronem Code zu arbeiten.
  • Es basiert auf Promises und nutzt die Event Loop, um nicht-blockierende Ausführung zu ermöglichen.
  • Fehlerbehandlung und parallele Ausführung sind wichtige Aspekte, die bei der Arbeit mit asyncFunktionen und Promises berücksichtigt werden sollten.

Erklärung von async Funktion - Wie funktioniert es

Der gezeigte Code demonstriert verschiedene Ansätze, wie man mit asynchronen Daten in JavaScript arbeitet. Es wird gezeigt, wie man Daten simuliert, die erst nach einer Verzögerung verfügbar sind, und wie man mit diesen Daten arbeitet. Der Fokus liegt auf der Verwendung von Promises und deren Fehlerbehandlung.

Version 001: Synchrone Daten

function getData(){
    return [
        { producer: 'Audi', model: 'RS 3', fuel: 'petrol'},
        { producer: 'BMW', model: 'X5', fuel: 'diesel'},
        ...
    ]
}
  • Was passiert hier?
    • Die Funktion getData gibt sofort ein Array von Objekten zurück. Jedes Objekt repräsentiert ein Auto mit den Eigenschaften producer, model und fuel.
    • Problem: Diese Funktion ist synchron, d. h., die Daten sind sofort verfügbar. In der Realität (z. B. bei API-Aufrufen) sind Daten oft nicht sofort verfügbar, sondern benötigen Zeit, um geladen zu werden.

Version 002: Asynchrone Daten mit setTimeout

function getData(){
    let result = []

    setTimeout(() => {
        result = [
            { producer: 'Audi', model: 'RS 3', fuel: 'petrol'},
            ...
        ]
    }, 1000)
    return result
}
  • Was passiert hier?
    • Die Funktion getData verwendet setTimeout, um die Daten nach 1 Sekunde zu simulieren.
    • Problem: Die Funktion gibt das Array result sofort zurück, bevor setTimeout die Daten setzen kann. Das bedeutet, dass result immer leer bleibt, da die Funktion beendet wird, bevor die Daten verfügbar sind.
    • Lösung: Man muss asynchrone Daten anders behandeln, z. B. mit Callbacks oder Promises.

Version 003: Asynchrone Daten mit einem Callback

function getData(callback){
    let result = []

    setTimeout(() => {
        result = [
            { producer: 'Audi', model: 'RS 3', fuel: 'petrol'},
            ...
        ]
        callback(result)
    }, 1000)
}
  • Was passiert hier?
    • Die Funktion getData akzeptiert einen Callback als Parameter. Ein Callback ist eine Funktion, die aufgerufen wird, sobald die Daten verfügbar sind.
    • Nach 1 Sekunde wird callback(result) aufgerufen, und die Daten werden an die Callback-Funktion übergeben.
    • Vorteil: Die Daten werden erst verarbeitet, wenn sie verfügbar sind.
    • Nachteil: Callbacks können zu unübersichtlichem Code führen, wenn viele verschachtelte asynchrone Operationen nötig sind (sogenannte "Callback-Hölle").

Version 004: Asynchrone Daten mit einem Promise

function getData(){
    return new Promise((resolve, reject) => {
        let result = []
        setTimeout(() => {
            result = [
                { producer: 'Audi', model: 'RS 3', fuel: 'petrol'},
                ...
            ]
            resolve(result)
        }, 1000)
    })
}
  • Was passiert hier?
    • Die Funktion getData gibt ein Promise zurück.
    • Ein Promise ist ein Objekt, das einen zukünftigen Wert repräsentiert. Es hat zwei mögliche Zustände:
      • resolve: Die Operation war erfolgreich, und der Wert wird zurückgegeben.
      • reject: Die Operation ist fehlgeschlagen, und ein Fehler wird zurückgegeben.
    • Nach 1 Sekunde wird resolve(result) aufgerufen, um die Daten bereitzustellen.
    • Vorteil: Promises machen den Code lesbarer und einfacher zu handhaben als Callbacks.

Version 005: Promise mit Fehlerbehandlung

function getData(){
    return new Promise((resolve, reject) => {
        let result = []
        setTimeout(() => {
            if (Math.random() > 0.2){
                result = [
                    { producer: 'Audi', model: 'RS 3', fuel: 'petrol'},
                    ...
                ]
                resolve(result)
            } else {
                reject('Ich mag dein Gesicht net!')
            }
        }, 1000)
    })
}
  • Was passiert hier?
    • Die Funktion getData simuliert eine Situation, in der die Daten manchmal nicht verfügbar sind.
    • Mit einer Wahrscheinlichkeit von 80 % wird resolve(result) aufgerufen, um die Daten bereitzustellen.
    • Mit einer Wahrscheinlichkeit von 20 % wird reject('Ich mag dein Gesicht net!') aufgerufen, um einen Fehler zu simulieren.
    • Vorteil: Promises erlauben eine einfache Fehlerbehandlung mit .catch().

Verwendung von getData in Kombination mit Promises

Die Funktion getCarByFuelType soll Autos nach ihrem Kraftstofftyp filtern. Jede Version zeigt eine andere Herangehensweise, wie man mit den Daten arbeitet, die von der Funktion getData bereitgestellt werden. Hier ist eine detaillierte Erklärung der Versionen (ausgenommen Version 005).

Version 002: Es geht nicht

function getCarByFuelType(fuel){
    const data = getData()
    const result = data.filter(car => car.fuel === fuel)

    return result
}

Was passiert hier?

  1. Die Funktion getData wird aufgerufen, um die Daten zu erhalten.
  1. Die Rückgabe von getData wird in der Variablen data gespeichert.
  1. Es wird versucht, die Autos mit dem gewünschten Kraftstofftyp (fuel) zu filtern, indem die filterMethode aufgerufen wird.
  1. Das Ergebnis wird zurückgegeben.

Warum funktioniert das nicht?

  • Die Funktion getData in dieser Version arbeitet asynchron (z. B. mit setTimeout), gibt aber die Daten nicht sofort zurück. Stattdessen gibt sie ein leeres Array zurück, bevor die Daten verfügbar sind.
  • data ist also leer, wenn filter aufgerufen wird, und das Ergebnis ist immer ein leeres Array.

Version 003: Mit einem Callback

function getCarByFuelType(fuel){
    getData((data) => {
        const result = data.filter(car => car.fuel === fuel)
        console.log(result)
    })
}

Was passiert hier?

  1. Die Funktion getData wird aufgerufen und ein Callback (eine Funktion) wird übergeben.
  1. Sobald die Daten in getData verfügbar sind (nach der Verzögerung durch setTimeout), wird das Callback aufgerufen und die Daten (data) an das Callback übergeben.
  1. Innerhalb des Callbacks wird die filterMethode verwendet, um die Autos mit dem gewünschten Kraftstofftyp (fuel) zu filtern.
  1. Das Ergebnis wird in der Konsole ausgegeben.

Warum funktioniert das?

  • Die Funktion getData verwendet ein Callback, um sicherzustellen, dass der Code erst ausgeführt wird, wenn die Daten verfügbar sind.
  • Das Callback wird erst nach der Verzögerung (1 Sekunde) aufgerufen, wenn die Daten bereitstehen.

Nachteile:

  • Der Code wird unübersichtlich, wenn mehrere asynchrone Operationen verschachtelt werden (sogenannte "Callback-Hölle").
  • Es ist schwieriger, Fehler zu behandeln, da die Fehlerbehandlung manuell in jedem Callback implementiert werden muss.

Version 004: Mit einem Promise und .then

function getCarByFuelType(fuel){
    getData().then((data) => {
        const result = data.filter(car => car.fuel === fuel)
        console.log(result)
    })
}

Was passiert hier?

  1. Die Funktion getData wird aufgerufen. Sie gibt ein Promise zurück.
  1. Mit .then wird eine Funktion registriert, die ausgeführt wird, sobald das Promise aufgelöst wird (d. h., wenn die Daten verfügbar sind).
  1. Innerhalb der .thenFunktion wird die filterMethode verwendet, um die Autos mit dem gewünschten Kraftstofftyp (fuel) zu filtern.
  1. Das Ergebnis wird in der Konsole ausgegeben.

Warum funktioniert das?

  • Promises sind eine moderne und strukturierte Möglichkeit, mit asynchronem Code zu arbeiten.
  • Die .thenMethode stellt sicher, dass der Code erst ausgeführt wird, wenn die Daten verfügbar sind.

Vorteile gegenüber Callbacks:

  • Der Code ist lesbarer und weniger verschachtelt.
  • Fehler können einfacher behandelt werden, indem .catch hinzugefügt wird (wie in Version 005).

Version 005: Daten filtern mit Fehlerbehandlung

function getCarByFuelType(fuel){
    getData()
        .then(data => {
            const result = data.filter(car => car.fuel === fuel)
            console.log(result)
        })
        .catch(message => {
            console.log('etwas ist schief gelaufen', message)
        })
}

getCarByFuelType('diesel')
  • Was passiert hier?
    1. getData():
      • Ruft die Funktion getData auf, die ein Promise zurückgibt.
    1. .then(data => {...}):
      • Wenn das Promise erfolgreich aufgelöst wird (mit resolve), wird die Funktion im thenBlock ausgeführt. Die Daten (data) werden gefiltert, um nur Autos mit dem angegebenen fuelTyp zurückzugeben.
    1. .catch(message => {...}):
      • Wenn das Promise abgelehnt wird (mit reject), wird die Funktion im catchBlock ausgeführt. Der Fehler wird in der Konsole ausgegeben.
    1. Ergebnis:
      • Wenn alles gut geht, werden die gefilterten Autos in der Konsole ausgegeben.
      • Wenn ein Fehler auftritt, wird die Fehlermeldung ausgegeben.

Zusammenfassung

  1. Synchroner Code (Version 001):
    • Einfach, aber nicht geeignet für asynchrone Operationen.
  1. Asynchroner Code mit setTimeout (Version 002):
    • Funktioniert nicht, da die Daten nicht rechtzeitig zurückgegeben werden.
  1. Callbacks (Version 003):
    • Funktioniert, aber kann zu unübersichtlichem Code führen.
  1. Promises (Version 004 und 005):
    • Bieten eine saubere und lesbare Möglichkeit, mit asynchronen Daten zu arbeiten.
    • Fehler können einfach mit .catch() behandelt werden.
  1. Fehlerbehandlung:
    • Es ist wichtig, immer mit Fehlern zu rechnen, besonders bei asynchronen Operationen.

Warum Promises?

  • Promises sind eine moderne und flexible Lösung für asynchrone Operationen.
  • Sie machen den Code lesbarer und vermeiden die "Callback-Hölle".
  • Mit Promises kannst du sowohl erfolgreiche als auch fehlgeschlagene Operationen einfach behandeln.
