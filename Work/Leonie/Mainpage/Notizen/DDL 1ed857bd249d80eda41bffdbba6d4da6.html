Fragen zum Thema DDL (SQL):

1. Was ist eine Tabelle?

  • Eine Tabelle ist eine strukturierte Sammlung von Daten in einer relationalen Datenbank. Sie besteht aus Zeilen (Datensätzen) und Spalten (Felder), die die Struktur der Daten definieren.
  • Beispiel: Eine Tabelle Employees könnte Spalten wie Name, Age, Department und Salary enthalten.

2. Was ist eine Spalte?

  • Eine Spalte ist ein Attribut oder ein Feld in einer Tabelle, das eine bestimmte Art von Daten speichert.
  • Jede Spalte hat einen Namen und einen Datentyp, der definiert, welche Art von Daten in dieser Spalte gespeichert werden kann.
  • Beispiel: In der Tabelle Employees könnte die Spalte Name Textdaten (z. B. VARCHAR) speichern.

3. Welche Datentypen existieren in MySQL?

MySQL bietet verschiedene Datentypen, die in drei Hauptkategorien unterteilt sind:

  1. Numerische Datentypen:
    • INT, BIGINT, DECIMAL, FLOAT, DOUBLE, TINYINT, etc.
  1. Zeichen- und Textdatentypen:
    • CHAR, VARCHAR, TEXT, TINYTEXT, MEDIUMTEXT, LONGTEXT, etc.
  1. Datum- und Zeitdatentypen:
    • DATE, DATETIME, TIMESTAMP, TIME, YEAR.

4. Welche sind die wichtigsten 3-4 Datentypen?

Die wichtigsten Datentypen, die häufig verwendet werden, sind:

  1. INT: Für Ganzzahlen (z. B. IDs, Zähler).
  1. VARCHAR: Für Zeichenketten mit variabler Länge (z. B. Namen, E-Mails).
  1. DATE: Für Datumswerte (z. B. Geburtsdatum).
  1. DECIMAL: Für präzise Dezimalzahlen (z. B. Preise, Geldbeträge).

5. Was ist ein Primärschlüssel?

  • Ein Primärschlüssel ist eine eindeutige Kennung für jede Zeile in einer Tabelle.
  • Eigenschaften:
    • Er muss eindeutig sein (keine Duplikate).
    • Er darf keine NULL-Werte enthalten.
  • Beispiel: In einer Tabelle Employees könnte die Spalte EmployeeID der Primärschlüssel sein.

6. Wie erstelle ich eine Beziehung zwischen Tabellen?

  • Beziehungen zwischen Tabellen werden durch Primärschlüssel und Fremdschlüssel erstellt.
  • Ein Fremdschlüssel in einer Tabelle verweist auf den Primärschlüssel einer anderen Tabelle.
  • Beispiel:
    CREATE TABLE Departments (
    DepartmentID INT PRIMARY KEY,
    DepartmentName VARCHAR(100)
);

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    Name VARCHAR(100),
    DepartmentID INT,
    FOREIGN KEY (DepartmentID) REFERENCES Departments(DepartmentID)
);

7. Was ist ein Fremdschlüssel? Was bedeutet ein Fremdschlüssel zu haben?

  • Ein Fremdschlüssel ist eine Spalte, die auf den Primärschlüssel einer anderen Tabelle verweist.
  • Bedeutung:
    • Er stellt sicher, dass die referenzierten Daten in der anderen Tabelle existieren (Datenintegrität).
    • Beispiel: In der Tabelle Employees verweist DepartmentID auf DepartmentID in der Tabelle Departments.

8. Was ist die Normalisierung in SQL? Warum ist sie wichtig?

  • Normalisierung ist der Prozess, eine Datenbank in kleinere, logisch zusammenhängende Tabellen zu zerlegen, um Redundanz zu minimieren und Datenintegrität zu gewährleisten.
  • Wichtig, weil:
    • Sie reduziert Datenredundanz.
    • Sie verbessert die Konsistenz und Integrität der Daten.
    • Sie erleichtert die Wartung und das Verständnis der Datenbank.

9. Wie kann ich entscheiden, was in eine Tabelle kommt oder was separat angelegt werden muss?

  • Entscheidungskriterien:
    • Daten, die sich häufig ändern oder mehrfach vorkommen, sollten in separate Tabellen ausgelagert werden (z. B. Normalisierung).
    • Daten, die logisch zusammengehören, sollten in derselben Tabelle gespeichert werden.
    • Beispiel: Employees und Departments sollten getrennt sein, da ein Mitarbeiter zu einem bestimmten Department gehört, aber mehrere Mitarbeiter in einem Department sein können.

10. Was ist ein Index?

  • Ein Index ist eine Datenstruktur, die den Zugriff auf Daten in einer Tabelle beschleunigt.
  • Er funktioniert ähnlich wie ein Inhaltsverzeichnis in einem Buch.
  • Beispiel: Ein Index auf der Spalte EmployeeID in der Tabelle Employees ermöglicht eine schnellere Suche nach Mitarbeitern.

11. Wann soll ich einen Index verwenden / vermeiden?

  • Verwenden:
    • Für Spalten, die häufig in WHERE, JOIN, ORDER BY oder GROUP BY verwendet werden.
    • Für Primärschlüssel und Fremdschlüssel.
  • Vermeiden:
    • Für Spalten mit vielen Duplikaten (z. B. Gender mit den Werten M und F).
    • Wenn die Tabelle sehr klein ist, da der Overhead des Indexes die Vorteile überwiegen kann.

Fragen zum Thema Datenbank-Administration

1. Wo sind die DB-Benutzer gespeichert?

  • In MySQL werden die Benutzerinformationen in der Systemdatenbank mysql gespeichert, insbesondere in der Tabelle mysql.user.

2. Wie erstelle ich einen neuen Benutzer?

  • Ein neuer Benutzer wird mit dem Befehl CREATE USER erstellt:
    CREATE USER 'username'@'host' IDENTIFIED BY 'password';
  • Beispiel:
    CREATE USER 'john'@'localhost' IDENTIFIED BY 'securepassword';

3. Wo sind Serverrollen gespeichert?

  • In MySQL gibt es keine direkten "Serverrollen" wie in anderen Datenbanksystemen (z. B. SQL Server). Stattdessen werden Berechtigungen auf Benutzerbasis in der Tabelle mysql.user und verwandten Tabellen (z. B. mysql.db, mysql.tables_priv) gespeichert.

4. Wo sind Datenbank-Berechtigungen gespeichert?

  • Datenbank-Berechtigungen werden in den Systemtabellen der Datenbank mysql gespeichert:
    • mysql.user: Globale Berechtigungen für Benutzer.
    • mysql.db: Berechtigungen auf Datenbankebene.
    • mysql.tables_priv: Berechtigungen auf Tabellenebene.
    • mysql.columns_priv: Berechtigungen auf Spaltenebene.
