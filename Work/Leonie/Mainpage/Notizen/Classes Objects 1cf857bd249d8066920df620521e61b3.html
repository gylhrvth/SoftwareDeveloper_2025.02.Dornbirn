In JavaScript werden Classes verwendet, um Objekte mit einer gemeinsamen Struktur und Funktionalität zu erstellen. Sie sind eine moderne Syntax (eingeführt mit ES6), die das Erstellen und Verwalten von Objekten erleichtert und die objektorientierte Programmierung (OOP) unterstützt.

Warum braucht man Classes bei Objekten in JavaScript?

  1. Wiederverwendbarkeit:

    Classes ermöglichen es, eine Vorlage (Blueprint) für Objekte zu erstellen, die mehrfach verwendet werden kann. Dadurch wird der Code sauberer und wartbarer.

    Beispiel:

    class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log(`Hallo, mein Name ist ${this.name} und ich bin ${this.age} Jahre alt.`);
    }
}

const person1 = new Person("Alice", 30);
const person2 = new Person("Bob", 25);

person1.greet(); // Hallo, mein Name ist Alice und ich bin 30 Jahre alt.
person2.greet(); // Hallo, mein Name ist Bob und ich bin 25 Jahre alt.
  1. Kapselung (Encapsulation):

    Mit Classes kann man Daten und Methoden zusammenfassen und den Zugriff auf bestimmte Teile des Codes einschränken (z. B. mit privaten Feldern). Dies verbessert die Sicherheit und Modularität.

    Beispiel:

    class BankAccount {
    #balance; // privates Feld

    constructor(initialBalance) {
        this.#balance = initialBalance;
    }

    deposit(amount) {
        this.#balance += amount;
    }

    getBalance() {
        return this.#balance;
    }
}

const account = new BankAccount(100);
account.deposit(50);
console.log(account.getBalance()); // 150
// console.log(account.#balance); // Fehler: Zugriff auf privates Feld
  1. Vererbung (Inheritance):

    Classes ermöglichen es, Eigenschaften und Methoden von einer anderen Klasse zu erben. Dies reduziert Redundanz und fördert die Wiederverwendbarkeit.

    Beispiel:

    class Animal {
    constructor(name) {
        this.name = name;
    }

    speak() {
        console.log(`${this.name} macht ein Geräusch.`);
    }
}

class Dog extends Animal {
    speak() {
        console.log(`${this.name} bellt.`);
    }
}

const dog = new Dog("Rex");
dog.speak(); // Rex bellt.
  1. Lesbarkeit und Struktur:

    Die Class-Syntax ist klarer und strukturierter als ältere Methoden wie das Erstellen von Objekten mit Konstruktorfunktionen oder Object.create. Dies macht den Code leichter verständlich.

    Beispiel (Vergleich mit Konstruktorfunktion):

    // Mit Class
class Car {
    constructor(brand) {
        this.brand = brand;
    }

    drive() {
        console.log(`${this.brand} fährt.`);
    }
}

// Mit Konstruktorfunktion
function CarOld(brand) {
    this.brand = brand;
}
CarOld.prototype.drive = function() {
    console.log(`${this.brand} fährt.`);
};
  1. Standardisierung:

    Die Class-Syntax ist Teil des ES6-Standards und wird von modernen JavaScript-Entwicklern bevorzugt. Sie bietet eine einheitliche Möglichkeit, objektorientierten Code zu schreiben.

Zusammenfassung

Classes in JavaScript sind nützlich, um:

  • Objekte mit einer gemeinsamen Struktur und Funktionalität zu erstellen.
  • Wiederverwendbaren und modularen Code zu schreiben.
  • Vererbung und Kapselung zu nutzen.
  • Den Code lesbarer und wartbarer zu machen.

Obwohl Classes nur syntaktischer Zucker für Prototypen sind (da JavaScript prototypenbasiert ist), machen sie die Arbeit mit Objekten und OOP deutlich einfacher und intuitiver.
