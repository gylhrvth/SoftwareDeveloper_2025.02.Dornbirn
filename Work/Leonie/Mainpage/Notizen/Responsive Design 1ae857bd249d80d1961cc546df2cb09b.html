Responsive Design

Für ein responsives Design ist es wichtig, dass die Webseite auf verschiedenen Geräten und Bildschirmgrößen gut aussieht und funktioniert. Hier sind einige wichtige Konzepte, Größenangaben, Variablen und Funktionen, die du kennen solltest:

Wichtige Konzepte

  1. Fluid Layouts: Verwende relative Einheiten wie Prozent (%) und Viewport-Einheiten (vw, vh), um Layouts zu erstellen, die sich an die Bildschirmgröße anpassen.
  1. Media Queries: Verwende Media Queries, um CSS-Regeln basierend auf der Bildschirmgröße oder anderen Eigenschaften anzuwenden.
  1. Flexible Images: Stelle sicher, dass Bilder und andere Medien sich an die Größe ihres Containers anpassen.
  1. Responsive Typography: Verwende relative Einheiten wie em und rem für Schriftgrößen, um sicherzustellen, dass die Typografie auf verschiedenen Geräten gut aussieht.
  1. Mobile-First Design: Entwickle zuerst für mobile Geräte und füge dann Styles für größere Bildschirme hinzu.

Größenangaben

Absolute Einheiten

  • px (Pixel): Eine feste Einheit, die unabhängig von der Bildschirmgröße ist.
    .element {
    width: 200px;
}

Relative Einheiten

  • % (Prozent): Eine relative Einheit, die sich auf die Größe des übergeordneten Elements bezieht.
    .element {
    width: 50%;
}
  • em: Eine relative Einheit, die sich auf die Schriftgröße des übergeordneten Elements bezieht.
    .element {
    font-size: 2em; /* 2 mal die Schriftgröße des übergeordneten Elements */
}
  • rem: Eine relative Einheit, die sich auf die Schriftgröße des Root-Elements (html) bezieht.
    .element {
    font-size: 1.5rem; /* 1.5 mal die Schriftgröße des Root-Elements */
}

Flexible Einheiten

  • vw (Viewport Width): 1vw entspricht 1% der Breite des Viewports.
    .element {
    width: 50vw; /* 50% der Breite des Viewports */
}
  • vh (Viewport Height): 1vh entspricht 1% der Höhe des Viewports.
    .element {
    height: 50vh; /* 50% der Höhe des Viewports */
}
  • vmin: 1vmin entspricht 1% der kleineren Dimension des Viewports (Breite oder Höhe).
    .element {
    font-size: 5vmin; /* 5% der kleineren Dimension des Viewports */
}
  • vmax: 1vmax entspricht 1% der größeren Dimension des Viewports (Breite oder Höhe).

    .element {
    font-size: 5vmax; /* 5% der größeren Dimension des Viewports */
}

Variablen und Funktionen

CSS-Variablen (Custom Properties)

CSS-Variablen ermöglichen es dir, wiederverwendbare Werte zu definieren, die du in deinem CSS verwenden kannst.

:root {
    --main-color: #3498db;
    --padding: 10px;
}

.element {
    color: var(--main-color);
    padding: var(--padding);
}

CSS-Funktionen

  • calc(): Ermöglicht Berechnungen innerhalb von CSS.
    .element {
    width: calc(100% - 50px);
}
  • clamp(): Ermöglicht es, eine Größe innerhalb eines bestimmten Bereichs festzulegen.
    .element {
    font-size: clamp(1rem, 2.5vw, 2rem);
}

Media Queries

Media Queries ermöglichen es dir, CSS-Regeln basierend auf der Bildschirmgröße oder anderen Eigenschaften anzuwenden.

/* Mobile-First Ansatz */
.element {
    font-size: 1rem;
}

@media (min-width: 600px) {
    .element {
        font-size: 1.5rem;
    }
}

@media (min-width: 1200px) {
    .element {
        font-size: 2rem;
    }
}

Beispiel für ein responsives Design

Hier ist ein einfaches Beispiel, das all diese Konzepte kombiniert:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Responsive Design Example</title>
    <style>
        :root {
            --main-color: #3498db;
            --padding: 10px;
        }

        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
        }

        .container {
            width: 90%;
            max-width: 1200px;
            margin: 0 auto;
            padding: var(--padding);
        }

        .header {
            background-color: var(--main-color);
            color: white;
            padding: var(--padding);
            text-align: center;
        }

        .content {
            display: flex;
            flex-wrap: wrap;
            gap: var(--padding);
        }

        .content .box {
            flex: 1 1 calc(50% - var(--padding));
            background-color: lightgray;
            padding: var(--padding);
            box-sizing: border-box;
        }

        @media (min-width: 600px) {
            .content .box {
                flex: 1 1 calc(33.33% - var(--padding));
            }
        }

        @media (min-width: 900px) {
            .content .box {
                flex: 1 1 calc(25% - var(--padding));
            }
        }
    </style>
</head>
<body>
    <div class="container">
        <div class="header">
            <h1>Responsive Design Example</h1>
        </div>
        <div class="content">
            <div class="box">Box 1</div>
            <div class="box">Box 2</div>
            <div class="box">Box 3</div>
            <div class="box">Box 4</div>
        </div>
    </div>
</body>
</html>

In diesem Beispiel wird das Layout der Boxen in der .content-Klasse basierend auf der Bildschirmgröße angepasst, um ein responsives Design zu gewährleisten.

  1. Verwende KEINE großen Elemente mit fester Breite - Wenn zum Beispiel ein Bild breiter als der Viewport ist, führt dies zu horizontalem Scrollen. Denk daran, diese Inhalte so anzupassen, dass sie in die Breite des Viewports passen.
  1. Der Inhalt darf NICHT von einer bestimmten Breite abhängig sein - Da die Bildschirmmaße und Breiten in CSS-Pixeln zwischen den Geräten stark variieren, sollte sich der Inhalt nicht auf eine bestimmte Viewport-Breite verlassen, um gut dargestellt zu werden.
  1. Nutze CSS Media Queries für unterschiedliche Darstellungen auf kleinen und großen Bildschirmen - Das Festlegen von großen absoluten CSS-Breiten für Seitenelemente führt dazu, dass die Elemente für kleinere Geräte zu breit sind. Verwende stattdessen relative Breitenangaben, wie zum Beispiel width: 100%. Sei auch vorsichtig bei der Verwendung großer absoluter Positionierungswerte. Dies kann dazu führen, dass Elemente auf kleinen Geräten außerhalb des Viewports landen.
