Was ist Express.js?

Express.js ist ein minimalistisches und flexibles Web-Framework für Node.js, das die Entwicklung von Webanwendungen und APIs erleichtert. Es bietet eine einfache Möglichkeit, HTTP-Server zu erstellen, Routen zu definieren und Middleware zu verwenden.

Warum Express.js verwenden?

  1. Einfachheit und Flexibilität:
    • Express ist leichtgewichtig und bietet nur die grundlegenden Funktionen, die du für die Entwicklung benötigst.
    • Du kannst es mit zusätzlichen Paketen erweitern, um spezifische Anforderungen zu erfüllen.
  1. Schnelle Entwicklung:
    • Express abstrahiert viele komplexe Aufgaben, wie das Verarbeiten von HTTP-Anfragen, das Routing und das Bereitstellen von statischen Dateien.
  1. Große Community:
    • Es gibt eine riesige Community und viele Plugins, die dir helfen, deine Anwendung schneller zu entwickeln.
  1. Middleware-Unterstützung:
    • Middleware sind Funktionen, die zwischen der Anfrage (Request) und der Antwort (Response) ausgeführt werden. Sie ermöglichen es dir, Anfragen zu verarbeiten, zu validieren oder zu manipulieren.

Wichtige Konzepte in Express.js

1. Middleware

Middleware sind Funktionen, die Zugriff auf das Anfrage-Objekt (req), das Antwort-Objekt (res) und die nächste Middleware-Funktion (next) haben.

  • Beispiel:
    app.use((req, res, next) => {
    console.log(`Anfrage: ${req.method} ${req.url}`);
    next(); // Geht zur nächsten Middleware oder Route
});
  • Arten von Middleware:
    • Anwendungs-Middleware: Wird auf die gesamte Anwendung angewendet.
    • Router-Middleware: Wird nur auf bestimmte Routen angewendet.
    • Fehler-Middleware: Behandelt Fehler in der Anwendung.

2. Routing

Routing ist der Prozess, bei dem HTTP-Anfragen (z. B. GET, POST) an spezifische Funktionen weitergeleitet werden.

  • Beispiel:
    app.get('/', (req, res) => {
    res.send('Willkommen auf der Startseite!');
});

app.post('/person', (req, res) => {
    res.send('Person hinzugefügt!');
});
  • Routen-Parameter:
    app.get('/user/:id', (req, res) => {
    const userId = req.params.id;
    res.send(`Benutzer-ID: ${userId}`);
});

3. Statische Dateien

Express kann statische Dateien wie CSS, Bilder oder JavaScript bereitstellen.

  • Beispiel:
    Wenn du eine Datei
    public/style.css hast, kannst du sie über http://localhost:3000/style.css aufrufen.
    app.use(express.static('public'));

4. Template-Engines

Express unterstützt Template-Engines wie EJS, Pug oder Handlebars, um dynamische HTML-Seiten zu erstellen.

  • Beispiel mit EJS:
    app.set('view engine', 'ejs');

app.get('/', (req, res) => {
    res.render('index', { title: 'Startseite' });
});

5. Body-Parser

Express kann Daten aus dem Body einer Anfrage verarbeiten (z. B. Formulardaten oder JSON).

  • Middleware für Body-Parsing:
    app.use(express.urlencoded({ extended: true })); // Für Formulardaten
app.use(express.json()); // Für JSON-Daten

6. Fehlerbehandlung

Express bietet eine einfache Möglichkeit, Fehler zu behandeln.

  • Beispiel:
    app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Etwas ist schiefgelaufen!');
});

Wichtige Funktionen in Express.js

FunktionBeschreibung
app.get()Definiert eine Route für GET-Anfragen.
app.post()Definiert eine Route für POST-Anfragen.
app.use()Fügt Middleware hinzu.
app.listen()Startet den Server und lauscht auf einem bestimmten Port.
req.paramsEnthält die Parameter aus der URL (z. B. /user/:id).
req.queryEnthält die Query-Parameter aus der URL (z. B. ?name=Max).
req.bodyEnthält die Daten aus dem Body der Anfrage (z. B. bei POST).
res.send()Sendet eine einfache Antwort an den Client.
res.render()Rendert eine HTML-Seite mit einer Template-Engine.
res.json()Sendet eine JSON-Antwort an den Client.

Beispiel: Eine einfache Express-Anwendung

const express = require('express');
const app = express();

// Middleware
app.use(express.json());

// Startseite
app.get('/', (req, res) => {
    res.send('Willkommen auf der Startseite!');
});

// Benutzer hinzufügen
app.post('/user', (req, res) => {
    const { name, age } = req.body;
    res.json({ message: `Benutzer ${name} (${age} Jahre alt) hinzugefügt!` });
});

// Fehlerbehandlung
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Etwas ist schiefgelaufen!');
});

// Server starten
const PORT = 3000;
app.listen(PORT, () => {
    console.log(`Server läuft auf <http://localhost>:${PORT}`);
});

Wichtige Pakete für Express.js

  1. body-parser: Verarbeitet Formulardaten und JSON (heute in Express integriert).
  1. morgan: Protokolliert HTTP-Anfragen.
  1. cors: Aktiviert Cross-Origin Resource Sharing.
  1. dotenv: Verwendet Umgebungsvariablen aus einer .envDatei.
  1. express-validator: Validiert und bereinigt Benutzereingaben.

Zusammenfassung

  • Express.js ist ein leichtgewichtiges Framework für Node.js, das die Entwicklung von Webanwendungen und APIs erleichtert.
  • Es bietet Funktionen wie Routing, Middleware, Template-Engines und Fehlerbehandlung.
  • Es ist flexibel und kann mit zusätzlichen Paketen erweitert werden.
  • Es ist ideal für kleine bis große Projekte und hat eine große Community.

Was ist der Unterschied zwischen req.body und req.query

Der Unterschied zwischen req.body und req.query liegt in der Art und Weise, wie die Daten vom Client an den Server gesendet werden:

1. req.body

  • Definition: Enthält die Daten, die im Body (Nachrichtentext) einer HTTP-Anfrage gesendet werden.
  • Verwendung: Wird hauptsächlich bei POST, PUT oder PATCHAnfragen verwendet.
  • Datenquelle: Die Daten werden typischerweise durch ein Formular oder eine API-Anfrage im Body der Anfrage gesendet.
  • Middleware erforderlich: Damit req.body funktioniert, muss die Middleware express.urlencoded() oder express.json() in der server.js aktiviert sein:
    app.use(express.urlencoded({ extended: true }));
app.use(express.json());

Beispiel:

Ein Formular sendet Daten per POST:

<form action="/person" method="POST">
  <input type="text" name="vorname" value="Max">
  <input type="text" name="nachname" value="Mustermann">
  <button type="submit">Absenden</button>
</form>

In der Route:

app.post('/person', (req, res) => {
    console.log(req.body); // { vorname: 'Max', nachname: 'Mustermann' }
});

2. req.query

  • Definition: Enthält die Daten, die als Query-Parameter in der URL gesendet werden.
  • Verwendung: Wird hauptsächlich bei GETAnfragen verwendet.
  • Datenquelle: Die Daten werden in der URL als Schlüssel-Wert-Paare angehängt.
  • Kein Body erforderlich: req.query funktioniert ohne zusätzliche Middleware.

Beispiel:

Ein Link oder eine URL mit Query-Parametern:

<http://localhost:3000/search?vorname=Max&nachname=Mustermann>

In der Route:

app.get('/search', (req, res) => {
    console.log(req.query); // { vorname: 'Max', nachname: 'Mustermann' }
});

Vergleich:

Eigenschaftreq.bodyreq.query
DatenquelleBody der AnfrageQuery-Parameter in der URL
HTTP-MethodenPOST, PUT, PATCHGET

Wann verwende ich was?

  • req.body: Wenn du sensible oder größere Datenmengen senden möchtest (z. B. Formulardaten, JSON-Daten).
  • req.query: Wenn du einfache Filter- oder Suchparameter übergeben möchtest (z. B. ?name=Max).
