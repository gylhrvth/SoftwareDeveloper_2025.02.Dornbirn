    CSS Größenangabe

  • Was ist der Unterschied zwischen absoluten und relativen Größenangaben?

    • Der Unterschied zwischen absoluten und relativen Größenangaben liegt in ihrer Abhängigkeit vom Kontext:

    • Absolute Größenangaben

      • Sind fest und unabhängig von anderen Elementen.

      • Beispiele: px (Pixel), cm (Zentimeter), mm (Millimeter), in (Inch).

      • Beispiel: width: 200px; bedeutet immer genau 200 Pixel, egal in welchem Kontext.

    • Relative Größenangaben

      • Passen sich an andere Werte oder die Umgebung an.

      • Werte: % (Prozent, relativ zum Elternelement), em (relativ zur Schriftgröße des Elternelements), rem (relativ zur Schriftgröße des Wurzelelements), vw (relativ zur Breite des Viewports), vh (relativ zur Höhe des Viewports).

      • Beispiel: width: 50%; bedeutet, dass die Breite die Hälfte des übergeordneten Elements beträgt.

    Absolute Werte sind nützlich für feste Layouts, wie Drucklayouts oder wenn präzise Größen wichtig sind.

    Relative Werte sind ideal für responsive Designs, da sie sich an unterschiedliche Bildschirmgrößen anpassen können.


  • Beispiel:Interaktives Beispiel (Absolute vs Relative widths)

  • Was gibt es für absoluten Großenangaben?

    • Typografische Maßeinheiten:

      • Pixel (px) → Ein einzelner Punkt auf einem Bildschirm
      • Punkt (pt) → 1 pt = 1/72 Zoll (wird oft für Schriftgrößen genutzt)
      • Pica (pc) → 1 pc = 12 pt
      • Ex (ex) → Basierend auf der x-Höhe einer Schriftart

    • Physische Maßeinheiten

      • Millimeter (mm)
      • Zentimeter (cm)
      • Zoll (in oder “) → 1 Zoll = 2,54 cm
      • Meter (m)

      In der Webentwicklung werden meist px, pt, cm und mm für absolute Größenangaben verwendet.

  • Was gibt es für relative Großenangaben?

    • Schriftbezogene Einheiten:

      • em → Basierend auf der Schriftgröße des Elements (z. B. 2em ist das Doppelte der aktuellen Schriftgröße).
      • rem (Root em) → Basierend auf der Schriftgröße des html-Elements (z. B. 1rem entspricht der font-size von html).
      • ex → Basierend auf der Höhe des Buchstabens “x” in der aktuellen Schriftart.
      • ch → Basierend auf der Breite der Ziffer “0” in der aktuellen Schriftart.

    • Schriftbezogene Einheiten:

      • % → Bezieht sich auf die Größe des Elternelements (z. B. width: 50% bedeutet die Hälfte der Breite des übergeordneten Containers).

    • Viewport-basierte Einheiten (für responsive Designs)

      • vw (Viewport Width) → 1vw = 1% der Breite des Viewports (z. B. 50vw ist die Hälfte der Bildschirmbreite).
      • vh (Viewport Height) → 1vh = 1% der Höhe des Viewports.
      • vmin → 1% der kleineren Dimension des Viewports (Breite oder Höhe).
      • vmax → 1% der größeren Dimension des Viewports.

  • Was gibt es für flexiblen Großenangaben?

    • Flexbox- und Grid-Einheiten:

      • fr (Fractional Unit) → Wird im CSS Grid verwendet (1fr bedeutet “ein flexibler Anteil” des verfügbaren Platzes).
      • auto → Passt sich automatisch an den Inhalt an.

    • Skalierbare Maße mit CSS-Funktionen:

      • calc() → Ermöglicht Berechnungen mit verschiedenen Einheiten (width: calc(100% - 50px)).
      • min() / max() → Setzt minimale oder maximale Werte (width: min(50vw, 600px)).
      • clamp() → Kombiniert min/max (font-size: clamp(1rem, 2vw, 2rem)).

  • Was ist der Unterschied zwischen Physikalischer Pixel und CSS-Pixel?

    • Der Begriff Pixel kann in zwei unterschiedlichen Kontexten verwendet werden:

    • CSS-Pixel (logischer Pixel):

      • Definiert eine einheitliche Maßeinheit, die unabhängig von der Bildschirmauflösung ist.
      • Wird in Webdesign und Programmierung verwendet.
      • Beispiel: width: 100px; bedeutet, dass ein Element genau 100 CSS-Pixel breit ist, unabhängig von der tatsächlichen Pixeldichte des Geräts.

      -> CSS-Pixel ist eine abstrakte Einheit für Layouts, die auf verschiedenen Displays gleich groß erscheint.

    • Physikalischer Pixel (Hardware-Pixel):

      • Bezieht sich auf die kleinste darstellbare Einheit eines Bildschirms.
      • Moderne hochauflösende Displays (z. B. Retina-Displays) haben oft mehrere physische Pixel pro CSS-Pixel.
      • Beispiel: Ein Retina-Display kann 2× oder 3× so viele physikalische Pixel pro CSS-Pixel haben, um schärfere Darstellungen zu ermöglichen.

      -> Physikalischer Pixel ist die kleinste Leuchteinheit auf einem Bildschirm, deren Anzahl je nach Gerät variieren kann.

    Beispiel: Ein 100px breites Element bleibt auf jedem Gerät 100 CSS-Pixel breit, aber auf einem Retina-Display mit 2-facher Pixeldichte wird es mit 200 physikalischen Pixeln dargestellt.

    CSS Variablen / Funktionen

  • Was sind CSS Variablen?

    • CSS-Variablen, auch Custom Properties genannt, sind wiederverwendbare Werte in CSS. Sie ermöglichen eine dynamische Gestaltung und erleichtern das Ändern von Stilen an mehreren Stellen gleichzeitig.

  • CSS Variablen: Syntax und Verwendung

    1. Deklaration einer Variable: CSS-Variablen werden mit -- vor dem Namen definiert und meistens im :root-Selektor gespeichert.

      2. :root { --primary-color: #3498db; --font-size: 16px; }

    2. Nutzung einer Variable: Variablen werden mit der Funktion var() aufgerufen.

      3. body { background-color: var(--primary-color); font-size: var(--font-size); }

  • Was sind CSS Funktionen?

    • CSS-Funktionen sind spezielle Befehle, die innerhalb von CSS-Eigenschaften verwendet werden, um dynamische Werte zu generieren. Sie ermöglichen Berechnungen, Farbmanipulationen, Transformationen und vieles mehr.

  • Typische CSS Funktionen

    • Mathematische Funktionen:

      • calc() – Ermöglicht Berechnungen mit verschiedenen Einheiten.
      • min() und max() – Legt den kleineren oder größeren Wert aus mehreren Optionen fest.
      • clamp() – Kombiniert min, max und einen bevorzugten Wert.
        Struktur: clamp(MIN, IDEAL, MAX);

    • Farbmanipulation:

      • rgb() und rgba() – Definiert Farben mit Rot, Grün, Blau (+ Alpha für Transparenz)
      • hsl() und hsla() – Farben mit Farbton, Sättigung und Helligkeit
      • var() – Verwendet CSS-Variablen

    • Text- und Anzeige-Funktionen:

      • attr() – Holt den Wert eines HTML-Attributs
      • url() – Fügt eine Bild- oder Datei-URL ein
      • local() – Definiert eine Schriftart lokal oder aus dem Web

    • Transformationen & Effekte:

      • translate() – Verschiebt ein Element
      • rotate() – Dreht ein Element
      • scale() – Skaliert ein Element
      • skew() – verzerrt (schrägt) ein Element entlang der X- und/oder Y-Achse

  • Wie ruft man CSS Funktionen?

    • CSS-Funktionen werden direkt innerhalb von CSS-Eigenschaften aufgerufen, um dynamische Werte zu erzeugen. Der Aufruf erfolgt in der Regel durch die Angabe des Funktionsnamens gefolgt von runden Klammern, in denen die entsprechenden Parameter übergeben werden.

    property: function(parameter);

    Beispiele:

    -> calc() – Mathematische Berechnungen:

    div { width: calc(100% - 20px); /* Berechnet die Breite als 100% minus 20px */ }

    -> rgb() / rgba() – Farbdefinition:

    div { background-color: rgb(255, 0, 0); /* Rote Farbe */ color: rgba(0, 0, 255, 0.5); /* Halbtransparentes Blau */ }

    -> clamp() – Werte mit Mindest- und Höchstgrenzen:

    div { font-size: clamp(16px, 4vw, 24px); /* Schriftgröße wächst mit dem Viewport, bleibt aber zwischen 16px und 24px */ }

    -> translate() – Verschiebung eines Elements:

    div { transform: translate(50px, 100px); /* Verschiebt das Element um 50px nach rechts und 100px nach unten */ }

    CSS Responsiveness

  • Was bedeutet Responsive Design?

    • Responsive Design bezeichnet eine Designtechnik, bei der Webseiten so gestaltet werden, dass sie sich automatisch an die Bildschirmgröße und -auflösung des Geräts anpassen, auf dem sie angezeigt werden. Ziel ist es, eine benutzerfreundliche, ansprechende und gut lesbare Oberfläche auf allen Geräten zu bieten, sei es auf Desktops, Tablets oder Smartphones.

  • Warum verwendet man "Mobile First Approach"?

    • Der Mobile-First-Ansatz sorgt dafür, dass Websites optimal auf mobilen Geräten funktionieren, indem sie zuerst für die kleineren Bildschirme entwickelt und dann für größere Geräte erweitert werden. Dies verbessert nicht nur die Benutzererfahrung, sondern auch die Leistung und die Suchmaschinenoptimierung (SEO). In einer Welt, in der der mobile Zugriff auf das Internet dominiert, ist dieser Ansatz heute eine bewährte Praxis in der Webentwicklung.

    Google berücksichtigt Mobile-First-Indexierung, was bedeutet, dass die mobile Version einer Webseite bei der Suche höher bewertet wird.

  • Was ist ein Viewport? Warum ist Viewport wichtig?

    • Viewport ist der sichtbare Bereich im Browser oder auf einem Gerät, in dem Inhalte angezeigt werden.

    • Der Viewport ist entscheidend für die Gestaltung von Webseiten, insbesondere in einem Responsive Design.

    • Mit dem richtigen Umgang mit dem Viewport können Entwickler sicherstellen, dass ihre Webseite auf allen Geräten (von Smartphones bis Desktops) benutzerfreundlich und funktional bleibt.

    • Ein richtig definierter Viewport stellt sicher, dass eine mobile Webseite korrekt angezeigt wird und daher besser für Suchmaschinen optimiert ist.

  • Was ist eine Media Query?

    • Eine Media Query (Medienabfrage) ist eine CSS-Technik, mit der man das Design einer Webseite an verschiedene Bildschirmgrößen, Geräteauflösungen und andere Eigenschaften des Ausgabegeräts anpassen kann. Sie ermöglicht es, unterschiedliche CSS-Regeln basierend auf bestimmten Bedingungen anzuwenden, sodass eine Webseite auf verschiedenen Geräten wie Desktops, Tablets und Smartphones gut aussieht und funktioniert.

  • Aufbau einer Media Query:

    • @media (Bedingung) { /* CSS-Regeln */ }

    • @media: Der Befehl, um eine Medienabfrage zu starten.

    • Bedingung: Bestimmt unter welchen Umständen die CSS-Regeln angewendet werden sollen (z. B. Bildschirmbreite, Orientierung, Auflösung).

    • CSS-Regeln: Die Stilregeln, die angewendet werden, wenn die Bedingung erfüllt ist.

  • Media Query Beispiel:

    • /* Standard-Design für Desktops */ body { font-size: 16px; } /* Design für Bildschirme mit einer maximalen Breite von 768px (z. B. Tablets und Smartphones) */ @media (max-width: 768px) { body { font-size: 14px; } }

  • Typische Anwendungsfälle von Media Queries:

    • Bildschirmgrößen: Passen die Darstellung der Webseite an unterschiedliche Geräte an (Smartphones, Tablets, Desktops). -> Beispiel: @media (max-width: 768px)

    • Ausrichtung: Reagieren auf Hoch- oder Querformat. -> Beispiel: @media (orientation: landscape)

    • Auflösung: Für hochauflösende Displays wie Retina-Displays. -> Beispiel: @media (min-resolution: 2dppx)

    • Farben und Kontraste: Für barrierefreie Gestaltung oder Geräte mit besonderen Anzeigeanforderungen. -> Beispiel: @media (prefers-color-scheme: dark)

  • Kann man mehr als eine Media Query schreiben?

    • Ja, das ist sogar eine sehr gängige Praxis, um sicherzustellen, dass deine Webseite auf verschiedenen Geräten optimal aussieht und funktioniert.

CSS Transitions & Animations

    CSS Transform

  • Was ist CSS Transform?

    • Es ist eine CSS-Eigenschaft, die es ermöglicht, HTML-Elemente visuell zu verändern, ohne ihre tatsächliche Position im Dokumentenfluss zu ändern. Sie wird verwendet, um Elemente zu drehen, skalieren, verschieben oder kippen.

  • Welche Transform Eigenschaften sind üblich und wie verwendet man sie?

    • scale (x,y): Vergrößert oder verkleinert das Element.

      • div { transform: scale(1.5, 2); }

    • translate (x,y): Verschiebt ein Element entlang der X- und Y-Achse.

      • div { transform: translate(50px, 100px); }

    • rotate(angle): Dreht das Element im Uhrzeigersinn um den angegebenen Winkel.

      • div { transform: rotate(45deg); }

    • skew(x-angle, y-angle): Kippt das Element entlang der X- oder Y-Achse.

      • div { transform: skew(20deg, 10deg); }

    CSS Transitions

  • Was sind CSS Transitions?

    • CSS Transitions sind eine Möglichkeit, Änderungen von CSS-Eigenschaften sanft und animiert ablaufen zu lassen, anstatt dass sie sofort und abrupt passieren. Damit kann man z. B. eine Farbänderung, Größenveränderung oder Positionierung flüssig gestalten.

  • Wie kann man CSS Transitions verwenden?

    • Syntax in CSS:

    transition: property duration timing-function delay;

    • property → Die zu animierende Eigenschaft (z. B. background-color, opacity, transform).

    • duration → Wie lange die Animation dauert (z. B. 0.5s, 1s).

    • timing-function (optional) → Der Animationsverlauf (ease, linear, ease-in-out, etc.).

    • delay (optional) → Verzögerung vor Start der Animation.


  • Transition-Beispiele:

    • Farbänderung mit Hover

    • Größe & Drehung ändern

    • Position & Größe & Form ändern


  • Was ist die timing-function?

    • Die timing-function in CSS bestimmt, wie eine Transition oder Animation im Laufe der Zeit abläuft. Sie steuert die Geschwindigkeit der Änderung, indem sie festlegt, ob sie gleichmäßig, langsam oder in einem bestimmten Muster erfolgt.

    -> Die wichtigsten timing-function-Werte:

    • linear: Die Animation verläuft gleichmäßig, ohne Beschleunigung oder Verzögerung.

    • ease (Standard): Die Animation startet langsam, wird schneller und endet langsam.

    • ease-in: Langsamer Start, dann schneller.

    • ease-out: Schneller Start, dann langsamer.

    • ease-in-out: Langsamer Start und langsames Ende.

    • CSS Animations

  • Was sind CSS animations?

    • Mit CSS-Animationen kann man Elemente ohne JavaScript beweglich und dynamisch machen. Im Gegensatz zu Transitions sind Animationen komplexer und können mehrere Schritte enthalten.

  • Grundlegende Syntax
    • selector { animation: name duration timing-function delay iteration-count direction fill-mode; }

  • Erklärung der Syntax

    • name: Name der Animation. Beispiel: move

    • duration: Dauer der Animation. Beispiel: 2s

    • timing-function: Geschwindigkeitseffekt. Beispiel: ease-in-out

    • delay: Wartezeit vor Start. Beispiel: 1s

    • iteration-count: Anzahl der Wiederholungen. Beispiel: infinite, 3, 1

    • direction: Richtung der Animation. Beispiel: alternate, reverse

    • fill-mode: Verhalten nach Animation. Beispiel: forwards, backwards


  • Was sind @keyframes?

    • @keyframes ist die zentrale Technik für CSS-Animationen.

    Damit kann man festlegen, wie sich ein Element über die Zeit verändert – zum Beispiel Farbe, Position, Größe oder Drehung.

    Syntax on CSS:

    @keyframes animationName { 0% { /* Starting state */ } 50% { /* Midway state */ } 100% { /* Final state */ } }

    • 0% represents the starting point of the animation.

    • 100% represents the end point.

    • You can also define multiple percentage points (e.g., 25%, 75%) for more detailed control over the animation’s progression.

CSS Layout

  • Grid Eigenschaften:

    • Grid ist ein Zweidimensionales Layout – Es erlaubt sowohl die Steuerung von Zeilen (rows) als auch Spalten (columns), im Gegensatz zu Flexbox, das nur in eine Richtung arbeitet.

    Die wichtigsten Eigenschaften von CSS Grid sind:

    • Man erstellt ein Grid Container mit Display: Grid

    • Man definiert das Grid mit grid-template-columns und grid-template-rows.

    • Grid unterstützt absolute (px), relative (%, vw, vh), fraktionale Einheiten (fr) und automatische Größen (auto).

    • Mit gap, row-gap und column-gap lassen sich Abstände zwischen den Grid-Elementen steuern.

    • Man kann Elemente mit grid-column und grid-row über mehrere Spalten oder Zeilen hinweg platzieren.

    • Mit grid-area kann man grid-column und grid-row kombinieren. So kann man Elemente in einem bestimmten Bereich positionieren.

      • -> Grid-area kombiniert grid-row-start, grid-column-start, grid-row-end und grid-column-end in einer einzigen Zeile.

      .item { grid-area: 2 / 1 / 4 / 3; }

      -> Das ist dasselbe wie:
      .item { grid-row-start: 2; grid-column-start: 1; grid-row-end: 4; grid-column-end: 3; }

    • Für die gesamte Grid-Ausrichtung verwendet man: justify-content / align-content / place-content

    • Für die Ausrichtung innerhalb von Zellen verwendet man: justify-items / align-items / place-items

    • Mit Grid-Template-Areas kann man Namen für verschiedene Bereiche definieren und sie gezielt belegen.

    • Nested Grids: Ein Grid-Element kann selbst wieder ein Grid enthalten.

    • Responsives Design: Grid eignet sich perfekt für responsive Layouts, insbesondere mit minmax() und auto-fit / auto-fill.


  • Grid erstellen: Beispiel

    1. HTML-Struktur erstellen:

      2. <!DOCTYPE html> <html lang="de"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>CSS Grid Beispiel</title> <link rel="stylesheet" href="styles.css"> </head> <body> <div class="grid-container"> <div class="item item1">1</div> <div class="item item2">2</div> <div class="item item3">3</div> <div class="item item4">4</div> </div> </body> </html>

    2. CSS für das Grid hinzufügen:

      3. /* Basis-Styles für das Grid */ .grid-container { display: grid; /* Aktiviert Grid */ grid-template-columns: 1fr 1fr; /* Zwei gleich große Spalten */ grid-template-rows: 100px 100px; /* Zwei Zeilen mit fester Höhe */ gap: 10px; /* Abstand zwischen den Grid-Elementen */ width: 220px; /* Gesamtbreite des Grids */ background-color: #ddd; padding: 10px; } /* Styling der Grid-Elemente */ .item { background-color: steelblue; color: white; font-size: 20px; display: flex; justify-content: center; align-items: center; border-radius: 5px; }

    3. Elemente gezielt positionieren (optional):

      4. .item1 { grid-column: 1 / 3; /* Item 1 über zwei Spalten strecken */ } .item3 { grid-row: 2 / 3; /* Item 3 in die zweite Zeile verschieben */ }

CSS Layout

  • Was ist der z-index?

    • Der z-index ist eine CSS-Eigenschaft, die bestimmt, in welcher Reihenfolge sich überlappende Elemente auf einer Webseite gestapelt werden.

    Grundprinzipien des z-index:

    • z-index funktioniert nur bei position: relative, absolute, fixed oder sticky. Bei static (Standardpositionierung) hat z-index keine Wirkung.

      Beispiel: Ein relative gesetzter Eltern-Container mit einem absolute positionierten Kind kann z-index verwenden.

    • Standardmäßig haben Elemente den z-index: auto, was bedeutet, dass sie der normalen HTML-Reihenfolge folgen.

    • Elemente mit höherem z-index-Wert werden weiter vorne angezeigt.

    • Elemente mit niedrigerem z-index-Wert werden weiter hinten angezeigt.

CSS

CSS Layout

  • Welche CSS-Techniken kann man zur Gestaltung eines Layouts verwenden?

    • CSS Positioning (absolute, relative, fixed, sticky)

    • CSS Flexbox → Gut für einzeilige oder kleinere Layouts. Flexbox ist eindimensional: Das bedeutet, dass Flexbox entweder in einer Reihe (row) oder in einer Spalte (column) arbeitet – aber nicht gleichzeitig in beiden Richtungen.

    • CSS Grid → Perfekt für komplexe, flexible Layouts. Grid ist zweidimensional: Es ermöglicht sowohl die horizontale als auch die vertikale Ausrichtung von Elementen.

    • CSS Frameworks (z. B. Bootstrap, Tailwind CSS)

    • CSS Float (veraltet, aber noch in manchen Fällen genutzt): Wenn man Text um ein Bild oder ein anderes Element fließen lassen möchte, kann man float verwenden. Das Bild wird “links” oder “rechts” positioniert und der Text fließt darum herum.

  • Wie erstellt man ein Flexbox layout?

    • Ein Flexbox Layout wird mit der CSS-Eigenschaft display: flex erstellt. Flexbox ermöglicht eine einfache und flexible Möglichkeit, Elemente in einer Reihe oder Spalte anzuordnen und auszurichten.

    -> Grundstruktur:

    1. Container: Div Element (übergeordnetes Element) mit display: flex.
    2. Flex-Items: Die direkten Kinder des Containers werden zu Flex-Items und können flexibel im Layout positioniert werden.

  • Flexbox Eigenschaften:

    • -> Container-Eigenschaften:

    • display: flex -> Aktiviert das Flexbox-Modell für den Container und macht alle direkten Kinder zu Flex-Items.

    • flex-direction: Bestimmt die Ausrichtung der Flex-Items entlang der Hauptachse.
      Werte: row / column / row-reverse / column-reverse

    • flex-wrap: Bestimmt, ob Flex-Items in mehreren Zeilen oder Spalten umgebrochen werden können, wenn der Platz nicht ausreicht.
      Werte: wrap / wrap-reverse / nowrap

    • justify-content: Steuert die horizontale Ausrichtung der Flex-Items entlang der Hauptachse.
      Werte: flex-start / flex-end / center / space-between / space-around / space-evenly

    • align-items: Steuert die vertikale Ausrichtung der Flex-Items entlang der Querachse.
      Werte: flex-start / flex-end / center / baseline / stretch

    • align-content: Steuert den Abstand zwischen mehreren Flexbox-Zeilen (nur bei flex-wrap!!!).
      Werte: flex-start / flex-end / center / space-between / space-around / stretch

    -> Child Item-Eigenschaften:

    • order: Bestimmt die Reihenfolge der Flex-Items innerhalb des Containers. Der Standardwert ist 0.

    • align-self: Erlaubt es einem einzelnen Flex-Item, seine vertikale Ausrichtung zu überschreiben (überschreibt align-items für dieses Element).
      Werte: flex-start / flex-end / center / baseline / stretch

    • flex-grow: Bestimmt, wie viel ein Flex-Item wachsen kann, um den verfügbaren Platz zu füllen.
      Werte: Jede Zahl größer als 0 gibt an, wie viel das Element im Vergleich zu den anderen wächst.
      Standardwert: 0 (wächst nicht).

    • flex-shrink: Bestimmt, wie viel ein Flex-Item schrumpfen kann, wenn der Platz im Container begrenzt ist.
      Werte: Jede Zahl größer als 0 gibt an, wie viel das Element schrumpfen kann.
      Standardwert: 1 (kann schrumpfen).

    • flex-basis: Bestimmt die ursprüngliche Größe eines Flex-Items, bevor es wächst oder schrumpft.
      Werte: Eine feste Größe, z. B. 100px oder 50%.
      Standardwert: auto (die ursprüngliche Größe des Items basierend auf dessen Inhalt).

    • flex (Kurzform für flex-grow, flex-shrink und flex-basis): Eine Kombination der oben genannten Werte in einer einzigen Eigenschaft.

  • Wie erstellt man ein Grid layout?

    • -> Step by step:

    1. Container definieren: Der Container, der das Grid enthält, wird mit display: grid aktiviert.

    2. Spalten und Zeilen definieren: Festlegen, wie viele Spalten und Zeilen man im Grid haben möchte, mit den Eigenschaften grid-template-columns und grid-template-rows.

    3. Grid-Items anordnen: Die direkten Kinder des Containers (die Grid-Items) werden automatisch in den Grid-Zellen platziert.

    -> Grid Code-Beispiel:

    html:

    <div class="container"> <div class="item">Item 1</div> <div class="item">Item 2</div> <div class="item">Item 3</div> <div class="item">Item 4</div> <div class="item">Item 5</div> <div class="item">Item 6</div> <div class="item">Item 7</div> <div class="item">Item 8</div> <div class="item">Item 9</div> </div>

    CSS:

    .container { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-template-rows: 100px auto 100px; grid-gap: 10px; } .item { background-color: lightblue; padding: 20px; text-align: center; }

    -> Beispiel Erklärung:

    • Die erste und dritte Spalte sind gleich breit (1fr), und die mittlere Spalte ist doppelt so breit (2fr).

    • Die Zeilen sind so definiert, dass die erste und letzte Zeile jeweils 100px hoch sind, und die mittlere Zeile nimmt den verbleibenden Platz (auto: nimmt nur so viel Platz wie der Inhalt braucht).

CSS

CSS Basics

  • Was ist CSS und wofür wird es verwendet?

    • CSS (Cascading Style Sheets) ist eine Stylesheet-Sprache, die verwendet wird, um das Aussehen und die Gestaltung von HTML-Dokumenten zu definieren.

    CSS bestimmt, wie eine Webseite aussieht – es steuert Farben, Schriftarten, Abstände, Layouts und mehr.

  • Was für möglichkeiten gibt es CSS in einem Dokument zu verwenden?

    • Inline-CSS: Nützlich für einzelne, schnelle Änderungen / Unübersichtlich und schwer wartbar

    • Internes CSS: Nützlich für kleine Webseiten oder einzelne Seiten / Vermischt HTML und CSS → Unübersichtlich bei großen Projekten

    • Externe CSS-Datei: Trennung von Inhalt (HTML) und Design (CSS) / Mehrere HTML-Seiten können dasselbe CSS verwenden

    -> CSS-Use Beispiel

  • Wie bindet man eine CSS-Datei in ein HTML-Dokument ein?

    • Eine CSS-Datei wird mit dem <link>-Tag im <head>-Bereich eines HTML-Dokuments eingebunden.

    Beispiel:

    <head> <link rel="stylesheet" href="styles.css"> </head>

  • Was sind Selektoren?

    • Selektoren in CSS bestimmen, welche HTML-Elemente von den definierten Stilregeln betroffen sind. Sie ermöglichen es, gezielt einzelne oder mehrere Elemente zu stylen.

    -> Beispiele:

    1. Element-Selektor:
      2. p { color: blue; }

      Ergebnis: Alle <p>-Elemente werden blau.


    2. Klassen-Selektor (.)
      3. .special { color: red; }

      Ergebnis: Alle Elemente mit class="special" werden rot.


    3. ID-Selektor (#)
      4. #header { font-size: 24px; }

      Ergebnis: Nur dieses <h1>-Element wird beeinflusst.


  • Selektoren-Hierarchie:

    • Inline-Stil > Internes CSS > Externes CSS

    • #ID > .class > Tag

    • Im Code, wenn zwei CSS-Regeln die gleiche Spezifität haben und auf dasselbe Element zutreffen, dann gewinnt die Regel, die weiter unten in der CSS-Datei steht.

    • !important überschreibt alle anderen Regeln

  • Wie kann man Farben und Hintergründe in CSS definieren?

    • -> Farben mit color (Textfarbe):

    p { color: red; /* Name */ }

    -> Hintergrundfarben mit background-color:

    div { background-color: lightblue; }

    -> Hintergrundfarben mit background-color:

    div { background-color: rgba(0, 0, 255, 0.3); /* Blau mit 30% Transparenz */ }


  • Was sind Box-Model, Padding und Margin?

    • Das CSS-Box-Modell beschreibt, wie HTML-Elemente als rechteckige Boxen dargestellt werden. Jede Box besteht aus vier Bereichen:

    1. Content (Inhalt) → Der eigentliche Inhalt, z. B. Text oder ein Bild

    2. Padding (Innenabstand) → Abstand zwischen Inhalt und Rahmen

    3. Border (Rahmen) → Umrandung des Elements

    4. Margin (Außenabstand) → Abstand zwischen dem Element und anderen Elementen

    Jedes HTML-Element wird als Box dargestellt, die sich aus den oben genannten Bereichen zusammensetzt.
