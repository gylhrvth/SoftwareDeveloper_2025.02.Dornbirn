JavaScript Fetch API / JSON

  • Was bedeutet die Abkürzung JSON?

    • JSON steht für JavaScript Object Notation. Es ist ein leichtgewichtiges Datenformat, das auf der Syntax von JavaScript-Objekten basiert und häufig zum Austausch von Daten zwischen einem Server und einem Client verwendet wird.

    JSON ist einfach zu lesen und zu schreiben, sowohl für Menschen als auch für Maschinen. Es wird häufig in Webanwendungen verwendet, um Daten in einem strukturierten Format zu übertragen.

    Beispiel:

    { "name": "John Doe", "age": 30, "city": "New York" }

    In diesem Beispiel ist das JSON-Objekt eine einfache Darstellung von Daten mit den Eigenschaften "name", "age" und "city".


  • Wie erstellt man einen Text aus einem JS Object?

    • Um einen Text aus einem JavaScript-Objekt zu erstellen, kann man die JSON.stringify()-Methode verwenden. Diese Methode konvertiert ein JavaScript-Objekt in eine JSON-Zeichenfolge.

    Beispiel:

    const person = { name: "John Doe", age: 30, city: "New York" }; const jsonString = JSON.stringify(person); console.log(jsonString); // Ausgabe: {"name":"John Doe","age":30,"city":"New York"}

    In diesem Beispiel wird das JavaScript-Objekt person in eine JSON-Zeichenfolge umgewandelt und in der Konsole ausgegeben.


  • Wie erstellt man ein JS Object aus einem Text?

    • Um ein JavaScript-Objekt aus einer JSON-Zeichenfolge zu erstellen, kann man die JSON.parse()-Methode verwenden. Diese Methode konvertiert eine JSON-Zeichenfolge in ein JavaScript-Objekt.

    Beispiel:

    const jsonString = '{"name": "John Doe", "age": 30, "city": "New York"}'; const person = JSON.parse(jsonString); console.log(person.name); // Ausgabe: John Doe console.log(person.age); // Ausgabe: 30 console.log(person.city); // Ausgabe: New York

    In diesem Beispiel wird die JSON-Zeichenfolge jsonString in ein JavaScript-Objekt umgewandelt und die Eigenschaften des Objekts werden in der Konsole ausgegeben.


  • Was bedeutet die Abkürzung REST API?

    • REST steht für Representational State Transfer. Eine REST API ist eine Programmierschnittstelle, die auf den Prinzipien von REST basiert und es ermöglicht, über HTTP-Anfragen auf Ressourcen zuzugreifen und diese zu manipulieren.

    REST APIs verwenden die HTTP-Methoden GET, POST, PUT, DELETE usw., um Daten zu lesen, zu erstellen, zu aktualisieren und zu löschen. Sie sind plattformunabhängig und ermöglichen die Interaktion zwischen verschiedenen Systemen.

    Beispiel:

    GET /api/users - Ruft eine Liste von Benutzern ab POST /api/users - Erstellt einen neuen Benutzer PUT /api/users/1 - Aktualisiert den Benutzer mit der ID 1 DELETE /api/users/1 - Löscht den Benutzer mit der ID 1

    In diesem Beispiel werden verschiedene HTTP-Methoden verwendet, um auf eine Benutzerressource zuzugreifen und diese zu manipulieren.


  • Was bedeutet, dass ein API public ist?

    • Eine öffentliche API ist eine Programmierschnittstelle, die für die allgemeine Nutzung durch Entwickler und Anwendungen zugänglich ist. Sie erfordert in der Regel keine Authentifizierung oder spezielle Berechtigungen, um darauf zuzugreifen.

    Öffentliche APIs werden häufig von Unternehmen bereitgestellt, um Entwicklern den Zugriff auf ihre Dienste und Daten zu ermöglichen. Sie sind in der Regel gut dokumentiert und bieten eine Vielzahl von Funktionen.

    Beispiel:

    https://api.example.com/v1/users

    In diesem Beispiel ist die URL eine öffentliche API, die den Zugriff auf Benutzerdaten ermöglicht.


  • Was ist ein fetch API und was hat es zu tun mit REST API?

    • Die Fetch API ist eine moderne JavaScript-API, die es ermöglicht, HTTP-Anfragen zu senden und Antworten zu empfangen. Sie wird häufig verwendet, um Daten von REST APIs abzurufen oder an sie zu senden.

    Die Fetch API bietet eine einfache und flexible Möglichkeit, asynchrone HTTP-Anfragen zu erstellen und mit den Antworten zu arbeiten. Sie unterstützt Promises, was die Handhabung von asynchronen Operationen erleichtert.

    Beispiel:

    fetch('https://api.example.com/v1/users') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Fehler:', error));

    In diesem Beispiel wird die Fetch API verwendet, um eine GET-Anfrage an eine REST API zu senden und die Antwort als JSON zu verarbeiten.


  • Was ist die minimale Parameterliste die eine fetch() Aufruf haben muss?

    • Die minimale Parameterliste für einen fetch()-Aufruf besteht aus der URL, die abgerufen werden soll. Optional kann auch eine Konfigurationsoption angegeben werden, um die HTTP-Methode und andere Einstellungen festzulegen.

    Beispiel:

    fetch('https://api.example.com/v1/users');

    In diesem Beispiel wird die Fetch API mit der URL aufgerufen, um eine GET-Anfrage an die angegebene Ressource zu senden.


  • Welche Optionen sind möglich für einen fetch() Aufruf?

    • Die Fetch API bietet eine Vielzahl von Optionen, die beim Aufruf der fetch()-Funktion angegeben werden können. Zu den häufigsten Optionen gehören:

    • method: Gibt die HTTP-Methode an (z. B. GET, POST, PUT, DELETE).
    • headers: Ermöglicht das Festlegen von HTTP-Headern für die Anfrage.
    • body: Enthält den Inhalt der Anfrage (z. B. JSON-Daten für POST-Anfragen).
    • mode: Gibt den Modus der Anfrage an (z. B. cors, no-cors, same-origin).
    • credentials: Gibt an, ob Cookies und Authentifizierungsdaten gesendet werden sollen (z. B. include, same-origin).
    • cache: Gibt an, wie die Anfrage zwischengespeichert werden soll (z. B. default, no-cache, reload).
    • redirect: Gibt an, wie mit Weiterleitungen umgegangen werden soll (z. B. follow, manual).
    • referrer: Gibt den Referrer-Header an.
    • signal: Ermöglicht das Abbrechen der Anfrage mit einem AbortController.

    Beispiel:

    fetch('https://api.example.com/v1/users', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: 'John Doe', age: 30 }) });

    In diesem Beispiel wird die Fetch API mit der POST-Methode aufgerufen, um einen neuen Benutzer zu erstellen. Die Anfrage enthält Header und den Body mit den Benutzerdaten.


  • Wofür ist die Methode GET geeignet?

    • Die GET-Methode wird verwendet, um Daten von einem Server abzurufen. Sie ist geeignet, wenn du Informationen anfordern möchtest, ohne den Zustand des Servers zu ändern.

    GET-Anfragen sind idempotent, was bedeutet, dass sie keine Nebenwirkungen haben und mehrmals ausgeführt werden können, ohne das Ergebnis zu ändern.

    Beispiel:

    fetch('https://api.example.com/v1/users') .then(response => response.json()) .then(data => console.log(data));

    In diesem Beispiel wird die GET-Methode verwendet, um eine Liste von Benutzern von einer API abzurufen.


  • Wofür ist die Methode POST geeignet?

    • Die POST-Methode wird verwendet, um Daten an einen Server zu senden und eine neue Ressource zu erstellen. Sie ist geeignet, wenn du Informationen übermitteln möchtest, die den Zustand des Servers ändern.

    POST-Anfragen sind nicht idempotent, was bedeutet, dass sie bei mehrfacher Ausführung unterschiedliche Ergebnisse liefern können.

    Beispiel:

    fetch('https://api.example.com/v1/users', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: 'John Doe', age: 30 }) });

    In diesem Beispiel wird die POST-Methode verwendet, um einen neuen Benutzer an eine API zu senden.


  • Wofür ist die Methode PUT geeignet?

    • Die PUT-Methode wird verwendet, um eine vorhandene Ressource auf einem Server zu aktualisieren. Sie ist geeignet, wenn du Informationen ändern möchtest, die bereits auf dem Server vorhanden sind.

    PUT-Anfragen sind idempotent, was bedeutet, dass sie bei mehrfacher Ausführung das gleiche Ergebnis liefern.

    Beispiel:

    fetch('https://api.example.com/v1/users/1', { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: 'Jane Doe', age: 25 }) });

    In diesem Beispiel wird die PUT-Methode verwendet, um die Informationen eines Benutzers mit der ID 1 zu aktualisieren.


  • Wofür ist die Methode DELETE geeignet?

    • Die DELETE-Methode wird verwendet, um eine Ressource auf einem Server zu löschen. Sie ist geeignet, wenn du Informationen entfernen möchtest, die auf dem Server gespeichert sind.

    DELETE-Anfragen sind idempotent, was bedeutet, dass sie bei mehrfacher Ausführung das gleiche Ergebnis liefern.

    Beispiel:

    fetch('https://api.example.com/v1/users/1', { method: 'DELETE' });

    In diesem Beispiel wird die DELETE-Methode verwendet, um den Benutzer mit der ID 1 von einer API zu löschen.

JavaScript async Functions

  • Was sind Synchrone Funktionen?

    • Synchrone Funktionen sind Funktionen, die sofort ausgeführt werden, eine nach der anderen, in der Reihenfolge, in der sie im Code stehen. Das heißt: Man wartet, bis eine Funktion fertig ist, bevor die nächste startet.

    Nachteil: Wenn eine Funktion lange dauert, z. B. Daten laden oder auf einen Server warten, blockiert sie den Rest.

  • Was sind Asynchrone Funktionen?

    • Asynchrone Funktionen sind Funktionen, deren Ablauf nicht den Programmfluss blockiert. Statt direkt ein Ergebnis zurückzugeben, melden sie sich später, wenn der Vorgang abgeschlossen ist.

    Sie werden häufig mit Promises oder async/await verwendet.

  • Was ist eine Callback Funktion?

    • Eine Callback-Funktion ist eine Funktion, die als Argument an eine andere Funktion übergeben wird und dann aufgerufen wird, wenn ein bestimmtes Ereignis eintritt oder eine bestimmte Bedingung erfüllt ist.

    Sie wird oft verwendet, um asynchrone Operationen zu handhaben, z. B. beim Laden von Daten oder beim Warten auf Benutzerinteraktionen.

    Beispiel:

    function fetchData(callback) { setTimeout(() => { const data = "Daten geladen!"; callback(data); }, 2000); } fetchData((result) => { console.log(result); // Ausgabe: Daten geladen! });

    In diesem Beispiel wird die Funktion fetchData aufgerufen, die eine Callback-Funktion als Argument erhält. Nach 2 Sekunden wird die Callback-Funktion mit den geladenen Daten aufgerufen.

  • Was ist eine Promise?

    • Eine Promise ist ein Objekt, das einen zukünftigen Wert repräsentiert, der möglicherweise noch nicht verfügbar ist. Sie wird verwendet, um asynchrone Operationen zu handhaben und ermöglicht es, den Erfolg oder Misserfolg einer Operation zu behandeln.

    Eine Promise kann drei Zustände haben: pending (ausstehend), fulfilled (erfüllt) oder rejected (abgelehnt).

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); });

    In diesem Beispiel wird eine Promise erstellt, die nach 2 Sekunden erfüllt wird. Wenn die Promise erfolgreich ist, wird die then-Methode aufgerufen, andernfalls wird die catch-Methode aufgerufen.

  • Was ist .then() in JavaScript?

    • .then() ist eine Methode, die auf einem Promise-Objekt aufgerufen wird, um den Erfolg oder Misserfolg einer asynchronen Operation zu behandeln. Sie ermöglicht es, eine Funktion anzugeben, die ausgeführt wird, wenn das Promise erfüllt wird.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); });

    In diesem Beispiel wird die then-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die ausgeführt wird, wenn das Promise erfüllt wird.

  • Was ist .catch() in JavaScript?

    • .catch() ist eine Methode, die auf einem Promise-Objekt aufgerufen wird, um Fehler zu behandeln, die während einer asynchronen Operation auftreten können. Sie ermöglicht es, eine Funktion anzugeben, die ausgeführt wird, wenn das Promise abgelehnt wird.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const error = "Fehler beim Laden der Daten!"; reject(error); }, 2000); }); fetchData .then((result) => { console.log(result); }) .catch((error) => { console.error(error); // Ausgabe: Fehler beim Laden der Daten! });

    In diesem Beispiel wird die catch-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die ausgeführt wird, wenn das Promise abgelehnt wird.

  • Was ist eine final() Statement?

    • Ein finally-Statement ist eine Methode, die auf einem Promise-Objekt aufgerufen wird und eine Funktion angibt, die immer ausgeführt wird, unabhängig davon, ob das Promise erfüllt oder abgelehnt wurde. Es wird häufig verwendet, um Aufräumarbeiten durchzuführen.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); }) .finally(() => { console.log("Aufräumarbeiten abgeschlossen!"); // Immer ausgeführt });

    In diesem Beispiel wird die finally-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die immer ausgeführt wird, unabhängig davon, ob das Promise erfüllt oder abgelehnt wurde.

  • Was ist async/await in JavaScript?

    • async/await ist eine Syntax in JavaScript, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben. async wird vor einer Funktion verwendet, um anzugeben, dass sie eine Promise zurückgibt, und await wird verwendet, um auf das Ergebnis einer Promise zu warten.

    Beispiel:

    async function fetchData() { const data = await new Promise((resolve) => { setTimeout(() => { resolve("Daten geladen!"); }, 2000); }); console.log(data); // Ausgabe: Daten geladen! } fetchData();

    In diesem Beispiel wird die async-Funktion fetchData erstellt, die eine Promise zurückgibt. Innerhalb der Funktion wird await verwendet, um auf das Ergebnis der Promise zu warten, bevor es in der Konsole ausgegeben wird.

  • Was ist der Unterschied zwischen async/await und Promises?

    • async/await ist eine syntaktische Verbesserung von Promises, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben. async wird vor einer Funktion verwendet, um anzugeben, dass sie eine Promise zurückgibt, und await wird verwendet, um auf das Ergebnis einer Promise zu warten.

    Promises sind Objekte, die einen zukünftigen Wert repräsentieren und es ermöglichen, den Erfolg oder Misserfolg einer asynchronen Operation zu behandeln.

    async/await ist eine syntaktische Verbesserung von Promises, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben.

  • In welchen Situationen is await geeignet?

    • Das await-Schlüsselwort in JavaScript ist geeignet, wenn du auf das Ergebnis einer asynchronen Operation warten möchtest, bevor der nächste Code ausgeführt wird. Es wird in einer async-Funktion verwendet und pausiert die Ausführung der Funktion, bis die Promise aufgelöst (fulfilled) oder abgelehnt (rejected) wird.

    Beispiel:

    async function fetchData() { try { const response = await fetch("https://api.example.com/data"); const data = await response.json(); console.log(data); // Verarbeite die Daten } catch (error) { console.error("Fehler beim Abrufen der Daten:", error); } } fetchData();
