JavaScript async Functions

  • Was sind Synchrone Funktionen?

    • Synchrone Funktionen sind Funktionen, die sofort ausgeführt werden, eine nach der anderen, in der Reihenfolge, in der sie im Code stehen. Das heißt: Man wartet, bis eine Funktion fertig ist, bevor die nächste startet.

    Nachteil: Wenn eine Funktion lange dauert, z. B. Daten laden oder auf einen Server warten, blockiert sie den Rest.

  • Was sind Asynchrone Funktionen?

    • Asynchrone Funktionen sind Funktionen, deren Ablauf nicht den Programmfluss blockiert. Statt direkt ein Ergebnis zurückzugeben, melden sie sich später, wenn der Vorgang abgeschlossen ist.

    Sie werden häufig mit Promises oder async/await verwendet.

  • Was ist eine Callback Funktion?

    • Eine Callback-Funktion ist eine Funktion, die als Argument an eine andere Funktion übergeben wird und dann aufgerufen wird, wenn ein bestimmtes Ereignis eintritt oder eine bestimmte Bedingung erfüllt ist.

    Sie wird oft verwendet, um asynchrone Operationen zu handhaben, z. B. beim Laden von Daten oder beim Warten auf Benutzerinteraktionen.

    Beispiel:

    function fetchData(callback) { setTimeout(() => { const data = "Daten geladen!"; callback(data); }, 2000); } fetchData((result) => { console.log(result); // Ausgabe: Daten geladen! });

    In diesem Beispiel wird die Funktion fetchData aufgerufen, die eine Callback-Funktion als Argument erhält. Nach 2 Sekunden wird die Callback-Funktion mit den geladenen Daten aufgerufen.

  • Was ist eine Promise?

    • Eine Promise ist ein Objekt, das einen zukünftigen Wert repräsentiert, der möglicherweise noch nicht verfügbar ist. Sie wird verwendet, um asynchrone Operationen zu handhaben und ermöglicht es, den Erfolg oder Misserfolg einer Operation zu behandeln.

    Eine Promise kann drei Zustände haben: pending (ausstehend), fulfilled (erfüllt) oder rejected (abgelehnt).

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); });

    In diesem Beispiel wird eine Promise erstellt, die nach 2 Sekunden erfüllt wird. Wenn die Promise erfolgreich ist, wird die then-Methode aufgerufen, andernfalls wird die catch-Methode aufgerufen.

  • Was ist .then() in JavaScript?

    • .then() ist eine Methode, die auf einem Promise-Objekt aufgerufen wird, um den Erfolg oder Misserfolg einer asynchronen Operation zu behandeln. Sie ermöglicht es, eine Funktion anzugeben, die ausgeführt wird, wenn das Promise erfüllt wird.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); });

    In diesem Beispiel wird die then-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die ausgeführt wird, wenn das Promise erfüllt wird.

  • Was ist .catch() in JavaScript?

    • .catch() ist eine Methode, die auf einem Promise-Objekt aufgerufen wird, um Fehler zu behandeln, die während einer asynchronen Operation auftreten können. Sie ermöglicht es, eine Funktion anzugeben, die ausgeführt wird, wenn das Promise abgelehnt wird.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const error = "Fehler beim Laden der Daten!"; reject(error); }, 2000); }); fetchData .then((result) => { console.log(result); }) .catch((error) => { console.error(error); // Ausgabe: Fehler beim Laden der Daten! });

    In diesem Beispiel wird die catch-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die ausgeführt wird, wenn das Promise abgelehnt wird.

  • Was ist eine final() Statement?

    • Ein finally-Statement ist eine Methode, die auf einem Promise-Objekt aufgerufen wird und eine Funktion angibt, die immer ausgeführt wird, unabhängig davon, ob das Promise erfüllt oder abgelehnt wurde. Es wird häufig verwendet, um Aufräumarbeiten durchzuführen.

    Beispiel:

    const fetchData = new Promise((resolve, reject) => { setTimeout(() => { const data = "Daten geladen!"; resolve(data); }, 2000); }); fetchData .then((result) => { console.log(result); // Ausgabe: Daten geladen! }) .catch((error) => { console.error(error); }) .finally(() => { console.log("Aufräumarbeiten abgeschlossen!"); // Immer ausgeführt });

    In diesem Beispiel wird die finally-Methode auf dem Promise-Objekt aufgerufen, um die Funktion zu definieren, die immer ausgeführt wird, unabhängig davon, ob das Promise erfüllt oder abgelehnt wurde.

  • Was ist async/await in JavaScript?

    • async/await ist eine Syntax in JavaScript, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben. async wird vor einer Funktion verwendet, um anzugeben, dass sie eine Promise zurückgibt, und await wird verwendet, um auf das Ergebnis einer Promise zu warten.

    Beispiel:

    async function fetchData() { const data = await new Promise((resolve) => { setTimeout(() => { resolve("Daten geladen!"); }, 2000); }); console.log(data); // Ausgabe: Daten geladen! } fetchData();

    In diesem Beispiel wird die async-Funktion fetchData erstellt, die eine Promise zurückgibt. Innerhalb der Funktion wird await verwendet, um auf das Ergebnis der Promise zu warten, bevor es in der Konsole ausgegeben wird.

  • Was ist der Unterschied zwischen async/await und Promises?

    • async/await ist eine syntaktische Verbesserung von Promises, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben. async wird vor einer Funktion verwendet, um anzugeben, dass sie eine Promise zurückgibt, und await wird verwendet, um auf das Ergebnis einer Promise zu warten.

    Promises sind Objekte, die einen zukünftigen Wert repräsentieren und es ermöglichen, den Erfolg oder Misserfolg einer asynchronen Operation zu behandeln.

    async/await ist eine syntaktische Verbesserung von Promises, die es ermöglicht, asynchrone Funktionen einfacher und lesbarer zu schreiben.

  • In welchen Situationen is await geeignet?

    • Das await-Schlüsselwort in JavaScript ist geeignet, wenn du auf das Ergebnis einer asynchronen Operation warten möchtest, bevor der nächste Code ausgeführt wird. Es wird in einer async-Funktion verwendet und pausiert die Ausführung der Funktion, bis die Promise aufgelöst (fulfilled) oder abgelehnt (rejected) wird.

    Beispiel:

    async function fetchData() { try { const response = await fetch("https://api.example.com/data"); const data = await response.json(); console.log(data); // Verarbeite die Daten } catch (error) { console.error("Fehler beim Abrufen der Daten:", error); } } fetchData();
