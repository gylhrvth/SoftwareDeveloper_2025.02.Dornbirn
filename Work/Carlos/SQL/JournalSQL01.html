Mittwoch 07. Mai 2025

SQL (Query in Query)

Beispiel:

/* Case 1: Konstant Wert */
SELECT Code FROM country WHERE name = 'Austria';
SELECT * FROM city
WHERE country = (SELECT Code FROM country WHERE name = 'Austria');

/* Case 2: Liste */
SELECT Code FROM country WHERE name like '%ria';
SELECT * FROM city
WHERE country IN (SELECT Code FROM country WHERE name like '%ria');

/* Case 3: Tabelle */
SELECT Code, Name FROM country WHERE name like '%ria';
SELECT co.Name, c.Name AS city
FROM city c
JOIN (SELECT Code, Name FROM country WHERE name like '%ria') AS co ON c.Country = co.Code
ORDER BY co.Name, c.Name

/* The same with WITH statement */
WITH co AS (SELECT Code, Name FROM country WHERE name like '%ria')
SELECT co.Name, c.Name AS city
FROM city c JOIN co ON c.Country = co.Code
ORDER BY co.Name, c.Name

Important!: Keyword HAVING ist eigentlich eine Abkürzung für eine Sub-Querie.

  11. Alle Länder mit mindestens einem See mit mindestens 100 Meter Tiefe und mindestens einem Berg mit mindestens 1500 Höhenmeter

    12. SELECT
    country.Name AS Country,
    COUNT(DISTINCT lake.Name) AS NumberOfLakes,
    COUNT(DISTINCT mountain.Name) AS NumberOfMountains
    FROM country
    JOIN geo_lake ON country.Code = geo_lake.Country
    JOIN lake ON geo_lake.Lake = lake.Name
    JOIN geo_mountain ON country.Code = geo_mountain.Country
    JOIN mountain ON geo_mountain.Mountain = mountain.Name
    WHERE lake.Depth > 100
    AND mountain.Height > 1500
    GROUP BY country.Name
    HAVING NumberOfLakes > 0 AND NumberOfMountains > 0;

SQL (Select) Recherche:

  • Was ist eine Datenbank?

    • Eine Datenbank ist eine strukturierte Sammlung von Daten, die auf einem Computer gespeichert und verwaltet wird. Sie ermöglicht es Benutzern, Daten effizient zu speichern, abzurufen und zu bearbeiten.


  • Was ist eine relationale Datenbank?

    • Eine relationale Datenbank ist eine Art von Datenbank, die Daten in Tabellen organisiert, die durch Beziehungen miteinander verbunden sind. Jede Tabelle besteht aus Zeilen (Datensätzen) und Spalten (Attributen), und die Beziehungen zwischen den Tabellen werden durch Schlüssel definiert.


  • Was ist eine non-relationale Datenbank?

    • Eine non-relationale Datenbank ist eine Art von Datenbank, die nicht auf dem relationalen Modell basiert. Sie speichert Daten in verschiedenen Formaten, wie z.B. Dokumenten, Schlüssel-Wert-Paaren oder Graphen. Non-relationale Datenbanken sind oft flexibler und skalierbarer als relationale Datenbanken.


  • Was ist SQL?

    • SQL (Structured Query Language) ist eine standardisierte Programmiersprache, die zur Verwaltung und Manipulation von relationalen Datenbanken verwendet wird. Sie ermöglicht es Benutzern, Daten abzufragen, zu aktualisieren, zu löschen und zu erstellen.


  • Was sind die Vorteile von SQL?

    • Die Vorteile von SQL sind:

    • Standardisierte Sprache: SQL ist ein weit verbreiteter Standard, der in vielen Datenbankmanagementsystemen unterstützt wird.
    • Einfachheit: SQL ist relativ einfach zu erlernen und zu verwenden, was es Entwicklern erleichtert, mit Datenbanken zu arbeiten.
    • Flexibilität: SQL ermöglicht komplexe Abfragen und Datenmanipulationen, die auf verschiedene Anforderungen zugeschnitten werden können.
    • Transaktionen: SQL unterstützt Transaktionen, die sicherstellen, dass mehrere Operationen als atomare Einheit ausgeführt werden.


  • Was sind die wichtigsten SQL-Datenbankkonzepte?

    • Die wichtigsten SQL-Datenbankkonzepte sind:

    • Tabellen: Daten werden in Tabellen organisiert, die aus Zeilen und Spalten bestehen.
    • Primärschlüssel: Ein Primärschlüssel ist ein eindeutiger Bezeichner für jede Zeile in einer Tabelle.
    • Fremdschlüssel: Ein Fremdschlüssel ist ein Attribut, das auf den Primärschlüssel einer anderen Tabelle verweist und Beziehungen zwischen Tabellen definiert.
    • Indizes: Indizes sind Datenstrukturen, die den Zugriff auf Daten in einer Tabelle beschleunigen.


  • Was sind die wichtigsten SQL-Datenbankmanagementsysteme (DBMS)?

    • Die wichtigsten SQL-Datenbanktypen sind:

    • MySQL: Ein Open-Source-Datenbankmanagementsystem, das häufig für Webanwendungen verwendet wird.
    • PostgreSQL: Ein fortschrittliches Open-Source-Datenbankmanagementsystem mit umfangreichen Funktionen.
    • Oracle Database: Ein kommerzielles Datenbankmanagementsystem, das für große Unternehmen entwickelt wurde.
    • Microsoft SQL Server: Ein relationales Datenbankmanagementsystem von Microsoft, das häufig in Unternehmensumgebungen eingesetzt wird.
    • SQLite: Eine leichtgewichtige, serverlose SQL-Datenbank, die in vielen Anwendungen und mobilen Geräten verwendet wird.


  • Was sind die wichtigsten SQL-Befehle?

    • Die wichtigsten SQL-Befehle sind:

    • SELECT: Abfragen von Daten aus einer Tabelle
    • INSERT: Einfügen neuer Daten in eine Tabelle
    • UPDATE: Aktualisieren vorhandener Daten in einer Tabelle
    • DELETE: Löschen von Daten aus einer Tabelle
    • CREATE: Erstellen neuer Tabellen oder Datenbanken
    • DROP: Löschen von Tabellen oder Datenbanken


  • Was sind die wichtigsten SQL-Datenbankoperationen?

    • Die wichtigsten SQL-Datenbankoperationen sind:

    • Datenabfrage: Abfragen von Daten aus einer oder mehreren Tabellen mit dem SELECT-Befehl.
    • Datenmanipulation: Einfügen, Aktualisieren und Löschen von Daten in Tabellen mit den Befehlen INSERT, UPDATE und DELETE.
    • Tabellenverwaltung: Erstellen und Löschen von Tabellen und Datenbanken mit den Befehlen CREATE und DROP.
    • Transaktionsmanagement: Sicherstellen, dass mehrere Datenbankoperationen als atomare Einheit ausgeführt werden, um Datenintegrität zu gewährleisten.
    • Indizierung: Erstellen von Indizes, um den Zugriff auf Daten zu beschleunigen und die Abfrageleistung zu verbessern.
    • Datensicherung und Wiederherstellung: Sichern von Datenbanken und Wiederherstellen von Daten im Falle eines Ausfalls oder Datenverlusts.
    • Benutzerverwaltung: Verwalten von Benutzern und Berechtigungen, um den Zugriff auf Datenbanken zu steuern.
    • Optimierung: Optimieren von Abfragen und Datenbankdesign, um die Leistung zu verbessern.
    • Normalisierung: Organisieren von Daten in Tabellen, um Redundanz zu minimieren und die Datenintegrität zu gewährleisten.


  • Was ist die Grundstruktur eines SELECT Befehls?

    • Die Grundstruktur eines SELECT-Befehls lautet:

    • SELECT: Gibt an, welche Spalten abgerufen werden sollen.
    • FROM: Gibt die Tabelle an, aus der die Daten abgerufen werden sollen.
    • WHERE: Filtert die Daten basierend auf bestimmten Bedingungen.
    • ORDER BY: Sortiert die Ergebnisse nach einer oder mehreren Spalten.
    • GROUP BY: Gruppiert die Ergebnisse basierend auf einer oder mehreren Spalten.
    • HAVING: Filtert die gruppierten Ergebnisse basierend auf bestimmten Bedingungen.


  • Wie funktioniert das LIMIT Statement?

    • Das LIMIT-Statement wird verwendet, um die Anzahl der zurückgegebenen Zeilen in einer SQL-Abfrage zu begrenzen. Es wird häufig in Kombination mit dem SELECT-Befehl verwendet, um eine bestimmte Anzahl von Ergebnissen zurückzugeben.

    Zum Beispiel:

    SELECT * FROM table_name LIMIT 10;

    In diesem Beispiel werden nur die ersten 10 Zeilen aus der Tabelle "table_name" zurückgegeben.


  • Wie funktioniert das WHERE Statement?

    • Das WHERE-Statement wird verwendet, um Bedingungen für die Auswahl von Daten in einer SQL-Abfrage festzulegen. Es filtert die Ergebnisse basierend auf den angegebenen Bedingungen.

    Zum Beispiel:

    SELECT * FROM table_name WHERE column_name = 'value';

    In diesem Beispiel werden nur die Zeilen aus der Tabelle "table_name" zurückgegeben, bei denen der Wert in der Spalte "column_name" gleich 'value' ist.


  • Welche Funktionen kann man für die Bearbeitung von Zahlen verwenden?

    • In SQL gibt es verschiedene Funktionen zur Bearbeitung von Zahlen, darunter:

    • ABS(): Gibt den absoluten Wert einer Zahl zurück.
    • CEIL() oder CEILING(): Rundet eine Zahl auf die nächste ganze Zahl auf.
    • FLOOR(): Rundet eine Zahl auf die nächste ganze Zahl ab.
    • ROUND(): Rundet eine Zahl auf eine bestimmte Anzahl von Dezimalstellen.
    • SUM(): Gibt die Summe der Werte in einer Spalte zurück.
    • AVG(): Gibt den Durchschnitt der Werte in einer Spalte zurück.
    • MIN(): Gibt den kleinsten Wert in einer Spalte zurück.
    • MAX(): Gibt den größten Wert in einer Spalte zurück.
    • COUNT(): Gibt die Anzahl der Zeilen in einer Abfrage zurück.
    • MOD(): Gibt den Rest einer Division zurück.
    • POWER(): Gibt eine Zahl hoch einer anderen Zahl zurück.
    • SQRT(): Gibt die Quadratwurzel einer Zahl zurück.
    • EXP(): Gibt die Exponentialfunktion einer Zahl zurück.

    Diese Funktionen können in SQL-Abfragen verwendet werden, um Berechnungen und Aggregationen auf numerischen Daten durchzuführen.


  • Welche Funktionen kann man für die Bearbeitung von Texten verwenden?

    • In SQL gibt es verschiedene Funktionen zur Bearbeitung von Texten, darunter:

    • UPPER(): Wandelt einen Text in Großbuchstaben um.
    • LOWER(): Wandelt einen Text in Kleinbuchstaben um.
    • LENGTH(): Gibt die Länge eines Textes zurück.
    • SUBSTR() oder SUBSTRING(): Gibt einen Teilstring aus einem Text zurück.
    • TRIM(): Entfernt Leerzeichen am Anfang und Ende eines Textes.
    • CONCAT(): Verbindet mehrere Texte zu einem einzigen Text.
    • REPLACE(): Ersetzt Teile eines Textes durch einen anderen Text.
    • LEFT(): Gibt die linke Seite eines Textes zurück.
    • RIGHT(): Gibt die rechte Seite eines Textes zurück.
    • CHARINDEX(): Gibt die Position eines bestimmten Zeichens in einem Text zurück.
    • INSTR(): Gibt die Position eines bestimmten Zeichens in einem Text zurück.

    Diese Funktionen können in SQL-Abfragen verwendet werden, um Textdaten zu manipulieren und zu formatieren.


  • Welche logischen Operatoren kann man verwenden?

    • In SQL gibt es verschiedene logische Operatoren, die verwendet werden können, um Bedingungen in Abfragen zu kombinieren oder zu negieren. Die wichtigsten logischen Operatoren sind:

    • AND: Verknüpft zwei oder mehr Bedingungen, die alle erfüllt sein müssen.
    • OR: Verknüpft zwei oder mehr Bedingungen, von denen mindestens eine erfüllt sein muss.
    • NOT: Negiert eine Bedingung, sodass nur die Zeilen zurückgegeben werden, die die Bedingung nicht erfüllen.
    • BETWEEN: Überprüft, ob ein Wert innerhalb eines bestimmten Bereichs liegt.
    • IN: Überprüft, ob ein Wert in einer Liste von Werten enthalten ist.
    • LIKE: Überprüft, ob ein Textmuster mit einem bestimmten Muster übereinstimmt.
    • IS NULL: Überprüft, ob ein Wert NULL ist.
    • IS NOT NULL: Überprüft, ob ein Wert nicht NULL ist.
    • EXISTS: Überprüft, ob eine Unterabfrage mindestens eine Zeile zurückgibt.
    • ANY: Überprüft, ob eine Bedingung für mindestens einen Wert in einer Liste erfüllt ist.
    • ALL: Überprüft, ob eine Bedingung für alle Werte in einer Liste erfüllt ist.

    Diese logischen Operatoren können in SQL-Abfragen verwendet werden, um komplexe Bedingungen zu erstellen und die Ergebnisse zu filtern.


  • Wie funktioniert ORDER BY Statement?

    • Das ORDER BY-Statement wird verwendet, um die Ergebnisse einer SQL-Abfrage nach einer oder mehreren Spalten zu sortieren. Es kann in aufsteigender (ASC) oder absteigender (DESC) Reihenfolge erfolgen.

    Zum Beispiel:

    SELECT * FROM employees ORDER BY last_name ASC;

    In diesem Beispiel werden die Ergebnisse aus der Tabelle "employees" nach der Spalte "last_name" aufsteigend sortiert.


  • Wie funktioniert GROUP BY Statement?

    • Das GROUP BY-Statement wird verwendet, um die Ergebnisse einer SQL-Abfrage in Gruppen zu organisieren, basierend auf einer oder mehreren Spalten. Es wird häufig in Kombination mit Aggregatfunktionen wie COUNT(), SUM(), AVG(), MIN() und MAX() verwendet.

    Zum Beispiel:

    SELECT department, SUM(salary) AS total_salary FROM employees GROUP BY department;

    In diesem Beispiel werden die Ergebnisse aus der Tabelle "employees" nach der Spalte "department" je nach insgesammten "salary" gruppiert.


  • Wie funktioniert JOIN Statement?

    • Das JOIN-Statement wird verwendet, um Daten aus mehreren Tabellen zu kombinieren, basierend auf einer gemeinsamen Spalte. Es gibt verschiedene Arten von Joins, darunter INNER JOIN, LEFT JOIN, RIGHT JOIN und FULL OUTER JOIN.

    Zum Beispiel:

    SELECT employees.first_name, employees.last_name, departments.department_name FROM employees INNER JOIN departments ON employees.department_id = departments.id;

    In diesem Beispiel werden die Ergebnisse aus den Tabellen "employees" und "departments" kombiniert, basierend auf der gemeinsamen Spalte "department_id".


  • Wie funktioniert Left Join Statement?

    • Das LEFT JOIN-Statement wird verwendet, um alle Zeilen aus der linken Tabelle und die übereinstimmenden Zeilen aus der rechten Tabelle zurückzugeben. Wenn keine Übereinstimmung gefunden wird, werden NULL-Werte für die Spalten der rechten Tabelle zurückgegeben.

    Zum Beispiel:

    SELECT employees.first_name, employees.last_name, departments.department_name FROM employees LEFT JOIN departments ON employees.department_id = departments.id;

    In diesem Beispiel werden alle Zeilen aus der Tabelle "employees" zurückgegeben, zusammen mit den übereinstimmenden Zeilen aus der Tabelle "departments". Wenn ein Mitarbeiter keinem Abteilung zugeordnet ist, werden NULL-Werte für die Spalten der Tabelle "departments" zurückgegeben.


  • Wie funktionieren IS NULL und IS NOT NULL Kriterien?

    • Die Kriterien IS NULL und IS NOT NULL werden verwendet, um zu überprüfen, ob ein Wert NULL ist oder nicht. Sie werden häufig in WHERE-Klauseln verwendet, um Zeilen mit NULL-Werten oder ohne NULL-Werte zu filtern.

    Zum Beispiel:

    SELECT * FROM employees WHERE department_id IS NULL;

    In diesem Beispiel werden alle Zeilen aus der Tabelle "employees" zurückgegeben, bei denen der Wert in der Spalte "department_id" NULL ist.


  • Wie funktioniert DISTINCT Statement?

    • Das DISTINCT-Statement wird verwendet, um doppelte Werte aus den Ergebnissen einer SQL-Abfrage zu entfernen. Es wird häufig in Kombination mit dem SELECT-Befehl verwendet.

    Zum Beispiel:

    SELECT DISTINCT department FROM employees;

    In diesem Beispiel werden nur die eindeutigen Werte aus der Spalte "department" in der Tabelle "employees" zurückgegeben.
