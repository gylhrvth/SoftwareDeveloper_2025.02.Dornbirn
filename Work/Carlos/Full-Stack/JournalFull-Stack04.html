React (Recherche)

  • Woher kommt der Name "React"?

    • Der Name "React" stammt von der Fähigkeit der Bibliothek, auf Änderungen im Zustand der Anwendung zu reagieren und die Benutzeroberfläche entsprechend zu aktualisieren.

  • Was ist JSX?

    • JSX (JavaScript XML) ist eine Syntaxerweiterung für JavaScript, die es ermöglicht, HTML-ähnliche Syntax in JavaScript-Dateien zu verwenden. Es wird häufig in React verwendet, um Komponenten zu definieren.

  • Was ist ein Component?

    • Ein Component (Komponente) in React ist eine wiederverwendbare, isolierte Einheit, die eine bestimmte Funktionalität oder ein bestimmtes UI-Element kapselt. Komponenten können in andere Komponenten eingebettet werden und ermöglichen eine modulare Entwicklung.

  • Was sind Props?

    • Props (Eigenschaften) sind Parameter, die an Komponenten übergeben werden, um deren Verhalten oder Darstellung zu beeinflussen. Sie ermöglichen es, Daten von einer übergeordneten Komponente an eine untergeordnete Komponente weiterzugeben.

    Beispiel:

    <MyComponent title="Hello World" />

    In diesem Beispiel wird die Prop "title" mit dem Wert "Hello World" an die Komponente "MyComponent" übergeben.

  • Wie verwendet man Props in einem JSX code?

    • Props werden in JSX verwendet, indem sie als Attribute an Komponenten übergeben werden. Hier ist ein Beispiel:

    const MyComponent = (props) => {
    return <h1>{props.title}</h1>;
};

    In diesem Beispiel nimmt die Arrow Function "MyComponent" die Props als Argument entgegen und gibt ein <h1>-Element mit dem Titel zurück, der über die Prop "title" übergeben wurde.

    props ist ein Objekt, das alle Eigenschaften enthält, die der Komponente beim Aufruf übergeben werden.

    Beispiel:

    <MyComponent title="Hello World" />

    In diesem Beispiel wird die Prop "title" mit dem Wert "Hello World" an die Komponente "MyComponent" übergeben.

    Ergebnis im Browser:

    <h1>Hello World</h1>

    Fazit:

    • Props sind Parameter, die an Komponenten übergeben werden, um deren Verhalten oder Darstellung zu beeinflussen.
    • {props.title} liest den Wert aus der Prop "title" und gibt ihn in der Komponente aus.
    • JSX macht es möglich, HTML und JavaScript in einer Datei zu kombinieren, was die Entwicklung von Benutzeroberflächen erleichtert.

  • Was ist State?

    • State (Zustand) ist ein internes Datenobjekt einer Komponente, das den aktuellen Zustand der Komponente speichert. Änderungen am State führen zu einer Aktualisierung der Benutzeroberfläche.

    Beispiel:

    const [count, setCount] = useState(0);

    Hier wird der Zustand "count" initial auf 0 gesetzt und kann mit der Funktion "setCount" aktualisiert werden.

    Auf die Weise kann der Zustand einer Komponente dynamisch geändert werden, was zu einer Aktualisierung der Benutzeroberfläche führt.

  • Wie erstellt man ein Component ohne Props?

    • Ein Component ohne Props kann einfach als Funktion oder Klasse definiert werden, die JSX zurückgibt. Hier ist ein Beispiel für eine funktionale Komponente:

    const MyComponent = () => {
    return <div>Hello World</div>;
};

    In diesem Beispiel gibt die Komponente "MyComponent" einfach einen <div>-Block mit dem Text "Hello World" zurück. Es werden keine Props verwendet.

  • Wie fügt man className zu den JSX elementen?

    • In JSX wird das Attribut "className" verwendet, um CSS-Klassen zu einem Element hinzuzufügen. Hier ist ein Beispiel:

    <div className="my-class">Hello World</div>

    In diesem Beispiel wird die Klasse "my-class" dem <div>-Element hinzugefügt.

  • Kann man einen Text / Zahl als Props übergeben?

    • Ja, man kann sowohl Text (Strings) als auch Zahlen als Props an eine Komponente übergeben.

    Beispiel für Text:

    <MyComponent message="Hallo Welt" />

    Beispiel für Zahl:

    <MyComponent count={5} />

    In der Komponente kann man dann auf diese Props zugreifen:

    const MyComponent = (props) => {
    return (
        <div>
            <p>{props.message}</p>
            <p>{props.count}</p>
        </div>
    );
};

    So werden Text und Zahl als Props verwendet und angezeigt.

    Ergebnis im Browser:

    <div>
    <p>Hallo Welt</p>
    <p>5</p>
</div>

    Fazit:

    • Props können Text und Zahlen sein.
    • Props werden in der Komponente über das "props"-Objekt abgerufen.
    • JSX ermöglicht es, HTML-ähnliche Syntax in JavaScript zu verwenden, um Komponenten zu erstellen.

  • Kann man eine oder mehrere className als Props übergeben?

    • Ja, man kann eine oder mehrere className-Werte als Props an eine Komponente übergeben.

    Beispiel:

    In der Komponente kann man die Prop wie folgt verwenden:

    const MyComponent = (props) => {
    return <div className={props.className}>Hello World</div>;
};
    <MyComponent className="my-class another-class" />

    So kann man beliebige CSS-Klassen als String (mit Leerzeichen getrennt) übergeben und anwenden.

    Ergebnis im Browser:

    <div class="my-class another-class">Hello World</div>

    In diesem Beispiel wird die Klasse "my-class another-class" dem <div>-Element hinzugefügt.

    Die Komponente verwendet dann className={props.className} im JSX, um die übergebenen Klassen anzuwenden.

    Fazit:

    • className kann als Prop übergeben werden.
    • Mehrere Klassen werden als String mit Leerzeichen getrennt übergeben.
    • Die Komponente verwendet dann className={props.className} im JSX.

  • Welche Daten kann man als Props übergeben?

    • Als Props können verschiedene Datentypen an eine Komponente übergeben werden, darunter:

    • Strings (Text): <MyComponent title="Hallo" />
    • Zahlen: <MyComponent count={5} />
    • Booleans: <MyComponent isActive={true} />
    • Arrays: <MyComponent items={[1, 2, 3]} />
    • Objekte: <MyComponent user={ name: "Max", age: 25 } />
    • Funktionen: <MyComponent onClick={handleClick} />
    • React-Elemente/Komponenten: <MyComponent header={<Header />} />

    Props sind flexibel und können beliebige Werte enthalten, die in der Komponente verwendet werden sollen.

  • Kann man ein EventHandler als Props übergeben?

    • Ja, man kann einen EventHandler (z.B. eine Funktion für onClick) als Prop an eine Komponente übergeben.

    Beispiel:

    const MyButton = (props) => {
    return <button onClick={props.onClick}>Klick mich!</button>;
};

// Verwendung:
const handleClick = () => {
    alert('Button wurde geklickt!');
};

<MyButton onClick={handleClick} />

    In diesem Beispiel wird die Funktion handleClick als Prop onClick an die Komponente MyButton übergeben. Beim Klick auf den Button wird die Funktion ausgeführt.

    Fazit:

    • EventHandler wie onClick können als Props übergeben werden.
    • Die Komponente ruft die übergebene Funktion bei einem Event auf.

  • Kann man einer Komponente einen Event-Handler als Prop übergeben, der beim Aufruf zusätzliche Parameter enthält?

    • Ja, das ist möglich. Wenn man einer Komponente einen Event-Handler mit zusätzlichen Parametern übergeben möchte, kann man eine anonyme Funktion (Arrow Function) verwenden, um die gewünschten Parameter zu übergeben.

    Beispiel:

    const MyButton = (props) => {
        return <button onClick={() => props.onClick('Hallo', 42)}>Klick mich!</button>;
    };

    // Verwendung:
    const handleClick = (text, number) => {
        alert(`Text: ${text}, Zahl: ${number}`);
    };

    <MyButton onClick={handleClick} />

    In diesem Beispiel wird beim Klick auf den Button die Funktion handleClick mit den Parametern 'Hallo' und 42 aufgerufen.

    Die anonyme Funktion in onClick={() => props.onClick('Hallo', 42)} ermöglicht es, zusätzliche Argumente an den Event-Handler zu übergeben.

    Ergebnis im Browser:

    <button onClick="handleClick('Hallo', 42)">Klick mich!</button>

    Wenn der Button geklickt wird, erscheint ein Alert mit dem Text "Text: Hallo, Zahl: 42".

    Fazit:

    • Man kann eine anonyme Funktion verwenden, um zusätzliche Parameter an einen Event-Handler zu übergeben.
    • Der Event-Handler in der Elternkomponente erhält die gewünschten Werte als Argumente.

  • Wie schreibt man eine Schleife in JSX?

    • In JSX kann man keine klassische for-Schleife direkt im Markup schreiben. Stattdessen verwendet man Methoden wie map(), um über Arrays zu iterieren und Elemente zu rendern.

    Beispiel:

    const items = ['Apfel', 'Banane', 'Orange'];

            return (
                <ul>
                    {items.map((item, index) => (
                        <li key={index}>{item}</li>
                    ))}
                </ul>
            );

    In diesem Beispiel wird das Array items mit map() durchlaufen und für jedes Element ein <li>-Element erzeugt. Der key-Prop hilft React, die Elemente effizient zu verwalten.

    Fazit:

    • Für Schleifen in JSX verwendet man meist map() auf Arrays.
    • Jedes erzeugte Element sollte einen eindeutigen key erhalten.

    Ergebnis im Browser:

    <ul>
    <li>Apfel</li>
    <li>Banane</li>
    <li>Orange</li>
</ul>

    Dies erzeugt eine ungeordnete Liste mit den Früchten.

    Beispiel mit Array.from:

    const items = Array.from({ length: 5 }, (_, index) => `Item ${index + 1}`);
            return (
                <ul>
                    {items.map((item, index) => (
                        <li key={index}>{item}</li>
                    ))}
                </ul>
            );

    In diesem Beispiel wird ein Array mit 5 Elementen erstellt, die von "Item 1" bis "Item 5" reichen. Dann wird es mit map() in eine Liste umgewandelt.

    Ergebnis im Browser:

    <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
    <li>Item 4</li>
    <li>Item 5</li>
</ul>

    Dies erzeugt eine ungeordnete Liste mit den Items von 1 bis 5.

  • Wie schreibt man eine if-Bedingung in JSX?

    • In JSX kann man keine klassische if-Anweisung direkt im Markup verwenden. Stattdessen nutzt man den ternären Operator (?), logisches UND (&&) oder schreibt die Bedingung vor dem return im Funktionskörper.

    Beispiel mit ternärem Operator:

    const isLoggedIn = true;

            return (
                <div>
                    {isLoggedIn ? <p>Willkommen zurück!</p> : <p>Bitte einloggen.</p>}
                </div>
            );

    Hier wird je nach Wert von isLoggedIn ein anderer Text angezeigt.

    Beispiel mit logischem UND:

    const showMessage = true;

            return (
                <div>
                    {showMessage && <p>Nachricht sichtbar</p>}
                </div>
            );

    Das Element wird nur angezeigt, wenn showMessage true ist.

    Beispiel: Bedingung vor dem return

    let content;
            if (isLoggedIn) {
                content = <p>Willkommen zurück!</p>;
            } else {
                content = <p>Bitte einloggen.</p>;
            }

            return (
                <div>
                    {content}
                </div>
            );

    Fazit:

    • Für Bedingungen in JSX verwendet man meist den ternären Operator oder logisches UND.
    • Klassische if-Anweisungen schreibt man außerhalb des JSX.
