Full-Stack (Node JS Basics)

Node JS Projeckt erstellen:

  • Um ein neues Node.js-Projekt zu erstellen, öffne das Terminal und navigiere zu dem Verzeichnis, in dem du das Projekt erstellen möchtest. Verwende dann den Befehl npm init, um eine neue package.json-Datei zu erstellen. Folge den Anweisungen im Terminal, um die erforderlichen Informationen einzugeben.

  • Alternativ kannst du den Befehl npm init -y verwenden, um die Standardwerte automatisch zu übernehmen und eine package.json-Datei mit den Standardeinstellungen zu erstellen.

  • Beim 'type' darauf achten, ob 'commonjs' oder 'module' gewünscht ist.

  • Wenn der package.json File erstellt ist, beim 'scripts' hinzufügen -> "dev": "projectName.js"

  • Um das Projekt zu starten, verwende den Befehl npm run dev im Terminal. Dadurch wird das Skript mit dem Namen "dev" ausgeführt, das in der package.json-Datei definiert ist.

  • Um ein neues Paket zu installieren, verwende den Befehl npm install <paketname>. Dadurch wird das gewünschte Paket heruntergeladen und als Abhängigkeit im Projekt gespeichert. Die Information über das installierte Paket wird automatisch in der package.json und package-lock.json aktualisiert.

  • Um ein Paket zu entfernen, das du nicht mehr benötigst, verwende den Befehl npm uninstall <paketname>. Dadurch wird das Paket aus dem Projekt entfernt und die package.json sowie package-lock.json werden entsprechend aktualisiert.

  • Um alle installierten Pakete auf die in der package.json erlaubten Versionen zu aktualisieren, verwende den Befehl npm update.

Full-Stack (Node JS Basics)

Recherche (Node JS):

  • Was ist ein JS Engine?

    • Eine JS Engine ist ein Programm, das JavaScript-Code interpretiert und ausführt. Sie wandelt den Quellcode in Maschinencode um, den der Computer verstehen kann.

  • Was ist Node JS?

    • Node.js ist eine JavaScript-Laufzeitumgebung, die auf der V8-Engine von Google Chrome basiert. Sie ermöglicht es, JavaScript außerhalb des Browsers auszuführen, was bedeutet, dass JavaScript auch auf Servern verwendet werden kann.

  • Was ist der Unterschied zwischen JS Engine in einem Webbrowser und Node.JS?

    • Die JS Engine in einem Webbrowser ist darauf ausgelegt, JavaScript-Code auszuführen, der in Webseiten eingebettet ist. Node.js hingegen ist eine serverseitige Umgebung, die es ermöglicht, JavaScript-Code auf dem Server auszuführen und auf Systemressourcen zuzugreifen.

  • Was funktioniert NICHT in Node.JS?

    • Node.js unterstützt keine DOM-Operationen, da es keine grafische Benutzeroberfläche hat. Funktionen, die auf den Browser angewiesen sind, wie z.B. das Manipulieren von HTML-Elementen oder das Verwenden von Web APIs, funktionieren nicht in Node.js.

  • Wofür steht npm?

    • npm steht für Node Package Manager. Es ist ein Paketmanager für JavaScript, der es Entwicklern ermöglicht, Pakete und Bibliotheken zu installieren, zu verwalten und zu teilen.

  • Was bedeutet die Abkürzung CLI?

    • CLI steht für Command Line Interface. Es ist eine textbasierte Benutzeroberfläche, die es Benutzern ermöglicht, mit dem Computer über Befehle zu interagieren, anstatt über grafische Benutzeroberflächen.

  • Was ist ein Node.JS Project?

    • Ein Node.js-Projekt ist eine Sammlung von Dateien und Ordnern, die zusammenarbeiten, um eine Anwendung oder ein Skript zu erstellen, das mit Node.js ausgeführt werden kann. Es enthält in der Regel eine package.json-Datei, die Informationen über das Projekt und seine Abhängigkeiten enthält.

  • Was ist eine package.json?

    • Die package.json-Datei ist eine JSON-Datei, die Informationen über ein Node.js-Projekt enthält, einschließlich des Projektnamens, der Version, der Abhängigkeiten und der Skripte. Sie wird verwendet, um das Projekt zu konfigurieren und Abhängigkeiten zu verwalten.

  • Wie erstellt man ein neues Projekt in Node JS?

    • Um ein neues Projekt in Node.js zu erstellen, verwendet man den Befehl npm init im Terminal. Dadurch wird eine package.json-Datei erstellt, in der die wichtigsten Informationen zum Projekt gespeichert werden. Optional kann man auch npm init -y verwenden, um die Standardwerte automatisch zu übernehmen.

  • Wie startet man ein Node JS Projekt?

    • Um ein Node.js-Projekt zu starten, verwendet man in der Regel den Befehl node <dateiname>.js im Terminal, wobei <dateiname>.js die Hauptdatei des Projekts ist (z.B. index.js). Alternativ kann man, wenn in der package.json ein Startskript definiert ist, auch npm start verwenden.

  • Was macht der Befehl npm run dev?

    • Der Befehl npm run dev führt das Skript mit dem Namen "dev" aus, das in der package.json-Datei definiert ist. Dies wird häufig verwendet, um Entwicklungsserver zu starten oder Entwicklungsumgebungen einzurichten.

  • Was sind Node Packeges (dependency)?

    • Node-Pakete sind Sammlungen von Code, die in Node.js-Projekten verwendet werden können. Sie werden in der Regel über npm installiert und können Bibliotheken, Frameworks oder andere nützliche Funktionen enthalten, die die Entwicklung erleichtern.

  • Was sind Kriterien für den Auswahl der Node Packages?

    • Bei der Auswahl von Node Packages sollte man auf folgende Kriterien achten:

    • Aktualität und Wartung des Pakets (letztes Update, aktive Community)
    • Beliebtheit und Anzahl der Downloads
    • Dokumentation und Benutzerfreundlichkeit
    • Kompatibilität mit dem eigenen Projekt (Versionen, Abhängigkeiten)
    • Sicherheitsaspekte (bekannte Schwachstellen, Vertrauenswürdigkeit)
    • Lizenz des Pakets

  • Wie installiert man neue Packages?

    • Um neue Pakete in einem Node.js-Projekt zu installieren, verwendet man den Befehl npm install <paketname> im Terminal. Dadurch wird das gewünschte Paket heruntergeladen und als Abhängigkeit im Projekt gespeichert. Die Information über das installierte Paket wird automatisch in der package.json und package-lock.json aktualisiert.

  • Wie aktualisiert man bestehende Packages?

    • Um bestehende Pakete zu aktualisieren, verwendet man den Befehl npm update im Terminal. Dadurch werden alle installierten Pakete auf die in der package.json erlaubten Versionen aktualisiert. Möchte man ein bestimmtes Paket aktualisieren, kann man npm install <paketname>@latest verwenden, um die neueste Version dieses Pakets zu installieren.

  • Wie entfernt man Packages, den man nicht mehr braucht?

    • Um Pakete zu entfernen, die man nicht mehr benötigt, verwendet man den Befehl npm uninstall <paketname> im Terminal. Dadurch wird das Paket aus dem Projekt entfernt und die package.json sowie package-lock.json werden entsprechend aktualisiert.

  • Was muss in .gitignore file stehen?

    • In der .gitignore-Datei sollte man Dateien und Ordner auflisten, die nicht ins Git-Repository aufgenommen werden sollen. Typische Beispiele in Node.js-Projekten sind:

    • node_modules/ – enthält installierte Abhängigkeiten, die jederzeit neu installiert werden können
    • .env – enthält sensible Umgebungsvariablen wie Passwörter oder API-Keys
    • dist/ oder build/ – generierte Build-Ordner
    • *.log – Logdateien

  • Wie importiert man eine Funktion / Variable aus einem anderen Skript?

    • Um eine Funktion oder Variable aus einem anderen Skript zu importieren, verwendet man in Node.js das require-Statement (CommonJS) oder das import-Statement (ES Module):

    • CommonJS (älter, Standard in Node.js):
      // In math.js
      module.exports = { add: (a, b) => a + b };
      // In main.js
      const math = require('./math');
      console.log(math.add(2, 3));
    • ES Module (moderner, mit type": "module" in package.json):
      // In math.js
      export function add(a, b) { return a + b; }
      // In main.js
      import { add } from './math.js';
      console.log(add(2, 3));
