Was ist ein Date-Objekt in JavaScript?

Ein Date-Objekt in JavaScript ist eine spezielle Art von Datenstruktur, die Datum und Uhrzeit repräsentiert.
Es speichert die Zeit in Millisekunden seit dem 1. Januar 1970 (UTC) – auch bekannt als Unix-Zeitstempel
oder "Epoch Time".

Warum gibt es das Date-Objekt?

JavaScript wurde ursprünglich für Webentwicklung konzipiert. Da Websites oft mit Datum und Zeit arbeiten
müssen (zum Beispiel für Kalender, Timer, Buchungssysteme oder Ablaufdaten), wurde das Date-Objekt
eingeführt, um diese Funktionen bereitzustellen.

Wie funktioniert das Date-Objekt?

Das Date-Objekt speichert nicht einfach nur ein Datum wie "3. April 2024", sondern eine genaue
Zeitangabe bis auf die Millisekunde.

Intern betrachtet JavaScript die Zeit als die Anzahl der Millisekunden seit Mitternacht am 1. Januar 1970
(UTC-Zeit). Zum Beispiel:

  • Der Wert 0 steht für den 1. Januar 1970, 00:00:00 Uhr UTC.
  • Der Wert 1.000 steht für den 1. Januar 1970, 00:00:01 Uhr UTC (eine Sekunde später).
  • Der Wert 86.400.000 steht für den 2. Januar 1970, 00:00:00 Uhr UTC (ein Tag später).

Das bedeutet: Jedes Datum, das du siehst, ist eigentlich eine Berechnung basierend auf dieser Zeitreferenz.

Wie speichert JavaScript Datum und Zeit?

Das Date-Objekt speichert intern die absolute Zeit als Zahl, aber wenn es ausgegeben wird, wandelt
JavaScript diese Zahl in ein menschenlesbares Format um – z. B. "Wed Apr 03 2024 12:00:00 GMT+0200".

Wichtige Komponenten eines Datums:

  1. Jahr (z. B. 2024)
  2. Monat (Achtung: JavaScript zählt ab 0, also Januar = 0, Dezember = 11)
  3. Tag des Monats (1 bis 31)
  4. Wochentag (0 = Sonntag, 6 = Samstag)
  5. Stunde, Minute, Sekunde & Millisekunde

Das Date-Objekt kann alle diese Werte abrufen oder verändern.

Zeitzonen und UTC

Ein großes Thema beim Arbeiten mit Datum und Uhrzeit sind Zeitzonen.

  • Die meisten Methoden von Date arbeiten mit der lokalen Zeit deines Computers.
  • Es gibt aber auch Methoden, um die Zeit in UTC (Universal Coordinated Time) zu berechnen, die für
    internationale Systeme wichtig ist.

Beispiel:

  • Wenn dein Computer in Deutschland (Zeitzone GMT+2) steht und du die aktuelle Zeit abrufst,
    zeigt JavaScript die Uhrzeit basierend auf dieser Zeitzone an.
  • Wenn du jedoch die UTC-Zeit abrufst, zeigt JavaScript eine um zwei Stunden frühere Zeit an.

Das ist besonders wichtig, wenn man z. B. eine weltweite Anwendung baut, die mit Servern
kommuniziert, die in unterschiedlichen Zeitzonen stehen.

Warum ist Date manchmal kompliziert?

  1. Monate starten bei 0, nicht bei 1 → Januar = 0, Februar = 1 …
  2. Zeitzonen beeinflussen Ergebnisse → Die gleiche Uhrzeit kann in verschiedenen Ländern anders
    angezeigt werden.
  3. Schaltjahre & ungleichmäßige Monatslängen → Februar hat manchmal 28, manchmal 29 Tage.
  4. Sommerzeit (Daylight Saving Time, DST) → Manchmal springt die Uhr eine Stunde vor oder zurück, was
    Berechnungen schwierig machen kann.

Möglichkeiten, ein Datum in JavaScript anzuzeigen
