JS Engine

Stell dir vor, du schreibst einen Text auf Deutsch (dein JavaScript-Code).
Aber der Computer versteht nur "Maschinensprache" .
👉 Die JavaScript Engine ist der Übersetzer.
Sie nimmt deinen Code und übersetzt ihn in eine Sprache, die der Computer versteht, und führt ihn dann aus.

Unterschied zwischen JS-Engine im Browser und Node.js

💻 im Browser:

  • Wird verwendet, um Webseiten interaktiv zu machen.
  • Läuft im Browser
  • Hat Zugriff auf HTML & CSS über document, window, usw.
  • Typische Dinge: Buttons klicken, Inhalte anzeigen, Animationen.

📜 in Node.js

  • Wird verwendet, um außerhalb vom Browser zu programmieren.
  • Läuft auf deinem Computer oder Server.
  • Kein Zugriff auf document oder window, dafür aber auf Dateien, Netzwerk, Server usw.
  • Typische Dinge: Server bauen, Dateien lesen/schreiben, Befehle ausführen.

🧠 Merksatz:

Im Browser: JavaScript spricht mit der Webseite.
In Node.js: JavaScript spricht mit dem Computer.

Dinge die in Node.js NICHT funktionieren

Wenn du JavaScript in Node.js verwendest, hast du keine Webseite – also funktioniert alles nicht, was direkt mit HTML, CSS oder dem Browser zu tun hat.

  • Kein DOM (Document Object Model)
  • Kein window-Objekt
  • Kein Zugriff auf HTML oder CSS
  • Keine direkten Animationen, kein Canvas, kein WebGL

🧠 Merksatz:

Node.js kann kein HTML anzeigen, keine Webseiten manipulieren und keine Fenster öffnen.

Es arbeitet ohne Browser – also auch ohne DOM, ohne CSS, ohne visuelle Dinge.

npm - Node Package Manager

  • „Package“ = ein Stück fertiger Code (z. B. ein Modul oder eine Bibliothek)
  • „Manager“ = ein Programm, das dir hilft, diesen Code zu installieren und zu verwalten

npm hilft dir, fertige Bausteine (Pakete) für deine Projekte zu benutzen.

Was macht npm

  1. Pakete initialisieren

    2. ➡️ npm lädt den Code aus dem Internet runter

    ➡️ und speichert ihn in deinem Projekt im Ordner node_modules.


  2. Pakete verwalten:

    3. Alle installierten Pakete werden in der Datei package.json gespeichert.

    Diese Datei merkt sich:

    • welche Pakete du nutzt
    • in welcher Version
    • und ob andere sie mitinstallieren können

  3. Eigene Projekte starten

    4. Du kannst mit npm init dein eigenes Projekt registrieren – dann bekommst du automatisch eine package.json.

  4. Skripte ausführen

🧠 Merksatz:

npm ist dein Werkzeugkasten für Node.js – es holt dir fertige Bausteine, damit du nicht alles selbst bauen musst.

CLI - Command Line Interface

Eine CLI ist ein Text-basiertes Interface, über das du Befehle direkt eingeben kannst, um Programme auszuführen oder mit dem Betriebssystem zu arbeiten.

Node.js-Project

Ein Node.js-Projekt ist ein JavaScript-Projekt, das nicht im Browser läuft, sondern direkt auf deinem Computer oder Server – mit Hilfe von Node.js.

package.jason

Die package.json ist das Herzstück eines Node.js-Projekts. Sie beschreibt dein Projekt und listet alle Abhängigkeiten (Pakete/Module) auf.
