TailwindCSS

das Problem mit Plain CSS...

  • Keine Variablen (bis CSS3)
  • Kein Scoping / Globale Regeln
  • Kein Logik / Bedingungen
  • Kein Wiederverwenden von Regeln
  • schlechte Wartbarkeit bei großen Projekten
  • Kein automatisches Modulieren

TailwindCSS

TailwindCSS löst die oben genantem probleme:

  • Keine benutzerdefinierten Klassen nötig – alles ist inline im HTML. Dadurch entstehen keine Konflikte zwischen verschiedenen Styles.
  • Utility-Klassen wie py-4, text-center, shadow-lg sind überall wiederverwendbar – keine neue Klasse nötig.
  • Tailwind liefert ein konfigurierbares Designsystem out of the box
  • Du stylst direkt im HTML/JSX mit Utilities – das spart Dateien und vermeidet „verweiste“ Klassen.
  • Eingebaute Responsive Prefixes & Dark Mode Unterstützung
  • Du kannst schnell Layouts, Buttons, Cards usw. erstellen mit Klassen

Voraussetzungen um TailwindCSS zuverwenden

  • Node.js und npm installiert
  • Tailwindcss installiert

so wird TailwindCSS im Terminal installiert

npm install tailwindcss @tailwindcss/vite
