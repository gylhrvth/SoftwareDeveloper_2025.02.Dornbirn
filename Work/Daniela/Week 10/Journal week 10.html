Use Cases für Asynchronität in JavaScript

1. Daten von einem Server laden (API-Calls)

Beispiel: fetch() ruft User-Daten von einer API ab.

Vorteil: Kein Einfrieren der Seite beim Warten auf Antwort.


2. Nutzeraktionen verarbeiten ohne UI zu blockieren

Beispiel: Beim Klick auf "Speichern" wird im Hintergrund gespeichert.

Vorteil: Der Nutzer kann währenddessen weiterarbeiten.


3. Verzögerungen (Timer, Ladeanzeigen, Animationen)

Beispiel: setTimeout() für „Bitte warten…“ oder Übergänge.

Vorteil: Kontrolle über Wartezeiten und Feedback.


4. Dateien hochladen (z. B. Bilder, PDFs)

Beispiel: fetch() mit FormData, um eine Datei an den Server zu schicken.

Vorteil: Ladeanzeige oder Fortschrittsanzeige möglich.


5. Stapelweise Aufgaben abarbeiten (Reihenfolge einhalten)

Beispiel: Mehrere Bilder nacheinander hochladen mit for...await.

Vorteil: Saubere Abfolge bei mehreren async Tasks.


6. Echtzeit-Kommunikation (Chats, Multiplayer, WebSockets)

Beispiel: WebSocket für Live-Updates im Chat.

Vorteil: Keine manuellen Abfragen nötig, Realtime-Kommunikation.


7. Browser APIs nutzen (GPS, Kamera, Dateien, etc.)

Beispiel: navigator.geolocation.getCurrentPosition() für Standort.

Vorteil: Zugriff auf externe Ressourcen ohne Unterbrechung des Codes.
