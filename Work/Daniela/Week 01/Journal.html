    Typografie

    Typografie ist die Kunst und Technik der Gestaltung von Schrift. Sie befasst sich mit der Auswahl, Anordnung und dem Design von Buchstaben, Wörtern und Texten, um Lesbarkeit, Ästhetik und ihrer visuelle Hierarchie zu optmieren.

    Schriftgrössen sind essenziell um die Hierarchie der Texte zu bestimmen. Aus diesem Grund werden z.B. für Überschriften grössere Drstellungen gewählt.
    Die Schriftart beeinflusst massgeblich die Lesbarkeit eines Textes. So werden für Texte auf Bildschirmen "Sans-Serif-Schriften" gerne genutzt da sie glatte enden haben, was es ermöglicht Texte gut und einfach zu lesen. Ebenso verleihen "Sans-Serif-Schriften" den Texten einen schlichten und modernen Look.
    Im gegensatz dazu gibt es die "Serif-Schriften" welche sich durch Verzierungen oder kleinen Füsschen am ende des Buchstaben bemerkbar machen. Diese Wiederspiegeln einen klassischen und traditionellen Look und werden gerne bei Printmedien verwendet.
    Neben der Funktionalität können Schriftarten aber auch einen Wiedererkennungswert bewirken. Die Wahl der Schrift kann für eine Marke neben Informationen auch Emotionen und Identität vermittel. Wer hat nicht schon einmal wirbelnde weiße Buchstaben auf rotem Grund gesehen und sofort an eine erfrischendes Cola gedacht?Die legendäre Schrift von Coca-Cola, die Spencerian, ist der Inbegriff für emotionales Branding und weckt überall Erinnerungen und Gefühle.
    Ebenso ist es wichtig auf den Zeilenabstand, den Contrasts des Hintergrundes zur Schrift und Buchstabenabstand zuachten, denn auch diese ELemente wirken sich auf die Lesbarkeit aus.

    Es ist wichtig dem Nutzer ein angenehme und einfache Leseerfahrung zu gewährleisten.

    Die WCAG-Richtlinien legen fest, dass Webdesigner ihre Typografie so gestalten müssen, dass sie für Menschen mit Sehbehinderungen, Farbfehlsichtigkeit oder anderen Einschränkungen gut lesbar ist. Wird sich an diese Richtlinie gehalten so kann eine Barrierefreiheit sichergestellt werden.

    Punkt 3: Persona

    Was ist eine Persona?

    Eine persona ist eine fiktive aber auf Daten gestützte Darstellung eines Nutzers oder Kunden. Durch Gespräche mit echten Kunden wird ein konktetes Profil mit realistischer biography erstellt. Ebenso fliessen auch Informationen aus einem Datenstamm ein. Eine Persona besteht aus einem Individum sowie einem Durchschnittswert der Zielgruppe.

    Der Grundgedanke hinter der Persona ist es, anstatt mit einer abstrackten vorstellung einer Zielgruppe zu arbeiten, wird mit einem konkretem prototypische Nutzer gearbeitet.

    Eine Persona basiert auf echten Daten und Forschungsergebnissen, die oft durch Umfragen, Interviews,
    Datenanalyse und Nutzerforschung gesammelt werden. Eine Persona umfasst typischerweise folgende Elemente:

    • Name und Hintergrund
    • Ziele und Bedürfnisse
    • Technische Vorlieben
    • ebenso können auch Zitat des Kunden eingefügt werden:

    z.B.:"Ich brauche eine schnelle, benutzerfreundliche Lösung, die mir hilft, meine Arbeit zu gestalten."

    Was ist der Unterschied zwischen einer guten und einer schlecht erstellten Persona?

    Eine gut erstellte Persona ist ausführlich beschrieben. Neben dem Namen, der altersgruppe und dem Wohnort, können auch Interessen, Bedürfnisse und Vorlieben erfasst werden.

    Eine gut erstellte Persona wiederspiegelt die Anforderugen und Bedürfnisse der Zielgruppe/Einzelperson.

    Eine Schlechte erstellte Persona ist nur vage ausgeführt. Es kann sein das einige Daten nicht erhoben wurden oder andere sehr allgemein gehalten sind. Eine schlechte Persona gibt käum aufschluss, welche Zielgruppe erreichtwerden soll und welche Bedürfnisse diese Zielgruppe hat.

    Punkt 2: Git

    Git ist ein VCS (Version-Controll-System).
    Ein Versionscontrollsystem wird verwendet um den Verlauf von Änderungen an einer Datei,
    nach verfolgen zu können. Es ist nützlich um Software-Projekte zu verwalten,
    bei denen mehrere Entwickler Veränderungen vornehmen.


  • Git Commit:

    • Speichert Änderungen mit einer Nachricht zur Dokumentation.

  • Git Repository

    • Speicherort für den Code

    Unterschied zwischen einem lokalen und einem remote Repository?


  • lokal
    • ist eine Kopie des Speicher Ortes für den Code, die auf dem eigenem Computer gespeichert ist.

  • remote
    • ist eine Kopie des Speicher Ortes für den Code, welcher auf einem entferntem Server
    oder einer Platform wie Git gespeichert ist.

    Was ist ein Merg?


    Ein Merge ist der Prozess, bei dem Änderungen aus einem Zweig in einen anderen zusammengeführt werden.
    Dies wird häufig verwendet, wenn du in einem separaten Branch arbeitest (z. B. für ein Feature)
    und deine Änderungen wieder in den Haupt-Branch (normalerweise main oder master) integrieren möchtest.


    Merge-Konflikte:


    Manchmal können beim Merge Konflikte auftreten, wenn dieselben Teile einer Datei in beiden Branches
    unterschiedlich verändert wurden. In diesem Fall kannst du den Konflikt manuell lösen.

    Beispiel für einen Merge-Konflikt:

    Du und ein Teammitglied habt beide an derselben Datei gearbeitet und unterschiedliche Änderungen vorgenommen.
    Git erkennt, dass es nicht automatisch entscheiden kann, welche Version behalten werden soll.

    Nutzen:

    Versioncontrol

    ist ein Werkzeug, es ermöglicht die Verwaltung von Codes und Dateien.
    ebenso bietet es folgende Vorteile:

  • Verfolgung von Änderungen und Historie

  • Zusammenarbeit und Teamarbeit

  • Fehlerbehebung und Rollback

  • Backup und Sicherheit

  • Experimentieren ohne Risiko

  • Nachvollziehbarkeit und Dokumentation

  • Arbeiten in mehreren Versionen parallel (Branching)

    • Schwierigkeiten bei erstellung des VCS:

    Ein Versionskontrollsystem zu entwickeln ist komplex, weil es viele technische
    und organisatorische Anforderungen erfüllen muss.Schwierigkeiten die sich ergeben sind:

  • Datenstruktur & Speicherung
  • Performance & Skalierbarkeit
  • Konfliktmanagement & Zusammenführung (Merging)
  • Netzwerk & verteilte Systeme (bei verteilten VCS wie Git)
  • Benutzerfreundlichkeit & Komplexität
  • Sicherheit & Zugriffskontrolle
  • Backup & Wiederherstellung

    Die Command Line Interface ist die Komandozeilen-Oberfläche von Git,mit der man
    Git-Befehle direkt in einem Terminal oder einer Eingabeaufforderung ausführen kann.

  • alle Git Funktionen von Git Vorhanden
  • Viele Befehle können schneller über die CLI ausgeführt werden

    • GitHub:

    GitHub ist eine webbasierte Plattform für Versionskontrolle und kollaborative
    Softwareentwicklung. Sie basiert auf Git und bietet eine zentrale Umgebung für
    Entwickler, um gemeinsam an Code zu arbeiten, Änderungen zu verwalten und
    Projekte zu veröffentlichen.

    GitHub Pages:


    GitHub Pages ist ein kostenloser Hosting-Service von GitHub, mit dem du statische
    Websites direkt aus einem GitHub-Repository veröffentlichen kannst. Damit kannst
    du z. B. eine persönliche Webseite, eine Projekt-Dokumentation oder einen Blog
    hosten – ohne einen separaten Webserver einrichten zu müssen.

    Was ist der Unterschied zwischen Befehle im Terminal oder Schritte in Visual Studio Code?


    Nutzung von Git im Terminal (CLI)

    bietet mehr Kontrolle, ist schneller und ideal für
    erfahrene Nutzer sowie Automatisierungen, erfordert jedoch das Lernen von Befehlen.

    Studio Code (GUI)

    ist hingegen einsteigerfreundlicher, visuell intuitiv und ermöglicht
    die Nutzung von Git per Mausklick, bietet jedoch weniger Funktionen als die CLI.

  • kein Merge konflikt vorliegt !
  • und ein Befehl wie: "git merge feature-x" einprogramiert wurde

    • Qalitätsmerkmale:

    Typische Fehler von Junior-Programmiere:


    Typische Probleme von Junior Entwicklern können durch verschiedene Faktoren verursacht werden, wie zum Beispiel fehlende Erfahrung,
    unzureichendes Verständnis für Best Practices oder das Fehlen einer klaren Arbeitsstruktur.

    Für Junior Entwickler ist es wichtig, Klarheit und Struktur zu entwickeln, Fragen zu stellen, Fehler zu akzeptieren und Lernprozesse
    aktiv zu gestalten. Mentoring, Code-Reviews und die ständige Verbesserung der eigenen Fähigkeiten helfen dabei, die typischen
    Herausforderungen erfolgreich zu meistern.


    Git CLI Befehle:


    Git add:

    Der Befehl git add ist ein wichtiger Schritt, um Änderungen in deiner Arbeitskopie in den Commit-Prozess von Git aufzunehmen.
    Erst nach git add werden die Änderungen im nächsten Commit gespeichert.

    Git commit:

    Ein Commit in Git speichert alle Änderungen, die du in der Staging Area vorbereitet hast, mit einer Commit-Nachricht,
    die beschreibt, was geändert wurde. Commits sind entscheidend für die Versionierung und den Verlauf deines Projekts und
    ermöglichen es dir, jederzeit auf frühere Versionen zurückzugreifen.

    Git pull:

    git pull ist der Befehl, den du verwendest, um Änderungen von einem Remote-Repository herunterzuladen und in deinen
    lokalen Branch zu integrieren

    Git push:

    Der Befehl git push wird verwendet, um deine lokalen Änderungen (Commits) zu einem Remote-Repository hochzuladen.
    Es sorgt dafür, dass deine Arbeit für andere Entwickler zugänglich ist und sichert deine Änderungen auf einem
    Remote-Server. Du kannst git push für den Standard-Branch verwenden oder einen bestimmten Remote und Branch
    angeben, um die Änderungen gezielt zu übertragen.

    Git init:

  • erstellen eines neuen Repository:

    • Wenn du ein neues Projekt startest oder ein bestehendes Projekt unter Git versionieren möchtest,
    ist git init der erste Schritt, um das Projekt unter Versionskontrolle zu stellen.

  • Einrichten eines lokalen Repositories:

    • Du kannst git init verwenden, um ein Projekt lokal zu verwalten, bevor du es mit einem Remote-Repository
    (z. B. auf GitHub oder GitLab) verbindest.

    Git clone:

    git clone wird verwendet, um eine vollständige Kopie eines bestehenden Git-Repositories von einem Remote-Server
    auf deinen lokalen Computer zu kopieren. Es lädt alle Dateien, Branches und die gesamte Historie des Projekts herunter,
    sodass du sofort mit der Arbeit am Projekt beginnen kannst.

    Begriffe UI/UX Definition:

    UI: User Interface: beschreibt wie das Produkt aussieht
    UX: User Experience: beschreibt die Benutzer erfahrung
    d.h. wie mit dem Produkt agiert wird bzw. wie gut es funktioniert und es sich anfühlt


    Wie sieht ein UI/UX design aus?:

    UX- und UI-Design auf einen Blick: Das UX-Design konzentriert sich mehr auf die allgemeine Benutzererfahrung.
    UI-Design konzentriert sich mehr darauf, wie verschiedene Elemente wie Schaltflächen, Text und Grafiken aussehen.

  • ungeordnete Struktur
