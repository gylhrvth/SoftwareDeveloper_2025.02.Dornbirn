    Kurzer Ausflug - GIT


    Qualitätsmerkmale

    Wenn du ein Version Control System (VCS)wie Git richtig verwendest, gibt es mehrere Anzeichen, die
    dir bestätigen können, dass du korrekt einsetzt:


    1. Verfolgen von Änderungen


  • alle Änderungen, die du einem Code vornimmst, im Verlauf einsehen. Jede Änderungen sollte klar dokumentiert sein

  • überprüfen, regelmäßig und sinnvolle Commits machen

    • 2. Teamarbeit


  • arbeitest problemlos mit anderen zusammen, z.B. durch Pull Requests oder Merge Requests

  • alle Änderungen sollten gut dokumentiert und koordiniert sein

  • Der Workflow sollte reibungslos sein, d.h., klare Prozesse zum Pushen

    • Zusammengefasst: größte Herausforderung für Junior-Entwickler ist oft, sich in die gewohnte Praxis
    der Softwareentwicklung einzuarbeiten, mit Tools umzugehen und dabei kontinuierlich zu lernen.
    Ein unterstützendes Team, regelmäßiges Feedback sind entscheiden, um diese Schwierigkeiten zu überwinden.


    Definition und Begriffe

    Was ist eine VCS?

  • ist eine Software, die dazu dient, Änderungen an Dateien, insbesondere Quellcode, zu verwalten und nachvollziehen.
    Es ermöglicht mehreren Entwicklern, gleichzeitig an einem Projekt zu arbeiten, ohne die Gefahr einzugehen,
    dass Änderungen verloren gehen oder überschrieben werden.

    • Was ist ein Git Commit?

  • es ist ein Momentaufnahme des aktuellen Stands eines Projekts. Die Dateien werden dauerhaft gespeichert
    in dem Git Repository.

    • Was ist ein Git Repository?

  • ist ein Speicherort, in dem der komplette Quellcode eines Projekts, der gesamten Versionshistorie gespeichert wird.

    • Was ist ein Merge?

  • ist der Prozess in Git, bei dem zwei oder mehr unterschiedlichen Branches zusammengeführt werden.

    • Was ist ein Merge Conflict?

  • es tritt auf, wenn zwei oder mehr Entwickler gleichzeitig an denselben Teilen einer Datei vorgenommen haben,
    Git nicht automatisch entscheiden kann, welche Änderungen übernommen werden soll. Es passiert oft, wenn
    mehrere Branches miteinander zusammengeführt werden und es Änderungen an derselben Zeilen oder sehr ähnlichen
    Bereich im Code gibt.

    • Welche Befehle braucht man?

    git add: um Änderungen an bestimmten Dateien für nächsten Commit vormerken
    git commit: erfasst man einen Snapshot, der aktuell bereitgestellten Änderungen des Projekts
    git pull: die neusten Änderungen werden an das Remote-Repository heruntergeladen
    git push: Der Befel git push wird verwendent, um eine lokalen Änderungen zu einem Remote-Repository hochzuladen
    sichert die Änderungen auf einem Remote-Server und sorgt dafür, dass deine Arbeit für andere Entwickler zugänglich ist

    UX/UI Design - Personas

    Definition und Begriffe
  • Mithilfe einer Persona ist es möglich, das Verhalten einer bestimmten Menschengruppe genau zu analy-
    sieren und Handlungsmuster somit besser einzuschätzen. Das Wort "Persona" stammt aus dem Lateinischen
    von persona zum Person. Personas sind fiktive Nutzertypen einer Website oder App.

    • Beispiele: Geschlecht, Familienzustand, Einkommen, Bildungsgrad etc..

  • Wichtig: Personas helfen dabei, die Nutzenden greifbarer zu machen.


    • Nutzen

    Wozu brauchen wir eine Persona?

    Es hilft Unternehmen dabei, ihre Zielgruppe besser einzugrenzen, zu charakterisieren und dementsprechend
    die Dienstleistungen und Produkte auf ebendiese anzupassen.

  • Punkt 1: Definition von UI/UX

    UI: UserInterface: es beschreibt wie das Produkt aussieht
    UX: UserExperience: beschreibt wie es mit dem Produkt agiert wird bzw. ob die Bedürfnisse und die Erwartungen erfüllt werden

  • Punkt 2: Wie kann ich UI/UX vorstellen?

    • UI ist ein Teil von UX, sie sind eng miteinander verbunden, aber haben dennoch verschiedene Schwerpunkte Beispiel: Ein Smartphone
    UI (Benutzeroberfläche) die Gestaltung der Apps, das Layout des Startbildschirm, sind alles Teil des UI
    UX (Benutzererfahrung) wie fühlt sich es an, das Smartphone zu benutzen, ist es einfach das Gerät zu bedienen.

  • Punkt 3: Wie betrifft UI/UX Design uns als Entwickler?

    • Es ist ein entscheidender Faktor für den Erfolg einer App. Wenn eine Anwendung dem Nutzer ein fließende,
    intuitives und ansprechendes Erlebnis bietet, kann dies zu einer höheren Nutzerbindung und
    einer zufriedenen Kundenfeedback führen.

  • Punkt 4: Was macht ein gutes UI/UX Design aus?

    • die ein Unternehmen langfristig zum Wachstum bringen können, das UI/UX Design spielt heute eine Schlüsselrolle für den Erfolg von Mobil-und Webanwendung. Ein gutes UI oder UX Design sollten keine Priorität haben,
    sondern Funktionen, die Hand in Hand gehen und Praktabilität nahtlos miteinander verbinden.

  • Punkt 5: Die Merkmale von schlechten UI/UX Design!

    • unter schlechtem Design versteht man die Entwicklung digitaler Produkte, die den Bedürfnissen und Erwartungen
    der Nutzer nicht gerecht oder ünerfüllt werden.
    Beispiele: schlechte Navigation, mangelnde Intuitivität und Unübersichtlichkeit

  • Punkt 6: Do's and dont's in UI/UX Design:

    • dont's: Schlechte Bedienbarkeit / Unübersichtiliche Struktur / mangelnde Performance
    do's: Klarheit / Performance / Verbesserung
    Schlüsselsatz: Die Schritte sollen sicherstellen, dass das Produkt sowohl ästhetisch ansprechend als auch benutzerfreundlich ist.

  • Punkt 7: ein Paar interessante Beispiele für inspiriendes UI/UX Design:

    • 1. Spotify: bietet ein schlankes und intuitives Design, leicht navigierbar und ermöglicht eine Nahtlose Musiksuche
    2. Slack: ein gut durchdachtes Design die Kommunikation und Zusammenarbeit im Team effizienter gestalten kann
    3. Airbnb: die Bilder das Design ist benutzerfreundlich und attraktiv, schnell und einfach das gewünschte Ziel finden

    AUFGABE 2
    Was ist Git?
  • ist eine freie Software zur verteilten Versionsverwaltung von Dateien es speichert und betrachtet
    die Informationen auf eine ganz andere Weise.

    • Was ist Git CLI?
  • ist ein Befehlszeilentool, bedeutet nichts anders wie Verwendung von Issues, GitHub Actions-Vorgängen
    und anderen GitHub Features, in deinem (euren) Terminal ermöglicht, sodass man die Arbeit an einem Ort bewältigen kann.

    • Was ist GitHub?
  • ist ein Online-Plattform zur Versionsverwaltung und gemeinsamen Arbet an Softwareprojekten, bietet Entwickler
    eine zentrale Anlaufstelle, um Code zu speichern, zu verwalten und gemeinsam weiterzuentwickeln.

    • Beispiele für die Nutzung von GitHub?
    1. Open-Source-Projekte: viele bekannte Softwareprojekte wie Linux, React werden auf Github gehostet, dient
    dient dazu das die Entwickler weltweit an deren Weiterentwicklung mitarbeiten und teilhaben können.
    2. Dokumentation: wird häufig verwendet, um Dokumentationen, Handbücher oder Tutorials in Form von
    "Markdown-Dateien" bereitzustellen.
